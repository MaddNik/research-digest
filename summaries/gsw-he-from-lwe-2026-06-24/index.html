Homomorphic Encryption from Learning with Errors: Conceptually-Simpler, Asymptotically-Faster, Attribute-Based

CRYPTO 2013 (Springer LNCS 8042); IACR ePrint 2013/340 · full view ↗

Problem and motivation: why LWE-FHE multiplication was hard

Fully homomorphic encryption (FHE) lets a party compute on ciphertexts without decrypting, producing an encryption of the result. By 2013 the simplest leveled FHE schemes based on the Learning With Errors (LWE) problem were those of Brakerski-Vaikuntanathan (BV11b), Brakerski-Gentry-Vaikuntanathan (BGV12), and Brakerski (Bra12). In all of these, a ciphertext $c$ and the secret key $s$ are $n$-dimensional vectors over $\mathbb{Z}_q$ whose dot product encodes the message $\mu$ up to a small error removed by rounding, so that $\langle c, s\rangle \approx \mu$. Homomorphic addition is just $c_1 + c_2$, which is easy. Multiplication is the hard part.

Multiplication in those schemes uses the tensor-product identity for dot products,

$$\langle u_1 \otimes u_2,\; v_1 \otimes v_2\rangle = \langle u_1, v_1\rangle \cdot \langle u_2, v_2\rangle.$$

This is Equation 1. Hence if $\langle c_1, s\rangle \approx \mu_1$ and $\langle c_2, s\rangle \approx \mu_2$, then $\langle c_1 \otimes c_2,\; s \otimes s\rangle \approx \mu_1 \cdot \mu_2$, where $c_1 \otimes c_2$ is the new ciphertext and $s \otimes s$ the new key, each of dimension $\Theta(n^2)$. Because tensoring blows up the ciphertext dimension, it can only run for a constant number of steps before the evaluator must relinearize: take the long $\Theta(n^2)$-dimensional ciphertext encrypting $\mu_1 \mu_2$ under $s \otimes s$ and compress it back into a normal $n$-dimensional ciphertext encrypting $\mu_1 \mu_2$ under a normal key $s'$. Concretely the evaluator multiplies the long ciphertext by a special $n \times \Theta(n^2)$ relinearization matrix.

Relinearization was the central insight that first enabled LWE-FHE (BV11b), but the authors argue it is unnatural, hard to explain intuitively, and costly. Each relinearization matrix has size $\Omega(n^3)$, and the public key must hold $L$ of them to support multiplicative depth $L$; the procedure costs $\Omega(n^3)$ operations per multiply, each operation having cost polynomial in $L$. These matrices constitute the user-specific evaluation key the evaluator must obtain. The paper poses the guiding question: can homomorphic add and multiply just be $c_1 + c_2$ and $c_1 \cdot c_2$ over some ring, with the product having the same form (not a long ciphertext), and with no evaluation key at all? Their answer is yes, via the approximate-eigenvector method.

Main results and honest caveats

The contributions are:

  • Conceptually simpler FHE from LWE. The whole scheme is described in the paper's introduction; homomorphic add and multiply are (almost) just matrix add and multiply.
  • Asymptotically faster FHE from LWE. Relinearization and its $\Omega(n^3)$ matrices are eliminated. Ciphertexts are matrices multiplied naturally, so sub-cubic matrix multiplication applies; the authors cite Strassen at $n^{2.807}$ (Str69) and Williams at $n^{2.3727}$ (Wil12).
  • No evaluation key. The evaluator needs only basic public parameters such as $n$, $m$, $\ell$, never a user-specific key.
  • Identity-based FHE (IBFHE). Because there is no evaluation key, the authors give a compiler that turns any suitable LWE-based IBE (GPV08, ABB10a, ABB10b, CHKP10) into an IBFHE, solving an open problem from Nac10, GHV10, Bra12, CHT13.
  • Attribute-based FHE (ABFHE). The same compiler, with minor changes, compiles the Gorbunov-Vaikuntanathan-Wee (GVW13) ABE-for-circuits into an ABFHE in which data encrypted under a common index can be processed homomorphically.

The scheme keeps the usual LWE-FHE advantages: bootstrapping is optional (BGV12); with bootstrapping, security rests on quasi-polynomial rather than sub-exponential LWE factors; modulus switching is eliminated (Bra12); and security can be based directly on classical $\mathrm{GapSVP}$ (Bra12).

The authors are careful not to oversell. LWE-FHE remains much slower than ring-LWE (RLWE) FHE, and even RLWE-FHE was not yet practical at the time. An RLWE version of this scheme is worse than the best RLWE schemes by log factors. Sub-cubic matrix multiplication may not beat cubic by much in practice. They view the asymptotic time result mainly as evidence of how fundamentally new the techniques are; the concrete practical gain is space, dropping from quasi-cubic to quasi-quadratic, a significant issue for LWE-FHE in practice.

The approximate-eigenvector method

The core idea: the secret key is an approximate eigenvector of the ciphertext matrix, and the message is the eigenvalue. For a modulus $q$ and a dimension $N$ (fixed later), a ciphertext $C$ is an $N \times N$ matrix over $\mathbb{Z}_q$ with small entries (much smaller than $q$), and the secret key $v$ is an $N$-dimensional vector over $\mathbb{Z}_q$ with at least one big coefficient $v_i$. The message $\mu$ is restricted to a small integer. We say $C$ encrypts $\mu$ when

$$C \cdot v = \mu \cdot v + e,$$

which is Equation 2, with $e$ a small error vector. To decrypt, extract row $C_i$, compute $x \leftarrow \langle C_i, v\rangle = \mu \cdot v_i + e_i$, and round to recover $\mu = \lfloor x / v_i \rceil$.

The homomorphic operations follow from elementary linear algebra. Suppose $C_1 \cdot v = \mu_1 \cdot v + e_1$ and $C_2 \cdot v = \mu_2 \cdot v + e_2$. For addition, $C_+ = C_1 + C_2$ gives

$$C_+ \cdot v = (\mu_1 + \mu_2)\cdot v + (e_1 + e_2),$$

which is Equation 3: the sum of ciphertext matrices encrypts the sum of messages, provided the error is still small. For multiplication, $C_\times = C_1 \cdot C_2$ gives

$$C_\times \cdot v = C_1\cdot(\mu_2 v + e_2) = \mu_2(\mu_1 v + e_1) + C_1 e_2 = \mu_1 \mu_2 \cdot v + \mu_2 e_1 + C_1 e_2 = \mu_1 \mu_2 \cdot v + \text{small},$$

which is Equation 4: the matrix product encrypts the message product, as long as the final error stays small (it does, because $\mu_2$, $C_1$, $e_1$, $e_2$ are all small). Interestingly $C_2 \cdot C_1$ also encrypts $\mu_1\mu_2$, even though matrix multiplication is non-commutative. The reason is visible in an idealized error-free picture where $C_i \cdot v = \mu_i \cdot v$ exactly: then $v$ is a genuine eigenvector of each $C_i$ with eigenvalue $\mu_i$, and if two matrices share eigenvector $v$ with eigenvalues $\mu_1, \mu_2$, then both $C_1 C_2$ and $C_2 C_1$ have eigenvector $v$ with eigenvalue $\mu_1 \mu_2$. Introducing the error $e$ (necessary to base security on LWE) makes $v$ only an approximate eigenvector, which is exactly why certain terms must be kept small so that homomorphic operations do not disrupt the ciphertext form. The authors name this the approximate eigenvector method.

Noise growth without flattening: a somewhat-homomorphic scheme

Before fully specifying the scheme, the authors estimate how homomorphic it is. The scheme works correctly until the coefficients of the error vector approach $q$ in magnitude. Call $C$ a $B$-bounded ciphertext when $\mu$ and the coefficients of $C$ and $e$ all have magnitude at most $B$. Then $C_+$ is $2B$-bounded and $C_\times$ is $(N+1)B^2$-bounded. So the error level grows worse than $B^{2^L}$, doubly exponentially in the multiplicative depth $L$.

Phrased by degree rather than depth: evaluating a multivariate polynomial $P(x_1,\dots,x_t)$ of total degree $d$ on $B$-bounded inputs yields a ciphertext that is $|P|(N+1)^{d-1}B^{d}$-bounded, where $|P|$ is the $\ell_1$-norm of $P$'s coefficient vector. Taking $q$ to comfortably exceed this bound, one can roughly evaluate polynomials of degree $\log_{NB} q$. Since security requires $q/B$ to be at most sub-exponential in $N$, the scheme-so-far evaluates only sublinear polynomial degree in $N$, equivalently only logarithmic depth. This is a somewhat-homomorphic encryption (SWHE) scheme: by far the most homomorphic LWE-based scheme using only natural operations, but not yet FHE.

The gadget toolkit: BitDecomp, Powersof2, Flatten, and the matrix G

To obtain leveled FHE without bootstrapping or relinearization, the error must grow only singly-exponentially in $L$. The fix is flattening, built on simple bit-decomposition transformations from BV11b, BGV12, and Bra12 that change a vector without affecting its dot products. Let $a, b$ be $k$-dimensional vectors over $\mathbb{Z}_q$, let $\ell = \lfloor \log_2 q\rfloor + 1$, and $N = k \cdot \ell$.

BitDecomp(a) = (a_{1,0}, ..., a_{1,ell-1}, ..., a_{k,0}, ..., a_{k,ell-1})
    where a_{i,j} is the j-th bit of a_i (least significant bit first).
    Result is an N-dimensional 0/1 vector.

BitDecomp^{-1}(a') = ( sum_j 2^j . a_{1,j}, ..., sum_j 2^j . a_{k,j} )
    the inverse of BitDecomp, but well-defined even when the
    input a' is NOT a 0/1 vector.

Flatten(a') = BitDecomp( BitDecomp^{-1}(a') )
    an N-dimensional vector with 0/1 coefficients.

Powersof2(b) = (b_1, 2.b_1, ..., 2^{ell-1}.b_1, ...,
                b_k, 2.b_k, ..., 2^{ell-1}.b_k)
    an N-dimensional vector.

For a matrix A, BitDecomp(A) / BitDecomp^{-1}(A) / Flatten(A)
    apply the operation to each row of A separately.

The key dot-product identities (which hold without knowing $b$) are

$$\langle \mathrm{BitDecomp}(a),\, \mathrm{Powersof2}(b)\rangle = \langle a, b\rangle,$$

which is Equation 5, and for any $N$-dimensional $a'$,

$$\langle a',\, \mathrm{Powersof2}(b)\rangle = \langle \mathrm{BitDecomp}^{-1}(a'),\, b\rangle = \langle \mathrm{Flatten}(a'),\, \mathrm{Powersof2}(b)\rangle,$$

which is Equation 6. The interesting feature is that $\mathrm{Flatten}$ makes the coefficients of a vector or matrix small ($0/1$) without affecting its product with $\mathrm{Powersof2}(b)$, and without knowing $b$. The scheme therefore gives the secret key the special form $v = \mathrm{Powersof2}(s)$ for a secret vector $s$. Since $v$'s coefficients climb by $\lfloor \log_2 q\rfloor$ powers of $2$, $v$ automatically has a big coefficient suitable for recovering $\mu \in \{0,1\}$. Consequently, for any $N \times N$ matrix $C$,

$$\mathrm{Flatten}(C) \cdot v = C \cdot v,$$

which is Equation 7, the crucial identity behind flattening. The matrix $G = \mathrm{BitDecomp}^{-1}(I_N)$ is (the transpose of) the Micciancio-Peikert primitive (gadget) matrix used in lattice trapdoors (MP12). The authors remark that $\mathrm{BitDecomp}$ could alternatively decompose with respect to bases other than $2$, or by the Chinese Remainder Theorem.

Flattening yields leveled FHE

Call $C$ a $B$-strongly-bounded ciphertext if its $\mu$ and all coefficients of $C$ have magnitude at most $1$, while the coefficients of its error $e$ have magnitude at most $B$. Evaluating a NAND gate as

$$C_3 \leftarrow I_N - C_1 \cdot C_2$$

(Equation 8) keeps the message in $\{0,1\}$ and produces error of magnitude at most $(N+1)B$, but the coefficients of $C_3$ are no longer small. Applying $\mathrm{Flatten}$ fixes exactly this: because $\mathrm{Flatten}(C) \cdot v = C \cdot v$, setting $C_{\mathrm{NAND}} = \mathrm{Flatten}(C_3)$ produces a ciphertext with $0/1$ coefficients that is still strongly bounded. Iterating a depth-$L$ NAND circuit keeps the error at most $(N+1)^L \cdot B$, only singly-exponential in $L$. With $q/B$ sub-exponential in $N$, the scheme can now evaluate circuits of polynomial depth rather than merely polynomial degree, giving a leveled FHE scheme, with no relinearization and under a single fixed approximate-eigenvector secret key. (Leveled FHE means the parameters may depend on the circuit depth but not its size; it converts to pure FHE via Gentry's bootstrapping plus a circular-security assumption.)

LWE preliminaries and the security reduction

LWE (Definition 1). For security parameter $\lambda$, dimension $n = n(\lambda)$, modulus $q = q(\lambda) \ge 2$, and an error distribution $\chi$ over $\mathbb{Z}$, the $\mathrm{LWE}_{n,q,\chi}$ problem is to distinguish two distributions of samples $(a_i, b_i) \in \mathbb{Z}_q^{n+1}$. In the first, $(a_i, b_i)$ is uniform. In the second, one draws $s \leftarrow \mathbb{Z}_q^n$ uniformly, then samples $a_i \leftarrow \mathbb{Z}_q^n$ uniformly, $e_i \leftarrow \chi$, and sets $b_i = \langle a_i, s\rangle + e_i$. Viewing the rows $b_i \| a_i$ as a matrix $A$ and redefining $s$ as $(1, -s)$, either $A$ is uniform, or there is a vector $s$ with first coefficient $1$ such that $A \cdot s = e$ with $e$ drawn from $\chi$.

GapSVP and hardness (Theorems 1 and 2). A distribution ensemble $\{\chi_n\}$ is $B$-bounded (Definition 2) if $\Pr_{e \leftarrow \chi_n}[|e| > B] = \mathrm{negl}(n)$. By Reg05, Pei09, MM11, MP12 (as restated in Bra12), for $q$ a prime power or product of small distinct primes and $B \ge \omega(\log n)\cdot\sqrt{n}$, there is an efficiently sampleable $B$-bounded $\chi$ such that solving average-case $\mathrm{LWE}_{n,q,\chi}$ yields an efficient quantum algorithm for $\mathrm{GapSVP}_{\tilde{O}(nq/B)}$, and if $q \ge \tilde{O}(2^{n/2})$ an efficient classical one. The BLP+13 result (Theorem 2) shows that solving $n$-dimensional LWE with $\mathrm{poly}(n)$ modulus implies an equally efficient solution to a worst-case lattice problem such as $\mathrm{GapSVP}$ in dimension $\sqrt{n}$.

Key generation as an LWE instance. For parameters $q$, $n$, $m = O(n \log q)$, the public key $A$ is an $m \times (n+1)$ matrix such that there is an $(n+1)$-dimensional $s$ with first coefficient $1$ and $A \cdot s = e$ small. The approximate eigenvector is $v = \mathrm{Powersof2}(s)$, of dimension $N = (n+1)\cdot\ell$ with $\ell = \lfloor \log_2 q\rfloor + 1$. To encrypt $\mu \in \mathbb{Z}_q$, sample a uniform $0/1$ matrix $R \in \{0,1\}^{N\times m}$ and set

$$C = \mathrm{Flatten}\big(\mu \cdot I_N + \mathrm{BitDecomp}(R \cdot A)\big) \in \mathbb{Z}_q^{N\times N},$$

which is Equation 9. Because $\mathrm{Flatten}$ does not affect the product with $v$,

$$C \cdot v = \mu \cdot v + \mathrm{BitDecomp}(R \cdot A)\cdot v = \mu \cdot v + R \cdot A \cdot s = \mu \cdot v + R \cdot e,$$

which is Equation 10; the last term is small because $A \cdot s = e$ is small.

Security. Let $C' = \mathrm{BitDecomp}^{-1}(C)$. Because $C$ is $\mathrm{Flatten}$'d, $C = \mathrm{BitDecomp}(C')$, so $C$ reveals nothing more than $C'$. And

$$\mathrm{BitDecomp}^{-1}(C) = \mu \cdot G + R \cdot A,$$

which is Equation 11, where $G = \mathrm{BitDecomp}^{-1}(I_N)$ is the Micciancio-Peikert primitive matrix and the rows of $R \cdot A$ are simply Regev encryptions of $0$. By Lemma 1 (implicit in Reg05), if $\mathrm{LWE}_{n,q,\chi}$ holds and $m = O(n \log q)$, the joint distribution $(A, R\cdot A)$ is computationally indistinguishable from uniform over $\mathbb{Z}_q^{m\times(n+1)} \times \mathbb{Z}_q^{N\times(n+1)}$; concretely it suffices to take $m > 2n \log q$. Therefore $C$ is statistically independent of $\mu$ and security reduces directly to LWE. Security can also rest on classical $\mathrm{GapSVP}$ via Pei09 and BLP+13. Unlike previous FHE schemes, the fresh-ciphertext error bound $B$ need not be small; only $q/B$ must be sub-exponential, so $q$ may even be exponential.

The formal scheme: Setup, key generation, Enc, Dec, MPDec

$\mathrm{KeyGen}$ is split into $\mathrm{Setup}$, $\mathrm{SecretKeyGen}$, and $\mathrm{PublicKeyGen}$. Two decryption routines are given: $\mathrm{Dec}$ recovers a small message $\mu$ (e.g. in $\{0,1\}$), and $\mathrm{MPDec}$ is the Micciancio-Peikert procedure (MP12) that recovers any $\mu \in \mathbb{Z}_q$.

Setup(1^lambda, 1^L):
    Choose modulus q of kappa = kappa(lambda,L) bits, lattice
    dimension n = n(lambda,L), and error distribution chi for
    at least 2^lambda security against known attacks. Also choose
    m = m(lambda,L) = O(n log q).
    Let params = (n, q, chi, m).
    Let ell = floor(log q) + 1,   N = (n+1) . ell.

SecretKeyGen(params):
    Sample t <- Z_q^n.
    Output sk = s <- (1, -t_1, ..., -t_n) in Z_q^{n+1}.
    Let v = Powersof2(s).

PublicKeyGen(params, sk):
    Generate B <- Z_q^{m x n} uniformly,  e <- chi^m.
    Set b = B . t + e.
    Set A = [ b | the n columns of B ]   (an m x (n+1) matrix).
    Output pk = A.    (Remark: A . s = e.)

Enc(params, pk, mu):    for mu in Z_q
    Sample R in {0,1}^{N x m} uniformly.
    Output  C = Flatten( mu . I_N + BitDecomp(R . A) ) in Z_q^{N x N}.

Dec(params, sk, C):     (small message, e.g. {0,1})
    The first ell coefficients of v are 1, 2, ..., 2^{ell-1}.
    Among them pick v_i = 2^i in (q/4, q/2].
    Let C_i be row i of C. Compute x_i = <C_i, v>.
    Output mu' = round( x_i / v_i ).

MPDec(params, sk, C):   (any mu in Z_q; here q a power of 2)
    q = 2^{ell-1}; the first ell-1 coeffs of v are 1, 2, ..., 2^{ell-2}.
    If C.v = mu.v + small, the first ell-1 coeffs of C.v are
      mu.g + small  with  g = (1, 2, ..., 2^{ell-2}).
    Recover LSB(mu) from  mu . 2^{ell-2} + small;
    recover the next bit from (mu - LSB(mu)) . 2^{ell-3} + small;
    continue for each bit.  (See MP12 for the general q case.)

Correctness. If $C$ is well-formed then $C \cdot v = \mu \cdot v + R \cdot A \cdot s = \mu \cdot v + R \cdot e$. $\mathrm{Dec}$ uses only coefficient $i$, namely $x_i = \mu \cdot v_i + \langle R_i, e\rangle$, whose error has magnitude at most $\lVert e\rVert_1$. In general, if $x_i = \mu \cdot v_i + e'$ with $|e'| \le q/8$ and $v_i \in (q/4, q/2]$, then $x_i/v_i$ differs from $\mu$ by less than $(q/8)/v_i < 1/2$, so rounding recovers $\mu$. $\mathrm{Dec}$ is simply a $\mathrm{BitDecomp}$'d version of Regev decryption applied to a single row of $C$ (itself a $\mathrm{BitDecomp}$'d Regev ciphertext); the extra rows of $C$ matter only for the homomorphic operations. For the basic (non-homomorphic) scheme one may take $n$ quasi-linear in $\lambda$ and $\kappa = O(\log n)$.

Homomorphic operations: MultConst, Add, Mult, NAND

All four operations end with $\mathrm{Flatten}$ so the output is strongly bounded. The earlier primitives $\mathrm{BitDecomp}$, $\mathrm{BitDecomp}^{-1}$, $\mathrm{Flatten}$, $\mathrm{Powersof2}$ are themselves treated as basic homomorphic operations and underlie the analysis.

MultConst(C, alpha):    multiply ciphertext C by known alpha in Z_q
    M_alpha <- Flatten(alpha . I_N)
    output Flatten(M_alpha . C)

Add(C1, C2):
    output Flatten(C1 + C2)

Mult(C1, C2):
    output Flatten(C1 . C2)

NAND(C1, C2):   for mu1, mu2 in {0,1}
    output Flatten(I_N - C1 . C2)

$\mathrm{MultConst}$. With $M_\alpha = \mathrm{Flatten}(\alpha \cdot I_N)$,

$$\mathrm{MultConst}(C,\alpha)\cdot v = M_\alpha \cdot C \cdot v = M_\alpha(\mu v + e) = \alpha \mu \cdot v + M_\alpha e,$$

which is Equation 12. The error grows by a factor of at most $N$ regardless of which $\alpha \in \mathbb{Z}_q$ is used. Multiplying by a constant via repeated $\mathrm{Add}$ would instead multiply the error by at least $\alpha$. This enables native homomorphic FFTs over $\mathbb{Z}_q$ without error growth dependent on $q$, unlike GHS12a/GHS12b where error growth depended on the underlying field size.

$\mathrm{Add}$. Correctness is immediate; message addition is over the full base ring $\mathbb{Z}_q$.

$\mathrm{Mult}$.

$$\mathrm{Mult}(C_1,C_2)\cdot v = C_1 C_2 v = \mu_1\mu_2 \cdot v + \mu_2 e_1 + C_1 e_2,$$

which is Equation 13. $C_1$ contributes a factor of at most $N$ because its entries are in $\{0,1\}$, but the $\mu_2 e_1$ term means the message magnitude affects error growth, so the message space must be kept small (for example by using Boolean NAND circuits).

$\mathrm{NAND}$. For $\mu_1, \mu_2 \in \{0,1\}$,

$$\mathrm{NAND}(C_1,C_2)\cdot v = (I_N - C_1 C_2)\cdot v = (1 - \mu_1\mu_2)\cdot v - \mu_2 e_1 - C_1 e_2,$$

which is Equation 14. This keeps the message in $\{0,1\}$ and increases the error by a factor of at most $N+1$.

Circuits. A Boolean circuit converted to NAND gates, evaluated at NAND-depth $L$ over fresh ciphertexts with error bound $B$, yields error at most $(N+1)^L \cdot B$. For arithmetic circuits over $\mathbb{Z}_q$ where all inputs are bounded by $T$ and all intermediate values stay bounded by $T'$, the final error is at most $(N+T')^L \cdot B$; this gives somewhat-homomorphic evaluation of degree-$d$ polynomials when initial messages are bounded by roughly $q^{1/d}$. One can also use an arithmetic circuit to convert encryptions of a polynomially bounded set of small values into encryptions of binary values, after which a NAND-based Boolean circuit handles further computation.

Parameters, complexity, and bootstrapping

If $\mathrm{Flatten}$'d ciphertexts $C_1, C_2$ encrypt bits in $\{0,1\}$ under $v$ with $B$-bounded error (so $C_i \cdot v = \mu_i \cdot v + e_i$ with $|e_i|_\infty \le B$), then $\mathrm{NAND}(C_1, C_2)$ encrypts $\mathrm{NAND}(\mu_1, \mu_2) \in \{0,1\}$ with $(N+1)B$-bounded error. As long as

$$q/B > 8(N+1)^L,$$

which is Equation 15, a depth-$L$ NAND circuit over $B$-bounded ciphertexts yields a $q/8$-bounded ciphertext that $\mathrm{Dec}$ decrypts correctly. To maintain $2^\lambda$ security against known attacks, $n$ (hence $N$) must grow linearly with $\log(q/B)$, so $q/B$ grows like $\exp(L \log L)$. With $\kappa = \log q = O(L \log N) = O(L \log n)$ and $N = O(n\kappa) = \tilde{O}(nL)$, each NAND is dominated by one $N \times N$ matrix multiplication.

Theorem 3. For dimension $n$ and depth $L$, the scheme evaluates depth-$L$ NAND circuits with $\tilde{O}((nL)^{\omega})$ field operations per gate, where $\omega < 2.3727$ is the matrix-multiplication exponent. Previous LWE-FHE schemes (BV11b, BGV12, Bra12) need at least $\tilde{O}(n^3 L)$ field operations per gate. The theorem hides factors in both directions. On the good side, this scheme's $0/1$ ciphertext entries multiply faster than general $\mathbb{Z}_q$ matrices, and previous schemes' field operations multiply a small number by a general $\mathbb{Z}_q$ element at cost $\tilde{O}(\kappa) = \tilde{O}(L)$, giving them real complexity $\tilde{O}(n^3 L^2)$ versus this scheme's $\tilde{O}((nL)^{\omega})$. On the bad side, $N = O(nL \log n)$ hides log factors. Typically $n$ dominates $L$, since for very deep circuits one would bootstrap to make per-gate cost independent of $L$.

Bootstrapping. Because $\mathrm{Dec}$ is essentially Regev decryption, evaluable in $O(\log n)$ depth, bootstrapping works as in prior LWE schemes; BV11b-style dimension and modulus reduction make decryption complexity independent of the evaluated circuit's depth. The logarithmic decryption depth lets $q/B$ be quasi-polynomial in $n$, basing security on quasi-polynomial LWE factors. The caveat is that bootstrapping requires the user's secret key encrypted under its public key, in effect an evaluation key of quasi-cubic size, which (even with public-key compression, CNT12) removes both the identity-based property and the space advantage. Finding a natural alternative to bootstrapping, and achieving pure FHE without circular security, are left open.

No evaluation key, and the IBFHE / ABFHE compiler

The decisive structural property is that homomorphic evaluation needs only public parameters (such as $n$, $m$, $\ell$), never a user-specific evaluation key. This removes the obstacle that had kept identity-based FHE open. Prior FHE always required an evaluation key that cannot be computed non-interactively from a user's identity, undermining IBE's central appeal of non-interactivity; earlier IBE-with-homomorphism work (GHV10, CHT13) achieved only weak IBFHE where the evaluator still needs a user-specific key. Making merely the encryption/decryption keys identity-based (e.g. via dual-Regev, GPV08) solves only half the problem.

The compiler turns any LWE-based IBE $E$ with three properties into an IBFHE. Property 1 (ciphertext and key vectors): the decryption key $s_{\mathrm{ID}}$ and a ciphertext $c_{\mathrm{ID}}$ for identity $\mathrm{ID}$ are vectors in $\mathbb{Z}_q^{n'}$ with the first coefficient of $s_{\mathrm{ID}}$ equal to $1$. Property 2 (small dot product): if $c_{\mathrm{ID}}$ encrypts $0$ then $\langle c_{\mathrm{ID}}, s_{\mathrm{ID}}\rangle$ is small. Property 3 (security): encryptions of $0$ are indistinguishable from uniform vectors over $\mathbb{Z}_q$ under LWE.

Theorem 4 (IBE -> IBFHE compiler):
    Use E's Setup and KeyGen; supplement E's MPK with the FHE
    parameters (such as m, ell).
    Let N = (n+1) . ell,   ell = floor(log q) + 1.

    Enc(MPK, ID, mu):   for mu in {0,1}
        generate N encryptions of 0 via E.Enc;
        let C'_ID be the N x (n+1) matrix whose rows are these;
        output  C_ID = Flatten( mu . I_N + BitDecomp(C'_ID) ).

    Dec(sk_ID, C_ID):
        v_ID = Powersof2(s_ID);
        run the FHE Dec(v_ID, C_ID) to recover mu.
        Correctness: C_ID . v_ID
            = mu . v_ID + C'_ID . s_ID
            = mu . v_ID + small      (small by Property 2).

    Eval: homomorphic operations exactly as in Section 3.3.

Semantic security follows: any adversary breaking the IBFHE distinguishes $C'_{\mathrm{ID}}$ from uniform and hence breaks LWE (Property 3). All known LWE-based IBE schemes (GPV08, ABB10a, ABB10b, CHKP10) satisfy these properties. For ABFHE the authors re-interpret decryption in the GVW13 ABE: from a key $s_y$ with $R(x,y)=1$ one derives a sub-key $s_{x,y}$ satisfying a Property-2-like small-dot-product condition (if $c_x$ encrypts $0$ under index $x$ then $\langle c_x, s_{x,y}\rangle$ is small), so the same compiler applies, yielding the first ABFHE in which ciphertexts under a common index are processed homomorphically with no evaluation key, decryptable by any party entitled to decrypt the originals. Full IBE/ABE details are deferred to the full version of the paper.

Significance: foundation of FHEW and TFHE

This scheme, now universally called GSW after its authors, is the conceptual foundation of the FHEW and TFHE line of fast-bootstrapping schemes. Two GSW features are decisive for them. First, the asymmetric noise growth of $\mathrm{Mult}$: since $C_1$'s entries are $0/1$, the $C_1 e_2$ term contributes only a factor-$N$ blowup (Equation 13), so a chain of GSW products grows error roughly additively rather than multiplicatively. This makes GSW the natural vehicle for the homomorphic accumulator at the heart of FHEW/TFHE bootstrapping, where many controlled GSW products are chained. Second, the gadget machinery ($G$, $\mathrm{BitDecomp}$, $\mathrm{Flatten}$, equivalently the $G$ and $G^{-1}$ decomposition) is exactly the external-product structure that TFHE generalizes to its GGSW ciphertexts and the external product (a GGSW times a GLWE giving a GLWE), the operation driving the blind-rotation step of programmable bootstrapping. The approximate-eigenvector view (eigenvalue as message) and the absence of an evaluation key carry directly into how FHEW and TFHE structure their bootstrapping keys and their CMux / blind-rotation gates.

Algorithms

Complete, self-contained pseudocode for the GSW scheme and its IBFHE compiler. Throughout, $\ell = \lfloor \log_2 q\rfloor + 1$, $N = (n+1)\cdot\ell$, and $I_N$ is the $N \times N$ identity matrix. $\mathrm{Flatten}$, $\mathrm{BitDecomp}$, $\mathrm{BitDecomp}^{-1}$, and $\mathrm{Powersof2}$ are as defined in the gadget toolkit section.

Algorithm A1: Gadget primitives
--------------------------------
BitDecomp(a):                       // a in Z_q^k
    Input : k-dim vector a over Z_q
    Output: N-dim 0/1 vector
    1. for i = 1..k:
    2.     for j = 0..ell-1:
    3.         emit bit j of a_i (LSB first)
    4. return the concatenated N-dim vector

BitDecomp^{-1}(a'):                 // a' need NOT be 0/1
    Input : N-dim vector a'=(a_{1,0..ell-1},...,a_{k,0..ell-1})
    Output: k-dim vector over Z_q
    1. for i = 1..k:
    2.     b_i = sum_{j=0..ell-1} 2^j . a_{i,j}
    3. return (b_1, ..., b_k)

Flatten(a'):
    Input : N-dim vector a'
    Output: N-dim 0/1 vector
    1. return BitDecomp( BitDecomp^{-1}(a') )

Powersof2(b):                       // b in Z_q^k
    Input : k-dim vector b over Z_q
    Output: N-dim vector
    1. return (b_1,2.b_1,...,2^{ell-1}.b_1, ...,
               b_k,2.b_k,...,2^{ell-1}.b_k)

// On matrices, all four act row-by-row.
// Identities: <BitDecomp(a),Powersof2(b)> = <a,b>;
//             <a',Powersof2(b)> = <BitDecomp^{-1}(a'),b>
//                               = <Flatten(a'),Powersof2(b)>;
//             Flatten(C).v = C.v  when v = Powersof2(s).
Algorithm A2: Key generation, encryption, decryption
----------------------------------------------------
Setup(1^lambda, 1^L):
    1. choose q (kappa bits), n, error dist chi for 2^lambda security
    2. choose m = O(n log q)
    3. ell = floor(log q)+1;  N = (n+1).ell
    4. return params = (n, q, chi, m)

SecretKeyGen(params):
    1. t <- Z_q^n
    2. s = (1, -t_1, ..., -t_n) in Z_q^{n+1}
    3. v = Powersof2(s)
    4. return sk = s  (and v)

PublicKeyGen(params, sk):
    1. B <- Z_q^{m x n} uniform;  e <- chi^m
    2. b = B.t + e
    3. A = [ b | columns of B ]      // m x (n+1);  A.s = e
    4. return pk = A

Enc(params, pk, mu):                 // mu in Z_q
    1. R <- {0,1}^{N x m} uniform
    2. C = Flatten( mu.I_N + BitDecomp(R.A) )
    3. return C in Z_q^{N x N}        // C.v = mu.v + R.e

Dec(params, sk, C):                  // small mu, e.g. {0,1}
    1. among v's first ell coeffs (1,2,...,2^{ell-1})
         pick v_i = 2^i in (q/4, q/2]
    2. x_i = <C_i, v>                  // row i of C
    3. return mu' = round( x_i / v_i )

MPDec(params, sk, C):                // any mu in Z_q; q = 2^{ell-1}
    1. g = (1,2,...,2^{ell-2});  first ell-1 coeffs of C.v = mu.g+small
    2. recover LSB(mu) from mu.2^{ell-2}+small
    3. recover next bit from (mu - LSB(mu)).2^{ell-3}+small
    4. repeat for each remaining bit; return mu
Algorithm A3: Homomorphic gates
-------------------------------
MultConst(C, alpha):                 // alpha in Z_q known
    1. M_alpha = Flatten(alpha.I_N)
    2. return Flatten(M_alpha . C)    // err x N; encrypts alpha.mu

Add(C1, C2):
    1. return Flatten(C1 + C2)        // encrypts mu1 + mu2 over Z_q

Mult(C1, C2):
    1. return Flatten(C1 . C2)        // encrypts mu1.mu2;
                                      // err = mu2.e1 + C1.e2

NAND(C1, C2):                        // mu1, mu2 in {0,1}
    1. return Flatten(I_N - C1 . C2)  // encrypts NAND(mu1,mu2);
                                      // err factor <= N+1

// Depth-L NAND circuit over B-bounded fresh ciphertexts:
//   final error <= (N+1)^L . B;  correct if q/B > 8(N+1)^L.
Algorithm A4: IBE -> IBFHE compiler (Theorem 4)
-----------------------------------------------
// E is an LWE-based IBE with Properties 1-3.
Setup, KeyGen: as in E, MPK augmented with (m, ell).

Enc(MPK, ID, mu):                    // mu in {0,1}
    1. generate N encryptions of 0 via E.Enc(MPK, ID, 0)
    2. C'_ID = N x (n+1) matrix with those as rows
    3. return C_ID = Flatten( mu.I_N + BitDecomp(C'_ID) )

Dec(sk_ID = s_ID, C_ID):
    1. v_ID = Powersof2(s_ID)
    2. return Dec(v_ID, C_ID)         // C_ID.v_ID = mu.v_ID + small

Eval: apply Add / Mult / NAND from Algorithm A3.
// ABFHE: same compiler, replacing s_ID by the GVW13 sub-key s_{x,y}
// derived from sk_y when R(x,y)=1 (satisfies the Property-2 bound).
