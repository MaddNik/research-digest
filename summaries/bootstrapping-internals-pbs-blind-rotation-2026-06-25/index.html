Bootstrapping Internals: PBS, Blind Rotation, Sample Extraction

Research Tracker - background note

Why bootstrapping exists: noise growth and the refresh problem

Every fully homomorphic encryption (FHE) ciphertext carries a small additive noise (also called error) term that hides the message. Each homomorphic operation enlarges this noise: additions grow it modestly, multiplications and key-switching grow it more aggressively. Once the noise magnitude approaches the gap that separates encrypted message slots, decryption returns garbage. A leveled scheme can only evaluate a circuit of bounded multiplicative depth before this happens.

Bootstrapping is the operation that resets the noise to a fixed low level, regardless of how much noise was present beforehand, so that an unbounded-depth circuit can be evaluated. Conceptually, bootstrapping homomorphically evaluates the scheme's own decryption function on an encryption of the secret key, producing a fresh ciphertext that encrypts the same plaintext but with low, parameter-determined noise. Gentry's insight was that if the scheme can evaluate its own decryption circuit homomorphically (it is then bootstrappable), it can refresh ciphertexts indefinitely.

In the TFHE family (Chillotti, Gama, Georgieva, Izabachene), bootstrapping is unusually cheap and, crucially, can compute an arbitrary function of the message at the same time as it refreshes noise. This combined operation is called programmable bootstrapping (PBS). The refresh role is the non-negotiable part: even when no function is needed, PBS is the only way to drive noise back down. We make this explicit throughout: the output noise of a correct PBS depends only on the bootstrapping parameters (the gadget base, decomposition length, and the bootstrapping-key noise), never on the input ciphertext's accumulated noise.

The torus T and its discretization

TFHE is defined over the real torus $\mathbb{T} = \mathbb{R}/\mathbb{Z}$, the set of reals modulo $1$, i.e. the half-open interval $[0,1)$ with wraparound addition. The torus is an abelian group under addition but it is not a ring: there is no well-defined product of two torus elements (what would $0.5 \cdot 0.5$ mean modulo $1$ in a consistent way?). However, $\mathbb{T}$ is a module over $\mathbb{Z}$: an integer times a torus element is well defined. This single algebraic fact dictates the entire structure of the scheme - plaintexts and ciphertext components live on the torus, while secret keys and the multiplicands in homomorphic products are integers.

In implementation the torus is discretized. A torus element is represented by a machine integer modulo $q = 2^{32}$ or $q = 2^{64}$, where the integer $x$ stands for the torus value $x / q$. Thus the discretized torus is $\mathbb{Z}_q$, and a real torus value $t \in [0,1)$ maps to $\lfloor q\, t \rceil \bmod q$. We write the message space as the top bits of this representation: a plaintext $m$ in a small space $\{0, \dots, p-1\}$ is encoded as $\frac{m}{p}$ on the torus, leaving the low-order bits for noise.

The polynomial variants work over the ring

$$ \mathcal{R} = \mathbb{Z}[X]/(X^N + 1), \qquad \mathcal{R}_q = \mathbb{Z}_q[X]/(X^N + 1), $$

where $N$ is a power of two. The torus polynomials are $\mathbb{T}_N[X] = \mathbb{T}[X]/(X^N+1)$, i.e. degree-$negacyclic rotation of the coefficient vector, and rotating past index $N$ negates the wrapped coefficients (see Equation 5 below).

TLWE: the base ciphertext

A TLWE (Torus Learning With Errors) ciphertext encrypts a single torus message $\mu \in \mathbb{T}$ under a binary secret key $\mathbf{s} = (s_1, \dots, s_n) \in \{0,1\}^n$. It is a vector

$$ c = (a_1, \dots, a_n, b) \in \mathbb{T}^{n+1}, \qquad b = \sum_{i=1}^{n} a_i\, s_i + \mu + e, $$

where the mask coefficients $a_i$ are drawn uniformly from $\mathbb{T}$ and $e$ is a small Gaussian noise with standard deviation $\sigma$. This is Equation 1, the defining encryption relation. The hardness of recovering $\mu$ or $\mathbf{s}$ from $c$ is the (decisional) LWE assumption over the torus.

The phase of the ciphertext is the linear function

$$ \varphi_{\mathbf{s}}(c) = b - \sum_{i=1}^{n} a_i\, s_i = \mu + e, $$

which is Equation 2. Decryption computes the phase and then rounds it to the nearest valid encoded message: if messages occupy the points $\{k/p\}$, decryption returns the multiple of $1/p$ closest to $\varphi_{\mathbf{s}}(c)$. Correctness requires $\lVert e \rVert < \frac{1}{2p}$ so the noise cannot push the phase across the rounding boundary. Homomorphic addition is just coordinate-wise addition of two ciphertexts, which adds both messages and both noises; multiplication by a known integer $w$ scales the ciphertext (and the noise) by $w$.

TGLWE: packing many messages in a polynomial

TGLWE (the generalized, ring version) encrypts a polynomial message $\mu(X) \in \mathbb{T}_N[X]$ under a key that is a tuple of $k$ ring elements $\mathbf{S} = (S_1, \dots, S_k) \in \mathcal{R}^k$ with small (binary or ternary) coefficients. The ciphertext is

$$ C = (A_1, \dots, A_k, B) \in \mathbb{T}_N[X]^{k+1}, \qquad B = \sum_{j=1}^{k} A_j\, S_j + \mu + E, $$

with each $A_j$ uniform over $\mathbb{T}_N[X]$ and $E$ a small noise polynomial. This is Equation 3. TLWE is the special case $N = 1$ (and then $k = n$). The common bootstrapping parameterization uses $k = 1$ with a large $N$ (e.g. $N = 1024$ or $2048$), so a single ring element $S = S_1$ acts as the key and one TGLWE ciphertext packs $N$ torus coefficients. The phase is defined analogously, $\Phi_{\mathbf{S}}(C) = B - \sum_j A_j S_j = \mu + E$.

Ring multiplication of two degree-$N$ polynomials in $\mathcal{R}_q$ is the dominant cost, and it is performed with the Number-Theoretic Transform (NTT) - the modular analogue of the FFT - so that a product costs $O(N \log N)$ instead of $O(N^2)$. The iterative NTT with butterfly operations is given in the Algorithms appendix.

TGGSW and gadget decomposition

TGGSW (the ring GSW ciphertext, after Gentry-Sahai-Waters) is the object that lets us multiply an encrypted polynomial by a small cleartext-controlled value while keeping noise growth additive rather than multiplicative. A TGGSW ciphertext encrypting a small message $m \in \mathcal{R}$ (often a single bit) is a vertical stack of TGLWE ciphertexts. The construction relies on a gadget.

Fix a decomposition base $B = 2^{\beta}$ and a level count $\ell$. The gadget vector is

$$ \mathbf{g} = \left( \tfrac{q}{B},\ \tfrac{q}{B^2},\ \dots,\ \tfrac{q}{B^{\ell}} \right), $$

Equation 4, and the gadget matrix $G = I_{k+1} \otimes \mathbf{g}^{\top}$ tiles $\mathbf{g}$ across the $k+1$ ciphertext components. A TGGSW ciphertext of $m$ is then $C = Z + m\,G$, where $Z$ is a vertical stack of $(k+1)\ell$ fresh TGLWE encryptions of zero.

The companion operation is gadget (radix) decomposition: given a torus polynomial $a$, it returns small integer-coefficient polynomials $(u_1, \dots, u_{\ell})$ such that $a \approx \sum_{i=1}^{\ell} u_i \cdot \frac{q}{B^i}$ with each $\lVert u_i \rVert_{\infty} \le B/2$. Equivalently, $\langle \mathrm{Decomp}(a), \mathbf{g}\rangle \approx a$ up to a rounding error bounded by $\frac{q}{2 B^{\ell}}$. The decomposition truncates to the top $\ell$ base-$B$ digits, deliberately discarding the low-order noise bits - this is precisely why the external product's output noise does not inherit the input's accumulated noise. The pseudocode is in the appendix.

The external product and the CMux gate

The external product $\boxdot$ multiplies a TGGSW ciphertext (encrypting a small $m$) with a TGLWE ciphertext (encrypting $\mu$) to produce a TGLWE ciphertext of $m \cdot \mu$:

$$ \mathrm{TGGSW}(m) \;\boxdot\; \mathrm{TGLWE}(\mu) \;=\; \mathrm{Decomp}\big(\mathrm{TGLWE}(\mu)\big) \cdot \mathrm{TGGSW}(m) \;=\; \mathrm{TGLWE}(m \cdot \mu). $$

This is Equation 6. The operation decomposes each component of the TGLWE input into its $\ell$ small digit-polynomials, then takes the matrix-vector product of that small-coefficient row against the $(k+1)\ell$ rows of the TGGSW matrix. Because the multiplicands on the TGGSW side are scaled by the gadget while the TGLWE side contributes only small decomposition coefficients, the resulting noise is additive: roughly the TGGSW noise plus the decomposition rounding error, independent of the magnitude of the input message's noise. That asymmetry is the engine of bootstrapping.

The CMux (controlled multiplexer) gate is the external product specialized to a selector bit. Given a TGGSW encryption of a control bit $b \in \{0,1\}$ and two TGLWE ciphertexts $C_0, C_1$, it homomorphically selects:

$$ \mathrm{CMux}(\mathrm{TGGSW}(b), C_0, C_1) \;=\; C_0 + \mathrm{TGGSW}(b) \boxdot (C_1 - C_0) \;=\; \begin{cases} C_0 & b = 0 \\ C_1 & b = 1 \end{cases}. $$

This is Equation 7. The encrypted bit $b$ chooses between $C_0$ and $C_1$ without revealing which. A chain of CMux gates is exactly what implements blind rotation.

Blind rotation: turning the phase into a rotation

Blind rotation is the heart of PBS. The goal: take a TLWE ciphertext whose phase is $\tilde{\mu} = b - \sum_i a_i s_i$, and rotate a known test polynomial (the lookup table) by an amount equal to that phase - without decrypting. The output is a TGLWE ciphertext whose constant term holds the looked-up value.

First the torus phase is rescaled to an integer modulo $2N$. Each $a_i$ and $b$ is rounded to $\tilde{a}_i = \lfloor 2N a_i \rceil$ and $\tilde{b} = \lfloor 2N b \rceil$ in $\mathbb{Z}_{2N}$. Multiplying the test polynomial by $X^{-\tilde{\mu}}$ rotates it by $\tilde{\mu}$ positions. The negacyclic property,

$$ X^{N} \equiv -1 \pmod{X^N + 1} \quad\Longrightarrow\quad X^{i + N} \equiv -X^{i}, $$

which is Equation 5, means the rotation index lives modulo $2N$: a full $2N$ rotation returns to the start, and a rotation by $N$ flips the sign of every coefficient. The needed exponent is $-\tilde{\mu} = -\big(\tilde{b} - \sum_i \tilde{a}_i s_i\big)$, so the rotation amount is

$$ X^{-\tilde{\mu}} \;=\; X^{-\tilde{b}} \cdot \prod_{i=1}^{n} X^{\tilde{a}_i s_i}. $$

This is Equation 8, and it is the key decomposition: we know $\tilde{b}$ and all $\tilde{a}_i$ in the clear, but $\mathbf{s}$ is secret. Each factor $X^{\tilde{a}_i s_i}$ equals $X^{\tilde{a}_i}$ when $s_i = 1$ and $1$ when $s_i = 0$. That is a per-bit choice between two polynomials, i.e. a CMux! The bootstrapping key $\mathrm{BSK}$ is a list of $n$ TGGSW encryptions of the secret bits $s_1, \dots, s_n$. Blind rotation starts from the accumulator $\mathrm{ACC} = X^{-\tilde{b}} \cdot v(X)$ (the rotated test polynomial, trivially encrypted) and applies $n$ CMux steps, the $i$-th choosing between $\mathrm{ACC}$ and $X^{\tilde{a}_i} \cdot \mathrm{ACC}$ under the encrypted bit $s_i$. After all $n$ steps the accumulator encrypts $X^{-\tilde{\mu}} v(X)$. Because each CMux contributes only additive noise (Equation 6), the total noise after $n$ steps is small and bounded by the parameters - this is where the refresh happens.

The lookup table evaluated for free

The test polynomial $v(X) = \sum_{j=0}^{N-1} v_j X^j$ is built so that its coefficients spell out the desired function $f$ evaluated across the message space. After blind rotation, the accumulator is $X^{-\tilde{\mu}} v(X)$, and its constant coefficient is $v_{\tilde{\mu} \bmod 2N}$ (with a sign from negacyclic wraparound). By choosing $v_j$ so that the block of indices mapping to message $m$ all carry the value $\frac{f(m)}{p}$, the constant term of the rotated polynomial is exactly $f(\tilde{\mu})$, an encoding of $f$ applied to the original message.

This is why PBS is programmable: the function $f$ is baked into the cleartext test polynomial, so evaluating any function $f$ over the message space costs nothing beyond the bootstrap that we had to do anyway for noise refresh. The identity function refreshes noise with no change to the message; a sign function, ReLU-like step, or arbitrary small lookup table is obtained simply by re-filling $v$.

Two structural constraints follow from the negacyclic ring. First, the message must fit in $\lfloor 2N \rfloor$ rotation slots, so the precision is tied to $N$. Second, the sign flip at $X^N \equiv -1$ means the table is effectively anti-periodic with period $N$: $v_{j+N} = -v_j$ is forced. Functions are therefore evaluated on the cleartext under the assumption that the encoded message occupies the upper half of the rotation range (the so-called negacyclic / padding-bit convention), with one bit of padding (the message kept in $[0, N)$) guaranteeing a well-defined, sign-consistent lookup. The whole table evaluation is data-independent rotation of a fixed polynomial, hence "for free."

Sample extraction: from TGLWE back to TLWE

Blind rotation leaves the answer in the constant coefficient of a TGLWE ciphertext (a polynomial encryption), but the rest of the pipeline (and the next gate) wants a plain TLWE ciphertext encrypting that single coefficient. Sample extraction is the (noise-free, key-free) rewriting that pulls coefficient $h$ out of a TGLWE ciphertext into a TLWE ciphertext under the coefficient-vector key.

Write the TGLWE ciphertext (take $k=1$ for clarity) as $C = (A, B)$ with $A(X) = \sum_j a_j X^j$ and $B(X) = \sum_j b_j X^j$, under key $S(X) = \sum_j s_j X^j$. The phase polynomial is $\Phi(C) = B - A\,S$, and its coefficient $h$ equals the looked-up message plus noise. Because the product $A \cdot S$ in $\mathcal{R}_q$ is negacyclic, the coefficient $h$ of $A S$ is a fixed signed permutation of the $a_j$:

$$ (A S)_h \;=\; \sum_{j=0}^{h} a_{j}\, s_{h-j} \;-\; \sum_{j=h+1}^{N-1} a_{j}\, s_{N + h - j}. $$

This is Equation 9. To extract coefficient $h$ we therefore build a TLWE ciphertext $(a'_0, \dots, a'_{N-1}, b')$ whose body is $b' = b_h$ and whose mask is the signed, reversed arrangement of the $a_j$ so that $b' - \sum_i a'_i s_i = h\text{-th coefficient of } \Phi(C)$. For $h = 0$ the rule is simply

$$ a'_0 = a_0, \quad a'_i = -a_{N-i}\ (1 \le i \le N-1), \quad b' = b_0. $$

This is Equation 10. No homomorphic operation, no key, and no extra noise are involved - it is a pure relabeling of existing ciphertext coefficients. The result is a TLWE ciphertext under the extracted key (the coefficient vector of $S$), encrypting $f(m)$ with the low, fresh noise produced by blind rotation. A final key-switch maps it back to the original TLWE key dimension to complete the PBS.

Putting it together: the PBS pipeline and its noise budget

A complete programmable bootstrap of an input TLWE ciphertext proceeds in three stages:

1. Modulus switch the input TLWE from $\mathbb{Z}_q$ down to $\mathbb{Z}_{2N}$, rounding each $a_i, b$ to $\tilde{a}_i, \tilde{b}$. This introduces a small rounding error but sets up the rotation indices.

2. Blind rotation (Equation 8) over the test polynomial $v$ using the bootstrapping key, an $n$-fold CMux chain. Output: a TGLWE ciphertext encrypting $X^{-\tilde{\mu}} v(X)$ with parameter-bounded noise.

3. Sample extraction (Equation 10) of the constant coefficient, yielding a TLWE ciphertext of $f(m)$ under the extracted key, followed by a key switch back to the original key.

The output noise variance is, to first order, $\sigma_{\mathrm{out}}^2 \approx n\,(k+1)\,\ell\, N \cdot \tfrac{B^2}{12}\, \sigma_{\mathrm{BSK}}^2 \,+\, \text{(decomposition)} \,+\, \text{(mod-switch)} \,+\, \text{(key-switch)}$, a quantity built entirely from the bootstrapping parameters $(B, \ell, N, n, \sigma_{\mathrm{BSK}})$. It contains no term in the input ciphertext's noise. That is the formal statement of "noise refresh": however dirty the input, the output is clean to a fixed level, which is exactly what makes the scheme fully (rather than merely leveled) homomorphic, while the choice of $v$ lets us fold an arbitrary lookup table into the same pass.

Algorithms

Complete at-a-glance pseudocode for every algorithm referenced above.

Algorithm A - Gadget (radix) decomposition.

Input:  torus element a in Z_q, base B = 2^beta, levels ell
Output: small integers (u_1, ..., u_ell), each |u_i| <= B/2
 1: a_hat <- round(a / (q / B^ell)) * (q / B^ell)   # keep top ell digits
 2: carry <- 0
 3: for i = ell down to 1 do
 4:     digit <- (a_hat / (q / B^i)) mod B           # base-B digit
 5:     digit <- digit + carry
 6:     if digit >= B/2 then                          # balanced representation
 7:         digit <- digit - B
 8:         carry <- 1
 9:     else
10:         carry <- 0
11:     u_i <- digit
12: return (u_1, ..., u_ell)

Algorithm B - Iterative NTT (Cooley-Tukey, butterfly form). Used for ring products in $\mathcal{R}_q$ at cost $O(N \log N)$.

Input:  coefficient vector a[0..N-1], N = 2^L, primitive root w mod q
Output: a holding the NTT of the input (in-place)
 1: bit-reverse-permute(a)                       # reorder indices
 2: for s = 1 to L do
 3:     m <- 2^s                                  # current block size
 4:     w_m <- w^((q-1)/m) mod q                  # m-th root of unity
 5:     for k = 0 to N-1 step m do
 6:         omega <- 1
 7:         for j = 0 to m/2 - 1 do
 8:             t <- (omega * a[k + j + m/2]) mod q   # butterfly: twiddle
 9:             u <- a[k + j]
10:             a[k + j]         <- (u + t) mod q       # butterfly: add
11:             a[k + j + m/2]   <- (u - t) mod q       # butterfly: sub
12:             omega <- (omega * w_m) mod q
13: return a
# inverse NTT: same structure with w^{-1}, then multiply by N^{-1} mod q

Algorithm C - External product (TGGSW box-dot TGLWE).

Input:  G = TGGSW(m)  (a ((k+1)*ell) x (k+1) matrix of torus polys)
        C = TGLWE(mu) = (A_1,...,A_k, B)
        base B, levels ell
Output: TGLWE(m * mu)
 1: for j = 1 to k+1 do                          # decompose each component
 2:     (u_{j,1}, ..., u_{j,ell}) <- Decomp(C_j)  # Algorithm A, per coefficient
 3: U <- concat over j of (u_{j,1},...,u_{j,ell}) # row of (k+1)*ell small polys
 4: D <- U * G                                    # matrix-vector product in R_q
 5: return D                                       # = TGLWE(m * mu)

Algorithm D - CMux gate.

Input:  TGGSW(b) with b in {0,1}; TGLWE ciphertexts C0, C1
Output: C0 if b=0, C1 if b=1
 1: D <- C1 - C0
 2: P <- ExternalProduct(TGGSW(b), D)   # Algorithm C
 3: return C0 + P

Algorithm E - Blind rotation.

Input:  TLWE ct (a_1..a_n, b) already mod-switched to Z_{2N}
        test polynomial v(X) in T_N[X]
        bootstrapping key BSK = (TGGSW(s_1), ..., TGGSW(s_n))
Output: TGLWE encrypting X^{-mu_tilde} * v(X)
 1: ACC <- trivial TGLWE of ( X^{-b} * v(X) )    # b = b_tilde
 2: for i = 1 to n do
 3:     rotated <- X^{a_i} * ACC                  # cleartext rotation, a_i known
 4:     ACC <- CMux(BSK_i, ACC, rotated)          # pick rotated iff s_i = 1
 5: return ACC

Algorithm F - Sample extraction (coefficient h).

Input:  TGLWE C = (A, B) with A=sum a_j X^j, B=sum b_j X^j; index h
Output: TLWE (a'_0,...,a'_{N-1}, b') under the extracted key
 1: b' <- b_h
 2: for i = 0 to h do
 3:     a'_i <- a_{h - i}
 4: for i = h+1 to N-1 do
 5:     a'_i <- -a_{N + h - i}                    # negacyclic sign flip
 6: return (a'_0, ..., a'_{N-1}, b')

Algorithm G - Full programmable bootstrap (PBS).

Input:  TLWE_in encrypting message m under key s (in Z_q)
        function f, encoded as test polynomial v_f
        bootstrapping key BSK, key-switching key KSK
Output: TLWE encrypting f(m) under key s, with fresh low noise
 1: ct' <- ModSwitch(TLWE_in, from q to 2N)       # round a_i, b
 2: ACC <- BlindRotate(ct', v_f, BSK)              # Algorithm E
 3: tlwe_ext <- SampleExtract(ACC, h = 0)          # Algorithm F
 4: TLWE_out <- KeySwitch(tlwe_ext, KSK)           # back to original key dim
 5: return TLWE_out

Algorithm H - Systolic dataflow for the CMux / external-product chain. A hardware view of blind rotation: the $n$ CMux stages stream through a 1-D array of processing elements (PEs), each PE owning one bootstrapping-key TGGSW and pipelining the decompose/multiply/accumulate of the external product.

Setup:  PE_1 ... PE_n in a line; PE_i holds BSK_i
        accumulator tile ACC streams left-to-right through the array
 1: load ACC into PE_1
 2: for each PE_i (i = 1..n), in pipeline order do
 3:     in parallel across coefficient lanes:
 4:         D_lane     <- (X^{a_i} * ACC_lane) - ACC_lane   # form C1 - C0
 5:         U_lane     <- Decomp(D_lane)                    # Algorithm A
 6:         partial    <- MAC(U_lane, BSK_i rows)           # systolic multiply-accumulate
 7:         ACC_lane   <- ACC_lane + partial                # CMux result
 8:     forward ACC to PE_{i+1}                             # one stage latency
 9: drain final ACC out of PE_n
# Throughput: one CMux per stage; n CMuxes fill the pipeline of depth n,
# so steady-state latency is dominated by the deepest external product.
