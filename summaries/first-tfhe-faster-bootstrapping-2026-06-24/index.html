Faster Fully Homomorphic Encryption: Bootstrapping in Less Than 0.1 Seconds

ASIACRYPT 2016 (IACR ePrint 2016/870) · full view ↗

Motivation: improving on FHEW

Fully homomorphic encryption (FHE) lets one compute on encrypted data. Since Gentry's 2009 breakthrough, the cost of bootstrapping (homomorphically evaluating the decryption function to reset accumulated noise) has been the practical bottleneck. The state of the art at the time was the FHEW scheme of Ducas and Micciancio, which bootstrapped a single NAND gate in about $0.69$ seconds with a roughly $1$ GB bootstrapping key.

This paper - the first TFHE paper, by Chillotti, Gama, Georgieva and Izabachène - revisits GSW-based FHE and makes a key observation: the expensive internal product of GSW (a matrix-by-matrix product, GSW times GSW) can be replaced by a much cheaper external product between a GSW ciphertext and an LWE ciphertext (a matrix-by-vector product). Because a GSW sample is just a stack of LWE samples, the external product is to the internal product roughly what a matrix-vector product is to a matrix-matrix product, a saving of a factor $(k+1)\ell$.

Re-expressing the whole FHEW bootstrapping purely in terms of this external product yields the headline results:

  • Gate bootstrapping speeds up from under $1$ second to under $0.1$ seconds (measured $52$ ms), about a $12\times$ speedup.
  • The bootstrapping key shrinks from about $1$ GB to about $24$ MB (about $46\times$ smaller), at the same noise overhead.
  • Exact gadget decompositions are replaced with approximate ones, trading a little extra noise for speed and memory.
  • The external product also explains the asymmetry in GSW noise propagation and enables efficient leveled evaluation of deterministic automata (up to about $5000$ transitions/second, versus about $10$/second before), with noise overhead only linear in word length.

The paper also gives a self-contained practical security analysis that expresses the security parameter directly as a function of the secret entropy $n$ and the LWE error rate $\alpha$, with no modulus $q$ in the picture.

Working over the torus: TLWE, TRLWE and TGSW

Everything is defined over the real torus $\mathbb{T} = \mathbb{R}/\mathbb{Z}$ (reals mod $1$). $\mathbb{T}$ is not a ring, but it is a $\mathbb{Z}$-module, so the external product of an integer by a torus element is well defined. The base ring is $R = \mathbb{Z}[X]/(X^N + 1)$ and the message space is $\mathbb{T}_N[X] = \mathbb{T}[X]/(X^N+1) = R[X]/(X^N+1) \bmod 1$, which is an $R$-module. This unifies LWE and Ring-LWE into a single family.

TLWE samples. Fix an integer $k \ge 1$, $N$ a power of $2$, and a noise parameter $\alpha$. A TLWE secret key is $s \in \mathbb{B}_N[X]^k$ ($k$ polynomials with binary coefficients, about $n = kN$ bits of entropy). A fresh TLWE sample of a message $\mu \in \mathbb{T}_N[X]$ is a pair $(a, b) \in \mathbb{T}_N[X]^k \times \mathbb{T}_N[X]$, where the mask $a$ is uniform and $b$ is Gaussian (standard deviation $\alpha$) around $\mu + s \cdot a$. Choosing $N$ large with $k = 1$ gives Ring-LWE; $N = 1$ with $k$ large gives plain LWE; intermediate choices interpolate between ideal-lattice and general-lattice security. The phase of a sample is defined in Equation 1:

$$\varphi_s(a,b) = b - s \cdot a = b - \sum_{i=1}^{k} s_i \cdot a_i.$$

The phase is linear and $(kN+1)$-Lipschitz, so samples that are close on the torus behave identically. The message $\mathrm{msg}(c)$ is the expectation $\mathbb{E}[\varphi_s(c)]$ of the phase, and the error is $\mathrm{Err}(c) = \varphi_s(c) - \mathrm{msg}(c)$. When the noise amplitude stays below $1/4$, the message function is exactly linear, which makes the noise analysis clean.

TRLWE is simply the ring instantiation of TLWE (the paper writes everything as TLWE over $\mathbb{T}_N[X]$ and recovers Ring-LWE as the $N$ large, $k = 1$ case).

TGSW samples. TGSW is the scale-invariant, torus generalization of GSW. It depends on an integer decomposition depth $\ell$ and a fixed gadget matrix $h \in M_{d,k+1}(\mathbb{T}_N[X])$ with $d = (k+1)\ell$, whose nonzero entries form the super-increasing sequence $1/B_g, 1/B_g^2, \dots, 1/B_g^\ell$. A fresh TGSW sample of an integer-polynomial message $\mu \in R/h^{\perp}$ is given in Equation 2:

$$C = Z + \mu \cdot h \in M_{d,k+1}(\mathbb{T}_N[X]),$$

where every row of $Z$ is a homogeneous (message-$0$) TLWE sample of noise $\alpha$. Equivalently, a TGSW sample is a stack of $(k+1)\ell$ TLWE samples, so all linear-combination, phase, message and error notions extend row-wise.

Approximate gadget decomposition

The external product needs to decompose a TLWE sample onto the gadget $h$. The novelty is that this decomposition is approximate: $\mathrm{Dec}_{h,\beta,\epsilon}(v)$ returns a small integer-polynomial vector $u$ with quality $\lVert u \rVert_\infty \le \beta$ and precision $\lVert u \cdot h - v \rVert_\infty \le \epsilon$, where the approximation error is zero-mean for uniform $v$ so the message is preserved. Replacing exact decomposition with this approximate one is what cuts both time and memory, for a controlled noise increase. With base $B_g$ and depth $\ell$, Algorithm 1 achieves $\beta = B_g/2$ and $\epsilon = 1/(2 B_g^\ell)$:

Algorithm 1  Gadget decomposition of a TLWE sample
Input:  A TLWE sample (a, b) = (a_1, ..., a_k, b = a_{k+1}) in T_N[X]^k x T_N[X]
Output: A combination [e_{1,1}, ..., e_{k+1,l}] in R^{(k+1)l}
 1: For each a_i take the unique representative sum_{j=0..N-1} a_{i,j} X^j, a_{i,j} in T,
    and set abar_{i,j} := closest multiple of 1/B_g^l to a_{i,j}
 2: Decompose each abar_{i,j} uniquely as sum_{p=1..l} abar_{i,j,p} (1/B_g^p),
    where each abar_{i,j,p} in [ -B_g/2 , B_g/2 [
 3: for i = 1 to k + 1 do
 4:    for p = 1 to l do
 5:       e_{i,p} := sum_{j=0..N-1} abar_{i,j,p} X^j   in R
 6: return  (e_{i,p})_{i,p}

The output vector $u = (e_{i,p})$ satisfies $u \cdot h \approx (a_1, \dots, a_k, b)$ to within $\epsilon$ per coefficient, by construction of the super-increasing gadget.

The external product (the core idea)

The external product maps a TGSW ciphertext (encoding an integer polynomial) and a TLWE ciphertext (encoding a torus polynomial) to a TLWE ciphertext encoding the product of the two messages. It is the central operation of the whole paper:

External product  []  :  TGSW x TLWE  -->  TLWE
   (A, b)  |-->  A [] b  :=  Dec_{h,beta,epsilon}(b) . A

Only the single TLWE vector $b$ is decomposed (not a whole matrix), so the cost is one decomposition plus one vector-by-matrix product, versus the $(k+1)\ell$ such operations of the GSW internal product. Writing $A$ for a valid TGSW sample of message $\mu_A$ and $b$ for a valid TLWE sample of message $\mu_b$, the worst-case noise bound (Theorem 3.14) is Equation 3:

$$\lVert \mathrm{Err}(A \boxdot b) \rVert_\infty \;\le\; (k+1)\,\ell\,N\,\beta\,\lVert \mathrm{Err}(A) \rVert_\infty \;+\; \lVert \mu_A \rVert_1\,(1 + kN)\,\epsilon \;+\; \lVert \mu_A \rVert_1\,\lVert \mathrm{Err}(b) \rVert_\infty.$$

The three terms of Equation 3 are, respectively, the gadget/key noise, the approximate-decomposition noise, and the input noise scaled by $\mu_A$. If $\lVert \mathrm{Err}(A \boxdot b) \rVert_\infty \le 1/4$, then $A \boxdot b$ is guaranteed valid and has message $\mu_A \cdot \mu_b$.

Sketch of proof: writing $A = Z_A + \mu_A \cdot h$ and $b = z_b + (0, \mu_b)$, and $u = \mathrm{Dec}(b)$, one gets $A \boxdot b = u \cdot Z_A + \mu_A \cdot \varepsilon_{\mathrm{dec}} + \mu_A \cdot z_b + (0, \mu_A \mu_b)$, where $\varepsilon_{\mathrm{dec}} = u \cdot h - b$. Taking expectations kills the three zero-mean terms, leaving message $\mu_A \mu_b$; bounding the three error terms gives the formula. The average-case version (Corollary 3.15), under the independence heuristic, replaces norms by variances, as in Equation 4:

$$\mathrm{Var}(\mathrm{Err}(A \boxdot b)) \;\le\; (k+1)\,\ell\,N\,\beta^2\,\mathrm{Var}(\mathrm{Err}(A)) \;+\; (1 + kN)\,\lVert \mu_A \rVert_2^2\,\epsilon^2 \;+\; \lVert \mu_A \rVert_2^2\,\mathrm{Var}(\mathrm{Err}(b)).$$

The classical GSW internal product is recovered as $(k+1)\ell$ independent external products, one per row of the right operand, which is exactly why it is a factor $(k+1)\ell$ slower:

Internal product  [X]  :  TGSW x TGSW  -->  TGSW
   A [X] B  =  [ A [] b_1 ; ... ; A [] b_{(k+1)l} ]
            =  [ Dec(b_1) . A ; ... ; Dec(b_{(k+1)l}) . A ]
   where b_i is the i-th row of B.
   Result is a TGSW sample of message mu_A . mu_B.

This decomposition also explains the asymmetry of GSW noise growth: in $A \boxdot b$ the noise of $A$ is amplified by the large factor $(k+1)\ell N \beta$, whereas the noise of $b$ is amplified only by $\lVert \mu_A \rVert_1$, which is tiny when $\mu_A$ is a single monomial.

Gate bootstrapping and blind rotation

To bootstrap a noisy LWE sample $(a, b) \in \mathbb{T}^{n+1}$ under key $s \in \mathbb{B}^n$, the idea (following FHEW) is to homomorphically evaluate the rounded phase. A fixed test polynomial $\mathrm{testv} \in \mathbb{T}_N[X]$ is the phase detector: its coefficients encode the value the bootstrap should output for each possible phase. $\mathrm{testv}$ is loaded into a (trivial, noiseless) TLWE accumulator, then the accumulator is blind-rotated by the hidden phase: each iteration multiplies the accumulator (via the external product) by a TGSW encryption of the monomial $X^{-s_i \bar{a}_i}$, computed on the fly from the bootstrapping key. After $n$ rotations the accumulator holds $X^{\bar\varphi} \cdot \mathrm{testv}$; extracting its constant term yields an LWE encryption of the bootstrapped bit, which is then key-switched back to the original key.

The bootstrapping key $\mathrm{BK}_{s \to s'', \alpha}$ consists of $n$ TGSW samples, where $\mathrm{BK}_i$ is a TGSW encryption of the secret bit $s_i$ under the TRLWE key $s''$.

Algorithm 3  Bootstrapping procedure
Input:  An LWE sample (a, b) in LWE_{s,eta}(mu), a bootstrapping key BK_{s -> s'', alpha},
        a keyswitch key KS_{s' -> s, gamma} where s' = KeyExtract(s''),
        two fixed messages mu_0, mu_1 in T
Output: An LWE sample LWE_s( mu_0 if phi_s(a,b) in ]-1/4, 1/4[ ; mu_1 otherwise )
 1: mubar  := (mu_1 + mu_0)/2  ;  mubar' := mu_0 - mubar
 2: bbar   := round(2N b)  ;  abar_i := round(2N a_i)  for each i in [1, n]
 3: testv  := (1 + X + ... + X^{N-1}) * X^{-N/2} . mubar'   in T_N[X]
 4: ACC    <- ( X^{bbar} . (0, testv) )   in T_N[X]^{k+1}
 5: for i = 1 to n do
 6:    ACC <- ( h + (X^{-abar_i} - 1) . BK_i ) [] ACC
 7: u := (0, mubar) + SampleExtract(ACC)
 8: return  KeySwitch_{KS}(u)

The blind-rotation step is line 6. There, $h + (X^{-\bar{a}_i} - 1) \cdot \mathrm{BK}_i$ is a TGSW sample of message $X^{-\bar{a}_i s_i}$ (check by substituting $s_i = 0$, giving $h$, which is the gadget encoding of $1 = X^0$; and $s_i = 1$, giving $X^{-\bar{a}_i}$), with noise at most $2 \lVert \mathrm{Err}(\mathrm{BK}_i) \rVert_\infty$. Multiplying the accumulator by it (one external product per loop iteration) rotates the encoded test vector by the hidden term $X^{-\bar{a}_i s_i}$, so after the loop the accumulator encodes $X^{\bar{b} - \sum_i \bar{a}_i s_i} \cdot \mathrm{testv} = X^{\bar\varphi} \cdot \mathrm{testv}$.

Three optimizations over FHEW are folded into this single algorithm. (1) The rescaling to mod $2N$ is explicit on the torus (line 2); for parameters where all coefficients are multiples of $1/(2N)$ the rounding error $\delta$ is zero. (2) The test vector is baked into the accumulator from the start while it is still noiseless, removing a factor $\sqrt{N}$ from the final noise (FHEW added it at the end). (3) Each $X^{-\bar{a}_i s_i}$ TGSW is built dynamically as the tiny combination $h + (X^{-\bar{a}_i} - 1) \cdot \mathrm{BK}_i$, so there is no need to decompose $\bar{a}_i$ in a base $B_r$ and precompute every possibility in the key: this is the source of the about $46\times$ key-size reduction and also removes the second per-iteration accumulator operation, giving an extra $2\times$ speedup.

The bootstrapping theorem (Theorem 4.6) gives the worst-case output noise as Equation 5:

$$\lVert \mathrm{Err}(v) \rVert_\infty \;\le\; 2 n (k+1) \ell\, \beta N \alpha \;+\; kN t \gamma \;+\; n (1 + kN)\,\epsilon \;+\; kN\,2^{-(t+1)}.$$

The terms of Equation 5 are, in order: bootstrapping-key noise, key-switch decomposition, approximate gadget decomposition, and key-switch rounding. The average case (Corollary 4.7), with $\vartheta_{\mathrm{BK}} = \tfrac{2}{\pi}\alpha^2$ and $V_{\mathrm{KS}} = \tfrac{2}{\pi}\gamma^2$, is given by Equation 6:

$$\mathrm{Var}(\mathrm{Err}(v)) \;\le\; 2 N n (k+1) \ell\, \beta^2 \vartheta_{\mathrm{BK}} \;+\; kN t\, V_{\mathrm{KS}} \;+\; n (1 + kN)\,\epsilon^2 \;+\; kN\,2^{-2(t+1)}.$$

Sample extraction and key switching

Two helper operations close the bootstrapping loop.

Sample extraction turns a TRLWE (TLWE over $\mathbb{T}_N[X]$) sample into a plain LWE sample encrypting the constant term. $\mathrm{KeyExtract}(s'')$ flattens the key polynomials into the integer vector $s' = (\mathrm{coefs}(s''_1(X)), \dots, \mathrm{coefs}(s''_k(X))) \in \mathbb{Z}^{kN}$, and $\mathrm{SampleExtract}(a'', b'')$ outputs $(a', b')$ with $a' = (\mathrm{coefs}(a''_1(1/X)), \dots, \mathrm{coefs}(a''_k(1/X)))$ and $b'$ equal to the constant term of $b''$. It adds no noise: $\lVert \mathrm{Err}(a',b') \rVert_\infty \le \lVert \mathrm{Err}(a'',b'') \rVert_\infty$.

Key switching brings the extracted LWE sample (under the large key $s' \in \mathbb{B}^{kN}$) back to the original small key $s \in \mathbb{B}^n$, without blowing up the noise, again tolerating approximation. The key-switching key $\mathrm{KS}_{s' \to s, \gamma, t}$ is, for each $i \in [1,n']$ and $j \in [1,t]$, a fresh LWE sample $\mathrm{KS}_{i,j}$ encrypting $s'_i \cdot 2^{-j}$ under $s$.

Algorithm 2  KeySwitch procedure
Input:  An LWE sample (a' = (a'_1, ..., a'_{n'}), b') in LWE_{s'}(mu),
        a keyswitching key KS_{s' -> s}, where s' in {0,1}^{n'}, s in {0,1}^n,
        a precision parameter t in N
Output: An LWE sample LWE_s(mu)
 1: abar'_i := closest multiple of 1/2^t to a'_i, so |abar'_i - a'_i| < 2^{-(t+1)}
 2: Binary-decompose each abar'_i = sum_{j=1..t} a'_{i,j} . 2^{-j}, with a'_{i,j} in {0,1}
 3: return  (a, b) := (0, b') - sum_{i=1..n'} sum_{j=1..t} a'_{i,j} . KS_{i,j}

The resulting noise (Lemma 4.3) is $\lVert \mathrm{Err}(a,b) \rVert_\infty \le \eta' + n' t \gamma + n'\,2^{-(t+1)}$, where $\eta'$ is the input noise; the average-case variance version is $\mathrm{Var}(\mathrm{Err}(a,b)) \le \eta' + n' t \gamma^2 + n'\,2^{-2(t+1)}$.

Homomorphic gates and leveled automata

With message space $\{0, 1/4\}$, every basic binary gate is one bootstrap of a cheap affine combination ($\mathrm{HomNOT}$ needs none):

HomNOT(c)        = (0, 1/4) - c                      (no bootstrapping)
HomAND(c1, c2)   = Bootstrap( (0, -1/8) + c1 + c2 )
HomNAND(c1, c2)  = Bootstrap( (0,  5/8) - c1 - c2 )
HomOR(c1, c2)    = Bootstrap( (0,  1/8) + c1 + c2 )
HomXOR(c1, c2)   = Bootstrap( 2 . (c1 - c2) )

For NAND, with inputs of error $\lVert \mathrm{Err} \rVert \le 1/16$, the intermediate $\bar{c} = (0, 5/8) - c_1 - c_2$ has message $5/8$ or $3/8$ when at least one input is $0$, and $1/8$ when both are $1$; bootstrapping with $(\mu_1, \mu_0) = (1/4, 0)$ then reads off the NAND.

For leveled (bootstrap-free) computation, the external product distinguishes two wire types: control wires carry a small integer polynomial and are TGSW samples; data wires carry a torus element and are TLWE samples. The key leveled gates are the controlled-AND $\mathrm{CAnd}(C, d) = C \boxdot d$ and the controlled multiplexer $\mathrm{CMux}(C, d_1, d_0) = C \boxdot (d_1 - d_0) + d_0$. The CMux noise grows additively (a max over the two branches plus a control-noise term, not a sum), thanks to the structure of the external product, as Equation 7 makes precise:

$$\lVert \mathrm{Err}(\mathrm{CMux}(C, d_1, d_0)) \rVert_\infty \;\le\; \max\!\big(\lVert \mathrm{Err}(d_0) \rVert_\infty,\, \lVert \mathrm{Err}(d_1) \rVert_\infty\big) + \eta(C), \quad \eta(C) = (k+1)\ell N \beta \lVert \mathrm{Err}(C) \rVert_\infty + (kN+1)\epsilon.$$

Because control inputs must be fresh TGSW ciphertexts (data wires cannot feed control inputs), the construction naturally forces the use of deterministic-automata / binary-decision-diagram circuits, which are exactly the noise-optimal circuits from prior work. This lets the scheme evaluate deterministic automata with noise overhead only linear in word length, at up to about $5000$ transitions/second, and compile constant-time (data-independent control flow) algorithms with logarithmic memory in a leveled manner.

Worked leveled example: integer multiplication automaton

The paper illustrates leveled evaluation with a constant-time program that computes the $p$-th bit of the product of two integers $a, b$, written as a deterministic automaton: each step reads a single input bit and updates an internal state $(x, y)$ stored in only $M = \log_2(4p)$ bits. The automaton states are $(j, (x, y))$ with $j \in [0, j_{\max}]$ the read count and $(x, y) \in \mathbb{B} \times [0, 2p[$; the initial state is $(0,0,0)$, $j_{\max} \le p^2$, and the final (accepting) states are those with $y$ odd. By Theorem 5.4 the output is computed with fewer than $\#Q \le 4 p^3$ CMux gates and $O(p)$ noise overhead (and the count drops by about $8\times$ after minimizing the automaton). For $p = 32$, one CMux gate takes about $0.0002$ s, so the whole program (about $16384$ CMux) evaluates homomorphically in about $3.2$ seconds. The point: mapping a problem to a low-memory program is generally easier than finding a low-multiplicative-depth circuit (as BGV or integer FHE would need), and compiling that program to a net-list of CMux gates is then straightforward.

Algorithm 4  Fixed-time algorithm computing the p-th bit of the product a*b
Input:  a and b as little-endian bit strings
Output: p-th bit of a*b
 1: Internal memory: x in {0,1}, y in [[0, 2p[[
 2: initialize x := 0, y := 0
 3: for k = 0 to p - 1 do
 4:    for i = 0 to k - 1 do
 5:       read a_i      ; x := a_i
 6:       read b_{k-i}   ; y := y + x * b_{k-i}
 7:    end for
 8:    read a_k          ; x := a_k
 9:    read b_0          ; y := floor( (y + x * b_0) / 2 )
10: end for
11: for i = 0 to p do
12:    read a_i          ; x := a_i
13:    read b_{p-i}      ; y := y + x * b_{p-i}
14: end for
15: accept iff y == 1 (mod 2)

Parameters, noise and timings

The implementation chooses $n = 500$, $N = 1024$, $k = 1$. Torus elements are rescaled by $2^{32}$ and stored as 32-bit integers (native mod-$2^{32}$ arithmetic); polynomials mod $X^N+1$ are kept either as coefficient lists or in a Lagrange half-complex (FFT) representation via FFTW, with $53$ bits of floating-point precision sufficient.

  • LWE samples: about $2$ KB. All LWE masks are clamped to multiples of $1/2048$, so the bootstrapping phase rescaling is exact ($\delta = 0$).
  • TLWE samples: about $8$ KB; TGSW samples: about $48$ KB.
  • Gadget: $\ell = 3$, $B_g = 1024$, hence $\beta = 512$ and $\epsilon = 2^{-31}$.
  • Bootstrapping key: $n$ TGSW samples, about $23.4$ MB (versus about $1$ GB in FHEW), with $\alpha = 9.0 \times 10^{-9}$ (a higher, i.e. more secure, noise than FHEW's about $3.25 \times 10^{-10}$, made affordable by the lower noise overhead; the TRLWE key is binary).
  • Key-switching key: about $29.2$ MB, with $\gamma = 3.05 \times 10^{-5}$ and $t = 15$ (decomposition precision $2^{-16}$). This key can be shrunk by a factor $n+1 = 501$ by deriving masks from a PRF seed; the bootstrapping key similarly by $k+1 = 2$.

Correctness: the final post-bootstrap error variance is $9.24 \times 10^{-6}$ (standard deviation $\sigma \approx 0.00961$, better than FHEW's $0.01076$); the output noise stays below $1/16$ with probability at least $1 - 2^{-33.56}$.

Operation counts and timing on a single-core i7-4930MX at $3.00$ GHz:

                       #Classical products    #FFT (Lagrange repr.)
External product               12                      8
Bootstrapping                6000                    4006
Bootstrapping [FHEW]       (72000)                  48000

Measured running time: $52$ ms per gate bootstrap using the Lagrange half-complex representation, consistent with the predicted $12\times$ speedup over FHEW. FFTs and complex multiplications dominate (over $90\%$ of the time); key switching is negligible.

Practical security analysis

The paper gives a concrete-security model that, unlike prior LWE estimates, has no modulus $q$ (TLWE works on the continuous torus). Since the phase is Lipschitz, a TLWE sample maps to its nearest bin-LWE / bin-RingLWE instance, reducible to standard LWE and then to worst-case BDD. For practical hardness, the analysis builds on Albrecht-Player-Scott, with the SIS-distinguisher (Lindner-Peikert model) as the typical best attack, and expresses the security parameter $\lambda$ directly as a function of $n$ and $\alpha$.

To distinguish $m$ homogeneous LWE samples from uniform, one seeks a short integer combination $v$ cancelling the masks, found via a short vector in an $(n+m)$-dimensional lattice; allowing approximate solutions makes even $m \lt n$ viable. After a scaling $f_q(B)$ by $q$ and lattice reduction of quality $\delta$, the distinguishing term $\sum_i v_i b_i$ is Gaussian and distinguishable once its variance exceeds the smoothing parameter $\eta_\epsilon(\mathbb{Z})^2$. Optimizing over $q$, $m$, $\tau = n/(m+n)$ and $\epsilon$ gives the system in Equation 8 (the security parameter), Equation 9 (the BKZ cost), Equation 10 (the achievable root-Hermite factor), and Equation 11 (the smoothing parameter and secret scaling):

$$\lambda(n, \alpha) = \log_2(t_{\mathrm{attack}}) = \min_{0 \lt \epsilon \lt 1} \log_2\!\Big( \tfrac{n}{\epsilon^2}\, t_{\mathrm{BKZ}}(n, \alpha, \epsilon) \Big).$$

$$\log_2(t_{\mathrm{BKZ}}) = \frac{0.009}{\log_2(\delta)^2} - 27.$$

$$\ln(\delta) = \max_{m \gt 1,\, q \gt 1} \frac{1}{2(m+n)} \left[ \ln\!\big(\eta_\epsilon(\mathbb{Z})^2\big) - \ln\!\left( \frac{\pi S^2}{2 q^2}\,\frac{n}{m+n} + q^{2n/m}\,\alpha^2\,\frac{m}{m+n} \right) \right].$$

$$\eta_\epsilon(\mathbb{Z}) \approx \sqrt{\tfrac{1}{\pi} \ln(1/\epsilon)}, \qquad S = \tfrac{\lVert s \rVert}{\sqrt{n}} \approx \tfrac{1}{\sqrt{2}},$$

with optimal $q_{\mathrm{best}} = \big(\tfrac{\pi S^2}{2 \alpha^2}\big)^{m/(2(m+n))}$ and $\tau_{\mathrm{best}} = \tfrac{l_1}{2(l_1 - l_2)}$, where $l_1 = \ln(\eta_\epsilon^2/\alpha^2)$ and $l_2 = \ln(2 \eta_\epsilon^2/(\pi S^2))$.

The model is plotted in Figure 1, which shows $\lambda$ as level curves over $(n, \log_2(1/\alpha))$ and locates the chosen parameter sets (Switch Key, Boot. Key, and FHEW's Boot. Key [11]). Evaluated on the chosen parameters, it predicts $194$ bits of security for the bootstrapping key ($n = 1024$, $\alpha = 9.0 \times 10^{-9}$) and $136$ bits for the key-switching key ($n = 500$, $\alpha = 2^{-15}$), so the key-switching key is the bottleneck. The same model re-scores FHEW's parameter set at $136$ bits (above the at least $100$ bits FHEW originally claimed), the gap coming from counting how many times the SIS-distinguisher must be rerun to gain non-negligible advantage. The main remaining drawback noted in the conclusion is the large ciphertext expansion factor (about $400000$) with limited batching.

This appendix collects every algorithm and helper operation referenced above as self-contained, at-a-glance listings.  5: for i = 1 to k + 1:
 6:    for p = 1 to l:
 7:       e_{i,p} := sum_{j=0..N-1} abar_{i,j,p} X^j   in R
 8: return (e_{i,p})_{i in [1,k+1], p in [1,l]}


Operation  ExternalProduct  []  :  TGSW x TLWE -> TLWE
Input:  TGSW sample A (message mu_A), TLWE sample b (message mu_b)
Output: TLWE sample of message mu_A * mu_b
 1: u := Dec_{h,beta,epsilon}(b)            // u in R^{(k+1)l}
 2: return u . A                            // vector-by-matrix product


Operation  InternalProduct  [X]  :  TGSW x TGSW -> TGSW
Input:  TGSW samples A (message mu_A), B (message mu_B); rows b_1..b_{(k+1)l} of B
Output: TGSW sample of message mu_A * mu_B
 1: for r = 1 to (k+1)l:
 2:    row_r := A [] b_r          // one external product per row
 3: return [ row_1 ; ... ; row_{(k+1)l} ]


Operation  SampleExtract / KeyExtract  :  TRLWE -> LWE (no noise added)
Input:  TRLWE sample (a'', b'') in T_N[X]^{k+1}, TRLWE key s'' in B_N[X]^k
Output: LWE sample (a', b') in T^{kN+1} of the constant term of msg(a'',b''),
        under extracted key s' = KeyExtract(s'')
 1: s' := ( coefs(s''_1(X)), ..., coefs(s''_k(X)) ) in Z^{kN}
 2: a' := ( coefs(a''_1(1/X)), ..., coefs(a''_k(1/X)) )   // reindex / reflect
 3: b' := constant coefficient of b''
 4: return (a', b')


Algorithm 2  KeySwitch
Input:  LWE sample (a' = (a'_1..a'_{n'}), b') in LWE_{s'}(mu),
        keyswitch key KS_{i,j} = LWE_s( s'_i * 2^{-j} )  for i in [1,n'], j in [1,t],
        precision t in N
Output: LWE sample LWE_s(mu)
 1: for i = 1 to n':
 2:    abar'_i := closest multiple of 1/2^t to a'_i      // |abar'_i - a'_i| < 2^{-(t+1)}
 3:    decompose abar'_i = sum_{j=1..t} a'_{i,j} 2^{-j} , a'_{i,j} in {0,1}
 4: return (a, b) := (0, b') - sum_{i=1..n'} sum_{j=1..t} a'_{i,j} . KS_{i,j}


Algorithm 3  Bootstrap (gate bootstrapping)
Input:  LWE sample (a, b) in LWE_{s,eta}(mu),
        bootstrapping key BK_i = TGSW_{s''}(s_i)  for i in [1,n],
        keyswitch key KS_{s' -> s} with s' = KeyExtract(s''),
        output messages mu_0, mu_1 in T
Output: LWE_s( mu_0 if phi_s(a,b) in ]-1/4,1/4[ else mu_1 )
 1: mubar  := (mu_1 + mu_0)/2 ;  mubar' := mu_0 - mubar
 2: bbar   := round(2N b) ;  abar_i := round(2N a_i)  for i in [1,n]
 3: testv  := (1 + X + ... + X^{N-1}) * X^{-N/2} . mubar'    in T_N[X]
 4: ACC    := X^{bbar} . (0, testv)         in T_N[X]^{k+1}
 5: for i = 1 to n:
 6:    ACC := ( h + (X^{-abar_i} - 1) . BK_i ) [] ACC     // blind rotation
 7: u := (0, mubar) + SampleExtract(ACC)
 8: return KeySwitch_{KS}(u)


Homomorphic gates (message space {0, 1/4}); each is one Bootstrap of an affine combination
  HomNOT(c)        = (0, 1/4) - c                          // no bootstrapping
  HomAND(c1, c2)   = Bootstrap( (0, -1/8) + c1 + c2 ; mu_0=0, mu_1=1/4 )
  HomNAND(c1, c2)  = Bootstrap( (0,  5/8) - c1 - c2 ; mu_0=0, mu_1=1/4 )
  HomOR(c1, c2)    = Bootstrap( (0,  1/8) + c1 + c2 ; mu_0=0, mu_1=1/4 )
  HomXOR(c1, c2)   = Bootstrap( 2 . (c1 - c2)       ; mu_0=0, mu_1=1/4 )


Leveled (bootstrap-free) gates: control wire = TGSW C, data wires = TLWE d
  CAnd(C, d)         = C [] d
  CMux(C, d1, d0)    = C [] (d1 - d0) + d0


Algorithm 4  Fixed-time p-th-bit-of-product automaton (compiled to a net of CMux)
Input:  a, b little-endian bit strings
Output: p-th bit of a*b
 1: memory x in {0,1}, y in [[0,2p[[ ;  x := 0 ; y := 0
 2: for k = 0 to p-1:
 3:    for i = 0 to k-1:
 4:       x := a_i
 5:       y := y + x * b_{k-i}
 6:    x := a_k
 7:    y := floor( (y + x * b_0) / 2 )
 8: for i = 0 to p:
 9:    x := a_i
10:    y := y + x * b_{p-i}
11: accept iff y == 1 (mod 2)
