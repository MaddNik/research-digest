Programmable Bootstrapping Enables Efficient Homomorphic Inference of Deep Neural Networks

PDF, IACR ePrint 2021/091 · full view

Overview and contribution

This paper, by Ilaria Chillotti, Marc Joye, and Pascal Paillier (Zama, France), reports first experiments with a new library implementing a variant of the TFHE fully homomorphic encryption (FHE) scheme, applied to the privacy-preserving inference of deep neural networks. The enabling technology is programmable bootstrapping (PBS), which homomorphically evaluates an arbitrary function of a ciphertext while at the same time resetting the noise to a controlled level. The authors claim this is the first demonstration that deep neural networks (tested up to 100 layers) are within reach of FHE.

The central departure from prior work is that the framework does not require re-training the model. Earlier FHE-for-NN efforts (CryptoNets and successors) replaced standard activations with FHE-friendly variants (for example the square function) or restricted signals to discretized values, then re-trained. Here the original trained model is used as is: no change to operations or topology. This matters because training is costly, and producing a new model may be impossible without access to the (possibly regulated) training dataset. Complex non-linear functions are evaluated through a combination of PBS and leveled operations, and because each PBS resets the noise, the process can be iterated arbitrarily, so network depth is no longer a fundamental limitation. The library is open source (Rust): the Concrete project at https://github.com/zama-ai/concrete.

Context on noise and bootstrapping. All known FHE instantiations produce noisy ciphertexts; homomorphic operations grow the noise, and once it crosses a threshold the ciphertext is no longer decryptable. A scheme supporting only a predetermined noise budget is leveled. Bootstrapping (Gentry, 2009) refreshes ciphertexts by homomorphically evaluating the decryption circuit, turning a leveled scheme into a fully homomorphic one. TFHE descends from the FHEW line (Ducas-Micciancio) that brought bootstrapping below one second. TFHE runs in two modes: a leveled mode (linear combinations plus a bounded number of external products, noise always grows, good for small-depth circuits) and a bootstrapped mode that finely controls noise and, crucially, is programmable. Large-depth circuits such as deep neural networks require the bootstrapped mode.

Preliminaries: torus, LWE/GLWE, and finite precision

The 'T' in TFHE is the real torus T = R/Z (reals modulo 1). (T, +) is an abelian group but not a ring: there is no internal product of torus elements, only an external product between integers and torus elements, so T has a Z-module structure. Polynomials over the torus use the M-th cyclotomic polynomial; for performance M is a power of 2, giving degree N = M/2 and Phi(X) = X^N + 1. This yields the module T_N[X] = T[X]/(X^N + 1) of polynomials mod X^N + 1 with coefficients in T, which can be added together and multiplied externally by polynomials of Z_N[X]. Two probability distributions are used: uniform (U) and normal/Gaussian (N(mu, sigma^2)); a real-valued normal induces a discretized normal over Z via rounding.

LWE and GLWE over the torus. The LWE assumption says r = sum_j s_j . a_j + e in T is indistinguishable from a random torus element even given (a_1, ..., a_n). This masks a message mu in T as a ciphertext c = (a_1, ..., a_n, r + mu) in T^(n+1), with secret key s in B^n (B = {0,1}) and noise e drawn from N(0, sigma^2). GLWE generalizes this to polynomial messages in T_N[X] with a key of k polynomials in B_N[X]^k. LWE ciphertexts are exactly the special case (k, N) = (n, 1) of GLWE, so the paper presents everything in the GLWE setting. Vectors are row vectors in bold; integers/torus elements in roman, polynomials in calligraphic letters.

Finite precision. In practice torus elements are represented with finite precision Omega bits (typically 32 or 64). This replaces T with the submodule (1/q)Z/Z where q = 2^Omega, and computations happen in the corresponding precision-limited polynomial module. Any precision-limited message can be written as an integer polynomial scaled by 1/q, so the working integer ring is Z_q[X]/(X^N + 1).

Encoding and decoding messages

Input messages can be in any format; the encoding/decoding step makes them compatible with the scheme. A plaintext is an integer polynomial mu entering encryption; it corresponds to a cleartext m in a finite message space M, via Encode (and inverse Decode), with the requirement Decode(Encode(m)) = m for all m.

Because the noise is added on the right (less significant bits), only the upper bits of each coefficient mu_i carry the cleartext. Some homomorphic operations also require a number of leading bits to be set to 0. Let phi (varpi) >= 0 be the number of these padding bits and omega >= 1 the number of bits actually used for the cleartext, with phi + omega <= Omega. Define the message modulus p = 2^(phi + omega); omega is the message bit-precision. Each coefficient has the form mu_i = 2^(Omega - (phi + omega)) . (nu_i mod 2^omega) for some nu_i. Figure 1 shows this bit layout: phi padding bits, then omega message bits (nu_i), then Omega - (phi + omega) lower bits where noise lives.

The function Upper recovers the message-bearing upper part. For x = x_H (q/p) plus-or-minus x_L in Z/qZ with 0 <= x_L <= q/(2p), Upper_{q,p}(x) = (q/p) . round(p . lift(x) / q) mod q, where lift maps Z/qZ to Z. It extends to polynomials coefficient-wise. For q = 2^Omega and p = 2^(phi + omega), if x = x_H 2^(Omega - (phi + omega)) plus-or-minus x_L with x_L <= 2^(Omega - (phi + omega) - 1), then Upper_{q,p}(x) = x_H 2^(Omega - (phi + omega)).

Discretized TFHE: encryption scheme

The scheme is presented in the general GLWE form; LWE is the case (k, N) = (n, 1).

KeyGen(1^lambda): Pick (k, N) with k >= 1 and N a power of 2, a normal error distribution chi = N(0, sigma^2), and sample the key uniformly as k polynomials in B_N[X]. The plaintext space is the (q/p)-scaled subring of Z_q[X]/(X^N + 1), with q = 2^Omega and message modulus p = 2^(phi + omega), Omega >= phi + omega. Public parameters are pp = {k, N, sigma, p, q}; the private key sk is the sampled key vector.

Encrypt_sk(mu): For plaintext mu, output C = (A_1, ..., A_k, B) where the noisy message is mu* = mu + E (mod (q, X^N + 1)), and B = sum_j key_j . A_j + mu* (mod (q, X^N + 1)), for a uniformly random polynomial vector (A_1, ..., A_k) and a discrete noise E = round(epsilon . q) with epsilon a polynomial whose coefficients are sampled from N(0, sigma^2).

Decrypt_sk(C): Given C = (A_1, ..., A_k, B), compute mu* = B - sum_j key_j . A_j (mod (q, X^N + 1)) and output Upper_{q,p}(mu*).

Correctness: If C = Encrypt(mu) then Decrypt(C) = mu provided the infinity-norm of the noise is below q/(2p) = 2^(Omega - (phi + omega) - 1); this holds even when some of the phi leading bits are non-zero. In approximate applications the decryption need only return mu approximately, relaxing the noise bound.

Leveled operations: addition, scalar multiplication, external product, CMux

Addition. GLWE ciphertexts are additively homomorphic: component-wise addition of two ciphertexts of mu_1 and mu_2 gives a ciphertext of mu_1 + mu_2, provided the summed noise E_3 = E_1 + E_2 stays small.

Scalar multiplication. For an integer K >= 0, K . C is an encryption of K . mu (K-fold addition); for K < 0, K . C = (-K) . (-C). More generally multiplying by a polynomial constant W in Z_N[X] gives an encryption of W . mu, provided W . E stays small.

External product (GGSW times GLWE). GLWE has no native internal multiplication, so the GSW matrix approach is used (called GGSW in general; GSW for (k,N)=(n,1)). With B = 2^beta and ell such that ell . beta <= Omega, define the gadget vector g = (2^(Omega - beta), 2^(Omega - 2 beta), ..., 2^(Omega - ell beta)). The GGSW encryption of a plaintext K in Z_N[X] is G = Z + K . G^transpose, where Z is a matrix whose (k+1) ell rows are each a fresh GLWE encryption of 0, and G^transpose = I_{k+1} tensor g^transpose is the gadget matrix. Any element d in Z/qZ is approximated by a signed-digit radix-B expansion of length ell, d approximately equal to sum_i d_i 2^(Omega - i beta) = g^{-1}(d) . g^transpose, with digits d_i in [-B/2, B/2) and approximation error bounded by q/(2 B^ell) = 2^(Omega - beta ell - 1). This extends to polynomials (g^{-1}) and to vectors of k+1 polynomials (G^{-1}), with the same error bound on the infinity norm. The external product of a GGSW ciphertext G_1 (of K_1) and a GLWE ciphertext C_2 (of mu_2) is C_3 = G_1 box-times C_2 := G^{-1}(C_2) . G_1. Algebra shows C_3 = (a GLWE encryption of 0) + K_1 . (G^{-1}(C_2) G^transpose) approximately equal to K_1 . C_2, i.e. a GLWE encryption of K_1 mu_2, provided the resulting noise (including the approximation error) stays small.

The CMux gate. Built on the external product, CMux is an encrypted selector controlled by an encrypted bit b. It takes two GLWE ciphertexts C_0, C_1 (of d_0, d_1) and a GGSW ciphertext G of a bit b, and returns a GLWE ciphertext of d_b:

C' <- CMux(G, C_0, C_1) := G ☐ (C_1 - C_0) + C_0 .

Here the box symbol denotes the external product. CMux is central to the performance of homomorphic computation and especially to the bootstrapping.

Programmable bootstrapping: blind rotation

Programmable bootstrapping extends bootstrapping so that it resets the noise to a fixed level while simultaneously evaluating a function on the input ciphertext. When that function is the identity, it reduces to a regular bootstrapping. The construction has three parts: blind rotation, sample extraction, and key switching.

Intuition. Take an LWE ciphertext c = (a_1, ..., a_n, b) in (Z/qZ)^(n+1) of mu, with b = sum_j s_j a_j + mu* and mu* = mu + e. Decryption (without rounding) computes mu* = b - sum_j s_j a_j and then mu = Upper_{q,p}(mu*). The bootstrapping trick is to place -mu* = -b + sum_j s_j a_j (mod q) in the exponent of X, giving the monomial X^(-mu*). There are q possible values of mu*. A test polynomial is built whose i-th coefficient encodes the noise-free value Upper_{q,p}(i mod q) corresponding to each possible mu*. Multiplying (rotating) the test polynomial by X^(-mu*) moves the desired value mu into the constant coefficient, where it can be extracted (Figure 2). The rotation is done homomorphically, hence 'blind rotation', and because X^(-mu*) . v is a polynomial, GLWE encryption is used.

Rescaling and drift. GLWE polynomials are mod X^N + 1, so X has multiplicative order 2N, whereas mu* is defined mod q. mu* must be rescaled mod 2N: instead of working mod q, one uses the approximation -mu*-tilde = -b-tilde + sum_j s_j a_j-tilde (mod 2N), where b-tilde = round(2N (b mod q) / q) and a_j-tilde = round(2N (a_j mod q) / q). This rescaling introduces a small extra error called drift, which depends on the input LWE size n and the GLWE ring size N. A smaller n or larger N reduces the drift. Because the test polynomial has only N coefficients, at most N values of mu*-tilde can be encoded; this is handled by forcing the most significant bit of mu*-tilde to 0 (parameter phi >= 1). The test polynomial is then v = v_0 + ... + v_{N-1} X^(N-1) with v_i = Upper_{q,p}((q / 2N) i mod q).

Implementation as a chain of CMux gates. The product X^(-mu*-tilde) . v under GLWE is computed by a sequence of CMux gates. An accumulator ACC is initialized to a GLWE encryption of X^(-b-tilde) . v, then for i = 1 to n it is multiplied by X^(s_i a_i-tilde), each multiplication done via CMux using the bootstrapping keys bsk[i] = GGSW_{key'}(s_i), which are GGSW encryptions of the secret key bits under a fresh key key'. The blind rotation algorithm (Appendix B.1) is:

Algorithm 1: Blind rotation.
ACC <- (0, ..., 0, X^(-b-tilde) . v)
for i = 1 to n do
    ACC <- CMux(bsk[i], ACC, X^(a_i-tilde) . ACC)
end for
return ACC

At the end of the loop, ACC contains a GLWE encryption of X^(-mu*-tilde) . v under key key'.

Sample extraction and key switching

Sample extraction (Appendix B.2). After blind rotation, ACC is a GLWE encryption of the polynomial u = X^(-mu*-tilde) . v whose constant coefficient is mu. Sample extraction pulls that constant coefficient out as an LWE ciphertext, simply by selecting and reordering coefficients of the GLWE ciphertext (no homomorphic arithmetic). Writing the extraction key key' = (key'_1, ..., key'_k) with each key'_j = s'_{j,0} + ... + s'_{j,N-1} X^(N-1), and parsing the GLWE ciphertext as (A'_1, ..., A'_k, B') with A'_j = a'_{j,0} + ... + a'_{j,N-1} X^(N-1) and B' = b'_0 + ... + b'_{N-1} X^(N-1), the extracted LWE ciphertext is:

c' := (a'_{1,0}, -a'_{1,N-1}, ..., -a'_{1,1}, ...,
       a'_{k,0}, -a'_{k,N-1}, ..., -a'_{k,1}, b'_0)
      in (Z/qZ)^(kN+1)

which is an LWE encryption of mu under the key s' = (s'_1, ..., s'_{kN}) in B^{kN}, where s'_{l + 1 + (j-1)N} := s'_{j,l} for 1 <= j <= k and 0 <= l <= N - 1.

Key switching (Appendix B.3). The blind-rotation output ciphertext c' is under key s' and has format (Z/qZ)^(kN+1), while the original input c was under key s with format (Z/qZ)^(n+1). Key switching converts c' back to the original key and format so the bootstrapping loop closes. It needs key-switching keys ksk[i,j] = LWE_s(s'_i . round(q / B_KS^j)) for all 1 <= i <= kN and 1 <= j <= ell_KS, with gadget-decomposition parameters B := B_KS and ell := ell_KS. On input c' = (a'_1, ..., a'_{kN}, b') under s', the output is:

c'' := (0, ..., 0, b') - sum_{i=1}^{kN} sum_{j=1}^{ell_KS} a'_{i,j} . ksk[i, j]
       in (Z/qZ)^(n+1)

where (a'_{i,1}, ..., a'_{i,ell_KS}) = g^{-1}(a'_i) is the gadget decomposition of a'_i. The result c'' is an LWE encryption of mu under the original key s, provided the resulting noise stays small.

Look-up table evaluation (the programmable part)

The same blind rotation that performs a bootstrapping can simultaneously evaluate an arbitrary function, by encoding that function as a look-up table inside the test polynomial. Suppose we want to evaluate over encrypted data a function f: D -> I, x -> y = f(x), given encoding functions Encode: D -> Z/qZ and Encode': I -> Z/qZ with matching decoders.

For a plain bootstrapping the test polynomial coefficients were v_i = Upper_{q,p}((q / 2N) i mod q), which transforms a ciphertext of mu into a less-noisy ciphertext of the same mu. To evaluate f instead, define the look-up table entries as pairs (i, T[i]) for 0 <= i <= N - 1 with:

T[i] = Encode' ○ f ○ Decode ○ Upper_{q,p}((q / 2N) i mod q)

(the circle denotes function composition). Then program the test polynomial as v = v_0 + ... + v_{N-1} X^(N-1) with v_i = T[i], leaving the rest of the blind-rotation / sample-extraction / key-switching process unchanged. Up to the drift, an input ciphertext of mu (encoding x in D) is transformed into a ciphertext encoding f(x), and because it is the output of a bootstrapping it has low noise. This whole procedure is the programmable bootstrapping.

Remark (negacyclic functions). The regular bootstrapping requires phi >= 1. This condition can be lifted in the programmable bootstrapping when the look-up table is negacyclic, i.e. T[i + N] is congruent to -T[i] (mod 2N). In that case 2N values (rather than N) can be programmed.

Application to neural networks: layers without PBS

Neural-network layers split into two classes over encrypted data: those evaluable with only leveled operations, and those whose non-linear or complex operations require one or more programmable bootstrappings. Layers of the first type may still trigger a bootstrap on intermediate values whenever noise exceeds a threshold. The techniques are generic and support all known layer types.

Dense / linear layer. Computes the dot product of inputs with a weight matrix, optionally adds a bias, then applies an activation component-wise (with no activation it is a 'linear' layer). Weights and bias are in the clear, so the layer (activation aside) is a series of constant-multiplications and additions, all leveled operations. The activation is handled by PBS (next section).

Convolution layer. Convolves the input with a clear-text filter (tensor of weights), optionally adds biases, optionally activates. Like the dense layer, the convolution itself (without activation) is constant-multiplications and additions.

Addition layer. Component-wise additions, i.e. leveled operations.

Flatten layer. Reshapes input to a lower-dimensional array (e.g. to feed a dense layer). Over encrypted data this just rearranges ciphertexts; no homomorphic operation is needed.

Global average pooling layer. Computes (sum of n components)/n. Homomorphically this reduces to computing the sum; the division by n is folded into the next programmable bootstrapping (e.g. in a following dense or convolution layer) by dividing those weights by n. So the only homomorphic cost is the ciphertext addition, a leveled operation.

Layers with PBS: activations and max-pooling

Activation layer (ReLU, sigmoid, tanh). Activations inject non-linearity and are crucial to the model. As established in the programmable-bootstrapping section, any function, including ReLU, sigmoid, or hyperbolic tangent, is evaluated homomorphically by a single PBS, with the function's outputs encoded in the test-polynomial look-up table. There is no need for FHE-friendly surrogate activations and no re-training.

Max-pooling layer. Extracts a fixed-size subset of components and returns their maximum. Although max is multivariate, the two-argument case reduces to ReLU: max(x, y) = y + ReLU(x - y). So from Encrypt(x) and Encrypt(y), one computes Encrypt(z) = Encrypt(x) - Encrypt(y), then Encrypt(max(x,y)) = Encrypt(y) + Encrypt(ReLU(z)). This costs a couple of ciphertext add/subtract operations plus one PBS (for the ReLU). For more than two arguments, max(x_1, ..., x_{k-1}, x_k) = max(y_k, x_k) with y_k = max(x_1, ..., x_{k-1}) is applied recursively, so a k-component max-pooling costs (k - 1) PBS.

Experimental results and benchmarks

Experiments used the MNIST dataset (28x28 handwritten-digit images). The authors designed depth-20, depth-50, and depth-100 networks (NN-20, NN-50, NN-100), each containing dense and convolution layers with activation functions; every hidden layer has at least 92 active neurons. The word size is Omega = 64 bits. Targeted security levels are 80 and 128 bits. Cryptographic parameters are given by (k, N, sigma) for GLWE and (n, sigma) for LWE, validated with the lwe-estimator. The three parameter sets support homomorphic inference of networks needing 8 to 12 bits of precision.

Conclusion and outlook

The authors present a general framework for evaluating deep neural networks under fully homomorphic encryption that scales efficiently with the number of layers while keeping good accuracy, using a versatile combination of encoding methods and programmable bootstrapping. They state the results set new records in homomorphic inference of deep neural networks. Future directions: assessing the impact of specialized hardware (where they expect several orders of magnitude in processing-time gains), and extending the techniques to homomorphic training of neural networks and to other intensive machine-learning tasks. The implementing library is an extended TFHE written in Rust, released open source as Concrete (https://github.com/zama-ai/concrete).

Security foundations (Appendix A). TFHE security rests on the hardness of torus-based problems: the LWE assumption and the GLWE assumption over the torus. The decisional LWE (resp. GLWE) assumption asserts that distinguishing a noisy linear combination from a uniformly random torus element is infeasible for a security parameter lambda. The paper gives the formal distinguishing-game definitions for both problems.
