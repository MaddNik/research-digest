Faster NTRU-based Bootstrapping with Extended and Sorting-based Techniques

IACR Cryptology ePrint Archive 2026/1447 · full view ↗

Overview and Contributions

NTRU-based bootstrapping is a high-performance variant of FHEW/TFHE-style bootstrapping. Because an NTRU ciphertext is a single polynomial (versus two for RLWE), it is simpler and faster than its RLWE counterpart, yet it remains expensive in both time and memory. The dominant cost is blind rotation, which performs many operations over an $N$-dimensional polynomial ring $R_{Q,N} = \mathbb{Z}_Q[X]/(X^N+1)$.

The central bottleneck the paper attacks is the constraint $q \mid 2N$, which ties the LWE ciphertext modulus $q$ to the NTRU ring dimension $N$. In applications such as homomorphic evaluation of Boolean functions, PPML, and PIR, $q$ can be as large as $2^{32}$; under $q \mid 2N$ a large $q$ forces a large $N$, and since blind rotation scales with $N$, efficiency degrades sharply. The paper's guiding question is:

Can we choose the NTRU dimension $N$ such that $2N < q$ when the ciphertext modulus $q$ becomes large?

Main contributions.

  • Extended bootstrapping (EBS). Using an extended isomorphism $\tau$ (from the EBS technique of Lee-Yoon [LY23] and the sorted-bootstrapping work [BORLT26]), the requirement $q \mid 2N$ is removed. One picks a large $N_{ext}$ with $q \mid 2N_{ext}$ and sets $N = N_{ext}/\eta$ (so that $\mathrm{NTRU}_{N,Q,\chi}$ stays hard), reducing every ring operation from dimension $N_{ext}$ to dimension $N$. Memory drops by a factor $\eta$.
  • Sorting-based bootstrapping (SBS). By reordering the external products so that only the two accumulators actually needed at the end are advanced, a large fraction of operations are eliminated, further cutting run time.
  • Multi-key extension. Both EBS and SBS are generalized to NTRU-based multi-key FHE (MK-FHE), where the dominant cost is the hybrid product rather than the external product.

Headline results (128-bit security, implemented with the CHIFHE and MKFHE libraries). With factor $\eta = 16$, single-key SBS is $1.82\times$ faster than FINAL (ASIACRYPT 2021) and $1.24\times$ faster than the LLWW key-unrolling method (CHES/TCHES 2024). With $\eta = 4$, multi-key SBS is $1.70\times$ faster than the prior art XZW (ASIACRYPT 2024). Both single-key and multi-key key sizes shrink by roughly a factor of $\eta$.

Preliminaries: Ciphertexts, Gadget Decomposition, and the External Product

Notation. $R_N = \mathbb{Z}[X]/(X^N+1)$ with $N$ a power of two, and $R_{q,N} = R_N/qR_N$. For $a \in R_N$, $\phi(a) = (a_0,\dots,a_{N-1})$ is its coefficient vector; $[n] = \{0,1,\dots,n-1\}$. The error distribution $\chi$ is a discrete Gaussian over $\mathbb{Z}_q$ or $\mathbb{Z}_q^n$.

Gadget decomposition. With decomposition base $B$ and $d = \lceil \log_B Q \rceil$, the gadget vector is $g = (1, B, \dots, B^{d-1})$. The signed decomposition of $a \in R_{Q,N}$ is

$$g^{-1}(a) = \left(\,[a]_B,\ \left[\left\lfloor \tfrac{a}{B} \right\rceil\right]_B,\ \dots,\ \left[\left\lfloor \tfrac{a}{B^{d-1}} \right\rceil\right]_B\,\right) = (a_0,\dots,a_{d-1}),$$

with $\lVert a_i \rVert_\infty \le B/2$ and $\langle g^{-1}(a), g\rangle_Q = a$.

Hard problems. Security rests on the decisional $\mathrm{LWE}_{n,q,\chi}$ problem, the decisional $\mathrm{NTRU}_{N,Q,\chi}$ problem (distinguish $g_e \cdot f^{-1} \in R_{Q,N}$ from uniform, with $f$ invertible), and the decisional vectorized-NTRU problem $\mathrm{VNTRU}_{N,Q,\chi}$ (the same over $R_{Q,N}^d$). The multi-key construction additionally relies on the decisional Hint-NTRU assumption [EEN+24].

Ciphertext types. LWE ciphertexts are B/FV-like: $\mathrm{LWE}^n_{s,q}(m) = (a, b)$ with $b = [\langle a, s\rangle + \lfloor q/t\rfloor m + e]_q$. Secret keys are binary, $s \in \{0,1\}^n$. The NTRU secret-key distribution $\chi_f$ draws $f = \sum f_i X^i$ with $f_i \in \{-1,0,1\}$ and $f$ invertible in $R_{Q,N}$. The two NTRU ciphertext forms are

$$\mathrm{NTRU}^N_{f,Q}(m) = \big[(g_e + m)\cdot f^{-1}\big]_Q, \qquad \mathrm{VNTRU}^N_{f,Q}(m) = \big[g_e \cdot f^{-1} + g\cdot m\big]_Q \in R_{Q,N}^d.$$

Multi-key NTRU ciphertexts $\mathrm{M.NTRU}^N_{\{s_i\},Q}(m) = (c_1,\dots,c_k)$ satisfy $\sum_i [c_i s_i]_Q = g_e + m$.

External product (Lemma 1). For $c = \mathrm{NTRU}^N_{f,Q}(u)$ and $c' = \mathrm{VNTRU}^N_{f,Q}(v)$, define $c \odot_N c' = \langle g^{-1}(c), c'\rangle$. The result is $\mathrm{NTRU}^N_{f,Q}(u\cdot v)$ with noise growth

$$\mathrm{Var}(\hat c) \le d\cdot N \cdot \frac{B^2}{12}\,\mathrm{Var}(c') + \lVert v\rVert_2^2\,\mathrm{Var}(c). \tag{1}$$

Functional bootstrapping evaluates a look-up table $h:\mathbb{Z}_t \to \mathbb{Z}_t$ in five steps: initialization (build the test polynomial $T(X)$ and an NTRU encryption of $T(X)\cdot X^{-\frac{2N}{q}b}$), blind rotation (homomorphically multiply by $X^{\frac{2N}{q}\sum a_i s_i}$ via external products), sample extraction (Lemma 2), key switching (Lemma 3, noise $+\,N\,d_{ks}\mathrm{Var}(ksk)$), and modulus switching (Lemma 4).

Single-Key Extended Bootstrapping (EBS)

Extended isomorphism. The key algebraic tool decomposes a large-ring polynomial into $\eta$ small-ring components. For $\eta \mid N$, the map $\iota: R_{q,N/\eta}\to R_{q,N}$, $\sum f_i X^i \mapsto \sum f_i X^{\eta i}$, is a ring homomorphism, making $R_{q,N}$ an $R_{q,N/\eta}$-module. The extended isomorphism (Lemma 6) is

$$\tau: R_{q,N} \to R_{q,N/\eta}^{\eta},\qquad m(X) = \sum_{i=0}^{N-1} m_i X^i \ \mapsto\ (m_0(X),\dots,m_{\eta-1}(X)),$$

where the $j$-th component collects every $\eta$-th coefficient, $m_j(X) = \sum_{i} m_{i\eta + j} X^i$. This is an $R_{q,N/\eta}$-module isomorphism (with explicit inverse $\hat\tau$), and $\tau_j$ denotes projection onto component $j$.

Monomial rotation under $\tau$ (Lemma 7). The crucial identity that lets rotation be tracked component-wise is

$$\tau_j\big(m(X)\cdot X^a\big) = m_{[(j-a)]_\eta}(X)\cdot X^{\left\lceil \frac{a-j}{\eta}\right\rceil}. \tag{2}$$

Equivalently (Corollary 1), if $c \in \mathrm{NTRU}^N_{f,Q}(\tau_{[j-a]_\eta}(m))$ then $c\cdot X^{\lceil (a-j)/\eta\rceil} \in \mathrm{NTRU}^N_{f,Q}(\tau_j(m\cdot X^a))$; the same holds for multi-key ciphertexts.

The EBS blind rotation. One chooses $N_{ext}$ with $q \mid 2N_{ext}$ and $N \mid N_{ext}$, keeping $\mathrm{NTRU}_{N,Q,\chi}$ hard, and sets $\eta = N_{ext}/N$. The bootstrapping starts from $T(X)\cdot X^{-\hat b} \in R_{Q,N_{ext}}$, applies $\tau$ to obtain $\eta$ small-ring polynomials, and takes one external product each with an initial key $evk' = \mathrm{VNTRU}^N_{f,Q}(f^{-1})$ to get $\eta$ accumulators $\{ACC_{0,j}\}_{j\in[\eta]}$. Each of the $n-1$ blind-rotation iterations applies $\eta$ CMux gates (Lemma 8):

$$ACC_{i+1,j} = ACC_{i,j} + \Big(ACC_{i,[j-\hat a_i]_\eta}\cdot X^{\left\lceil \frac{\hat a_i - j}{\eta}\right\rceil} - ACC_{i,j}\Big)\odot_N evk_i,\qquad evk_i = \mathrm{VNTRU}^N_{f,Q}(s_i). \tag{3}$$

A final single CMux produces $ACC_{n,0} \in \mathrm{NTRU}^N_{f,Q}\!\big(\tau_0(T(X)\cdot X^{-\hat b + \sum_i \hat a_i s_i})\big)$, whose constant coefficient equals the value needed by sample extraction. Figure 1 of the paper depicts this flow: an initialization producing $\eta$ accumulators, $n-1$ rounds of $\eta$ external products each, and one closing external product.

Cost and memory. EBS performs $\eta\cdot n + 1$ external products of dimension $N$, versus $n+1$ external products of dimension $N\cdot\eta$ in prior work. Because the working dimension drops from $N_{ext}$ to $N = N_{ext}/\eta$, the bootstrapping keys ($(n+1)\,d\,N\log_2 Q$ bits) and key-switching keys ($2\,N\,B_{ks}\,d_{ks}\log_2 Q_{ks}$ bits) both shrink by a factor $\eta$.

Correctness and noise (Theorem 1). Algorithm 1 outputs $ct' \in \mathrm{LWE}^n_{s,q}(h(m))$ with

$$\mathrm{Var}(ct') \le \frac{q^2}{Q_{ks}^2}\left(\frac{Q_{ks}^2}{Q^2}\,\sigma_{BR}^2 + \sigma_{MS1}^2 + \sigma_{KS}^2\right) + \sigma_{MS2}^2, \tag{4}$$

where $\sigma_{BR}^2 = (n+1)\,d\,N\,\frac{B^2}{12}\,\sigma^2$, $\sigma_{MS1}^2 = \frac{N+1}{12}$, $\sigma_{KS}^2 = N\,d_{ks}\,\sigma_L^2$, and $\sigma_{MS2}^2 = \frac{n+1}{12}$; $\sigma^2$ is the NTRU evaluation-key noise variance and $\sigma_L^2$ the LWE key-switch-key variance. The proof tracks the per-component invariant $\max_j \mathrm{Var}(ACC_{i+1,j}) \le \max_j \mathrm{Var}(ACC_{i,j}) + d\,N\,\frac{B^2}{12}\sigma^2$.

Sorting-Based Technique (SBS)

EBS lowers each external product from dimension $\eta N$ to $N$, but the number of products is still large ($\eta n + 1$). The sorting-based technique observes that the final accumulator $ACC_{n,0}$ depends on only a small subset of the intermediate accumulators, so most component updates are wasted work.

The observation. Suppose that from some iteration $i^\ast$ onward all remaining $\hat a_i$ share a factor $2^k$, i.e. $2^k \mid \hat a_i$ for $i^\ast \le i \le n-1$. Then in the recurrence (Equation 3) the index $[j - \hat a_i]_\eta$ preserves divisibility by $2^k$, so to compute the needed $\{ACC_{n-1,j}\}_{2^k \mid j}$ it suffices to keep only the $\eta/2^k$ accumulators with $2^k \mid j$. Each such iteration then needs only $\eta/2^k$ CMux gates instead of $\eta$.

General reordering. The $\hat a_i$ are bucketed by their 2-adic valuation,

$$I_k = \big\{\, i \in [n]\ \big|\ \hat a_i \equiv 0 \pmod{2^k},\ \hat a_i \not\equiv 0 \pmod{2^{k+1}} \,\big\},\qquad 0 \le k \le \log \eta,$$

and the sum is rewritten as $\sum_{i=0}^{n-1}\hat a_i s_i = \sum_{k=0}^{\log\eta}\sum_{i\in I_k}\hat a_i s_i$. Processing bucket $I_k$ requires only $\eta/2^k$ CMux gates per index. Because the monomials $X^{\hat a_i s_i}$ can be applied in any order, the buckets can be handled from least-divisible to most-divisible, progressively collapsing the number of live accumulators.

Resulting cost. On average, SBS (Algorithm 2) needs

$$\frac{2\eta^2 + 1}{3\eta}\,n + \eta \quad\text{external products of dimension } N, \tag{5}$$

compared with $\eta n + 1$ for EBS. The noise bound is identical to Equation 4 (Theorem 2), since the operations performed are a subset/reordering of the same external products.

Security caveat. Beyond LWE, NTRU, and VNTRU hardness, both schemes require a circular-security assumption because initialization publishes a VNTRU encryption of $f^{-1}$ under $f$. The paper also notes SBS's practical gains are bounded by the security constraints on the NTRU parameters (the achievable $\eta$).

Multi-Key Bootstrapping (MK-EBS and MK-SBS)

Multi-key FHE (MK-FHE) allows homomorphic computation across ciphertexts encrypted under $k$ distinct secret keys. A multi-key LWE ciphertext is $\mathrm{M.LWE}^n_{\{z_i\},q}(m) = (b, a_1,\dots,a_k)$ with $b = [\sum_i \langle a_i, z_i\rangle + \lfloor q/t\rfloor m + e]_q$. Multi-key bootstrapping mirrors the single-key five-step pipeline (Figure 2 of the paper: M.Initialization $\to$ M.Blind Rotation $\to$ M.SampleExtract $\to$ M.ModSwitch + M.KeySwitch), but the dominant operation is the hybrid product instead of the external product.

Hybrid product (Lemma 12, full scheme in Appendix A). Given a multi-key ciphertext $c \in \mathrm{M.NTRU}^N_{\{s_i\},Q}(m)$, public keys $\{b_j\}$, and a uni-encryption $(d_i, f_i) \in \mathrm{UniEnc}(\mu, s_i)$, $\mathrm{HbProd}$ outputs $c' \in \mathrm{M.NTRU}^N_{\{s_i\},Q}(\mu\cdot m)$ with noise growth

$$\mathrm{Var}(c') \le \mathrm{Var}(c) + (1 + 2k\cdot N)\,d\,N\,\frac{B^2}{12}\,\sigma^2.$$

The multi-key building blocks (M.SampleExtract, M.KeySwitch with $+\,k\,N\,d_{ks}\mathrm{Var}(ksk)$ noise, and M.ModSwitch) follow [XZW+25, KMS24]; in the implementation the $N$ LWE ciphertexts are packed into one RLWE ciphertext as in [XZW+25].

Extending EBS/SBS. The multi-key CMux gate (Lemma 13) replaces $\odot_N$ with $\mathrm{HbProd}$:

$$ACC_\ell \leftarrow ACC_\ell + \mathrm{HbProd}\!\Big(X^{\hat a_{i,j}}\!\cdot ACC_{[\ell - \hat a_{i,j}]_\eta} - ACC_\ell,\ \{pk_i\},\ evk_{i,j}\Big).$$

MK-EBS (Algorithm 3) runs over $k$ parties and $n$ positions, applying this gate for all $(i,j)\ne(k,n-1)$ and closing with a final gate. MK-SBS (Algorithm 4) buckets the pairs $(i,j)$ by 2-adic valuation, $I_m = \{(i,j) : \hat a_{i,j}\equiv 0 \bmod 2^m,\ \hat a_{i,j}\not\equiv 0 \bmod 2^{m+1}\}$, and updates only the $\eta/2^m$ accumulators per bucket.

Noise (Theorems 3, 4). Both output $ct'\in\mathrm{M.LWE}^n_{\{z_i\},q}(m)$ with

$$\mathrm{Var}(ct') \le \frac{q^2}{Q_{ks}^2}\!\left(\frac{Q_{ks}^2}{Q^2}\big(\sigma_{MBR}^2 + \sigma_{MMS1}^2 + \sigma_{MKS}^2\big) + \sigma_{MMS2}^2\right),$$

where the multi-key blind-rotation noise is $\sigma_{MBR}^2 = \big(k\,n\,(1 + 2k\,N) + 1\big)\,d\,N\,\frac{B^2}{12}\,\sigma^2$, $\sigma_{MKS}^2 = k\,N\,d_{ks}\,\sigma_L^2$, and the modulus-switching terms are $\sigma_{MMS1}^2 = \frac{1 + Nk}{12}$, $\sigma_{MMS2}^2 = \frac{1 + nk}{12}$. Security additionally needs the decisional Hint-NTRU assumption (Appendix B), which underlies NTRU uni-encryption.

Conclusions

The paper eliminates the $q \mid 2N$ constraint in NTRU-based bootstrapping via an extended isomorphism, letting a large modulus $q$ be used with a small ring dimension $N$. This alone shrinks keys by a factor $\eta$ and speeds up external-product and key-switching work; the sorting-based reordering then removes redundant external/hybrid products for a further time reduction. The techniques carry over to multi-key NTRU bootstrapping. Measured gains at 128-bit security are $1.82\times$/$1.24\times$ (single-key SBS vs. FINAL/LLWW) and $1.70\times$ (multi-key SBS vs. XZW), with $\sim 1/\eta$ memory. The authors note the achievable $\eta$ is limited by NTRU security constraints and suggest parallel implementations or hard problems with more computation-friendly structure as future directions.

Algorithms

The four bootstrapping algorithms and the hybrid-product scheme, transcribed from the paper.

Algorithm 1 — NTRU-based Bootstrapping with Extended Techniques (EBS).

Input:  LWE sample ct = (b, a) in LWE_{s,q}^n(m)
        evaluation keys {evk_i}, i in [n], evk_i in VNTRU_{f,Q}^N(s_i)
        special evaluation key evk' = VNTRU_{f,Q}^N(f^{-1})
        LWE key-switching key ksk_{phi(f)->s}
Output: LWE sample ct' in LWE_{s,q}^n(h(m))

1. Set T(X) = sum_{i=0}^{N_ext-1} (Q/t)*h(round((t/q)*i)) * X^i
   Set (b_hat, a_hat) = round((2*N_ext/q) * ct)

2. for j = 0 to eta-1:                       # initialization: eta accumulators
       ACC_{0,j} <- ( tau_j(T(X) * X^{-b_hat}) ) (X)_N evk'

3. for i = 0 to n-2:                          # n-1 rounds of eta CMux gates
       for j = 0 to eta-1:
           ACC_{i+1,j} <- ACC_{i,j}
                        + ( ACC_{i,[j-a_hat_i]_eta} * X^{ceil((a_hat_i - j)/eta)}
                            - ACC_{i,j} ) (X)_N evk_i

4. ACC_{n,0} = ACC_{n-1,0}
             + ( ACC_{n-1,[-a_hat_{n-1}]_eta} * X^{ceil(a_hat_{n-1}/eta)}
                 - ACC_{n-1,0} ) (X)_N evk_{n-1}

5. ct' <- SampleExtract(ACC_{n,0})
6. ct' <- ModSwitch_{Q->Q_ks}(ct')
7. ct' <- LWE.KeySwitch(ct')
8. ct' <- ModSwitch_{Q_ks->q}(ct')
   return ct'

# (X)_N denotes the N-dimensional external product; total = eta*n + 1 products.

Algorithm 2 — NTRU-based Bootstrapping with Sorting-based Techniques (SBS).

Input / Output: same keys and I/O as Algorithm 1.

1. Set T(X) = sum_{i=0}^{N_ext-1} (Q/t)*h(round((t/q)*i)) * X^i
   Set ct_hat = (a_hat, b_hat) = round((2*N_ext/q) * ct)

   for j = 0 to eta-1:                        # initialization
       ACC_j <- ( tau_j(T(X) * X^{-b_hat}) ) (X)_N evk'

2. # buckets I_k = { i in [n] : a_hat_i = 0 mod 2^k, a_hat_i != 0 mod 2^{k+1} }
   for k = 0 to log(eta):
       for i = 0 to |I_k| - 1:                # i ranges over indices in bucket I_k
           for j = 0 to eta/2^k - 1:          # only eta/2^k live accumulators
               ACC_{j*2^k} <- ACC_{j*2^k}
                   + ( ACC_{[j*2^k - a_hat_i]_eta} * X^{ceil((a_hat_i - j*2^k)/eta)}
                       - ACC_{j*2^k} ) (X)_N evk_i

3. ct' <- SampleExtract(ACC_0)
4. ct' <- ModSwitch_{Q->Q_ks}(ct')
5. ct' <- LWE.KeySwitch(ct')
6. ct' <- ModSwitch_{Q_ks->q}(ct')
   return ct'

# Average cost = ((2*eta^2 + 1)/(3*eta)) * n + eta external products.

Algorithm 3 — NTRU-based Multi-key Bootstrapping with Extended Techniques (MK-EBS).

Input:  multi-key LWE ciphertext ct = (b, a_1, ..., a_k) in Z_q^{kn+1}
        public keys {pk_i}_{1<=i<=k}
        evaluation keys {evk_{i,j} in UniEnc(z_{i,j}, s_i)}_{1<=i<=k, j in [n]}
        special key evk' in VNTRU_{s_1,Q}^N(s_1^{-1})
        key-switching keys ksk_{{phi(s_i)} -> {z_i}}
        test polynomial T(X) in R_{Q,N_ext}
Output: multi-key LWE ciphertext ct' in Z_q^{kn+1}

1. ct_hat = (b_hat, a_hat_1, ..., a_hat_k) <- round((2*N_ext/q) * ct)

2. for l = 0 to eta-1:                        # initialization
       c_bar_l <- ( tau_l(T(X) * X^{-b_hat}) ) (X)_N evk'
       ACC_l   <- (c_bar_l, 0) in R_{Q,N}^k

3. for i = 1 to k:
       for j = 0 to n-1:
           for l = 0 to eta-1:
               if (i,j) != (k, n-1):
                   ACC_l <- ACC_l
                       + HbProd( X^{ceil((a_hat_{i,j} - l)/eta)} * ACC_{[l - a_hat_{i,j}]_eta}
                                 - ACC_l, {pk_i}, evk_{i,j} )

4. ACC_0 <- ACC_0
          + HbProd( X^{ceil(a_hat_{k,n-1}/eta)} * ACC_{[-a_hat_{k,n-1}]_eta}
                    - ACC_0, {pk_i}, evk_{k,n-1} )

5. ct' <- M.SampleExtract(ACC_0)
6. ct' <- M.ModSwitch_{Q->Q_ks}(ct')
7. ct' <- M.KeySwitch_{{phi(s_i)} -> {z_i}}(ct')
8. ct' <- M.ModSwitch_{Q_ks->q}(ct')
   return ct'

# Cost = k*eta*n - eta + 1 hybrid products.

Input / Output: same keys and I/O as Algorithm 3.

1. ct_hat = (b_hat, a_hat_1, ..., a_hat_k) <- round((2*N_ext/q) * ct)

Appendix A — Hybrid Product scheme (uni-encryption based).

Setup(1^lambda):
    choose NTRU degree N, modulus Q, gadget dimension d,
    secret distribution chi_f, error distribution chi_e over R_N;
    sample a <- R_{Q,N}^d;
    return pp' = (N_ext, N, Q, d, a, chi_f, chi_e).

KeyGen(pp'):
    for i = 1..k: sample s_i <- chi_f (s_i invertible in R_{Q,N}), e_i <- chi_e^d;
    output public keys { b_i = -a * s_i + e_i }_{1<=i<=k}.

UniEnc(mu, s):                       # mu in Z, secret key s <- chi_f
    sample r <- chi_s, noise e_1, e_2 <- chi_e^d;
    output (d, f) = ( r*a + mu*g + e_1,   e_2*s^{-1} + r*g*s^{-1} )  in R_{Q,N}^d x R_{Q,N}^d.

HbProd(c, {b_j}_{1<=j<=k}, (d_i, f_i)):     # c in M.NTRU_{{s_i}}^N(m), (d_i,f_i) in UniEnc(mu, s_i)
    1. u_j = < g^{-1}(c_j), d_i >           for 1 <= j <= k
       v   = sum_{j=1}^{k} < g^{-1}(c_j), b_j >
    2. for 1 <= j <= k:
           c'_j = u_i + < g^{-1}(v), f_i >   if j = i
           c'_j = u_j                        otherwise
    return c' = (c'_1, ..., c'_k)  in R_{Q,N}^k, with c' in M.NTRU_{{s_i}}^N(mu * m).

Supporting sub-routines (used above, stated as lemmas in the paper):

SampleExtract(c):     # c in NTRU_{f,Q}^N(m), c = sum c_i X^i
    return (0, c_0, -c_{N-1}, -c_{N-2}, ..., -c_1)  in LWE_{phi(f),Q}^N(m_0).

KeySwitch_{phi(f)->s}(ct):    # ct=(b,a) in LWE_{phi(f),Q_ks}^N(m)
    return (b,0) - sum_{i=0}^{N-1} sum_{j=0}^{d_ks-1} ksk_{i,j,a_{i,j}}   in LWE_{s,Q_ks}^n(m).

ModSwitch_{Q->q}(ct):         # ct=(b,a) in LWE_{s,Q}^n(m)
    return ( round((q/Q)*b), round((q/Q)*a) )   in LWE_{s,q}^n(m).
