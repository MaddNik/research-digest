Homomorphic Encryption from Learning with Errors: Conceptually-Simpler, Asymptotically-Faster, Attribute-Based

IACR ePrint 2013/340 (full version of CRYPTO 2013, LNCS 8042, pp. 75-92) · full view ↗

Overview and Contributions

This paper by Craig Gentry (IBM Research), Amit Sahai (UCLA) and Brent Waters (UT Austin) introduces what is now universally called the GSW scheme: a comparatively simple leveled fully homomorphic encryption (FHE) scheme based on the Learning With Errors (LWE) problem. Its central idea is the approximate eigenvector method: a ciphertext is a matrix $C$, the secret key $v$ is a vector that is an approximate eigenvector of $C$, and the encrypted message $\mu$ is the corresponding eigenvalue. With this view, homomorphic addition and multiplication become (almost) ordinary matrix addition and multiplication.

The four headline results are:

  • Conceptually simpler FHE from LWE. The whole scheme is described in the introduction. Earlier LWE-based schemes (Brakerski-Vaikuntanathan, BGV, Brakerski) needed a "tricky" multiplication built on tensoring plus relinearization; GSW removes relinearization entirely.
  • Asymptotically faster FHE from LWE. Eliminating relinearization removes the large relinearization matrices and their $\Omega(n^3)$ cost; ciphertexts are matrices multiplied with sub-cubic matrix multiplication, giving per-gate cost $\tilde{O}((nL)^\omega)$ with $\omega < 2.3727$ versus the prior $\tilde{O}(n^3 L)$.
  • Identity-based FHE (IBFHE). Because the scheme has no evaluation key (the evaluator needs only public parameters such as $n, m, \ell$), the authors give the first identity-based FHE, solving a long-standing open problem, via a compiler that turns suitable LWE-based IBE schemes into IBFHE.
  • Attribute-based FHE (ABFHE). The same compiler, with minor changes, converts the Gorbunov-Vaikuntanathan-Wee (GVW) ABE-for-circuits scheme into an attribute-based FHE in which ciphertexts under a common index can be processed homomorphically with no evaluation key.

The scheme inherits the good properties of prior LWE FHE: bootstrapping is optional; with bootstrapping, security rests on LWE for quasi-polynomial (rather than sub-exponential) factors; no modulus switching is needed; and security can be based directly on the classical hardness of $\mathrm{GapSVP}$.

Background: Previous LWE-based FHE and the cost of relinearization

In prior LWE-based FHE schemes (Brakerski-Vaikuntanathan, BGV, Brakerski), a ciphertext $c$ and secret key $s$ are $n$-dimensional vectors whose dot product encodes the message up to small error, $\langle c, s \rangle \approx \mu$, with the error removed by rounding. Homomorphic multiplication exploits the tensor identity

$$\langle u_1 \otimes u_2,\; v_1 \otimes v_2 \rangle = \langle u_1, v_1 \rangle \cdot \langle u_2, v_2 \rangle,$$

so that if $\langle c_1, s\rangle \approx \mu_1$ and $\langle c_2, s\rangle \approx \mu_2$, then $\langle c_1 \otimes c_2,\; s \otimes s\rangle \approx \mu_1 \mu_2$. The product ciphertext $c_1 \otimes c_2$ and key $s \otimes s$ have dimension $\Theta(n^2)$, so tensoring blows up size and can only run for a constant number of steps. To recover, the evaluator must relinearize: multiply the long ciphertext by a special $n \times \Theta(n^2)$ matrix to compress it back to an $n$-dimensional ciphertext under a normal key.

Relinearization is powerful but unintuitive and expensive: each relinearization matrix has size $\Omega(n^3)$, the public key must hold $L$ of them for multiplicative depth $L$, and the procedure costs $\Omega(n^3)$ operations per gate (each operation polynomial in $L$). These matrices form the evaluation key the evaluator must obtain. GSW asks: can multiplication just be $c_1 \cdot c_2$ over some ring, with the product having the same form as the inputs, and the evaluation key eliminated entirely? The answer is yes.

The Approximate Eigenvector Method

Fix a modulus $q$ and dimension $N$. A ciphertext $C$ is an $N \times N$ matrix over $\mathbb{Z}_q$ with small entries (much smaller than $q$); the secret key $v$ is an $N$-dimensional vector over $\mathbb{Z}_q$ with at least one big coefficient $v_i$; the message $\mu$ is a small integer. We say $C$ encrypts $\mu$ when

$$C \cdot v = \mu \cdot v + e,$$

where $e$ is a small error vector (Equation 1). To decrypt, take row $C_i$, compute $x \leftarrow \langle C_i, v\rangle = \mu \cdot v_i + e_i$, and output $\mu = \lfloor x / v_i \rceil$. In short, $v$ is an approximate eigenvector of $C$ and $\mu$ is the eigenvalue.

Why matrix operations are correct homomorphisms: suppose $C_1 v = \mu_1 v + e_1$ and $C_2 v = \mu_2 v + e_2$. For addition, $C_+ = C_1 + C_2$ gives

$$C_+ \cdot v = (\mu_1 + \mu_2)\,v + (e_1 + e_2).$$

For multiplication, $C_\times = C_1 \cdot C_2$ gives

$$C_\times \cdot v = C_1(\mu_2 v + e_2) = \mu_2(\mu_1 v + e_1) + C_1 e_2 = \mu_1 \mu_2 \cdot v + \mu_2 e_1 + C_1 e_2 = \mu_1 \mu_2 \cdot v + \text{small}.$$

The final error is small because $\mu_2, C_1, e_1, e_2$ are all small. Notably $C_2 \cdot C_1$ also encrypts $\mu_1\mu_2$ even though matrix multiplication is not commutative. In the idealized error-free case $C_i v = \mu_i v$ exactly, $v$ is a genuine common eigenvector and the eigenvalues multiply. Introducing error is what lets security rest on LWE; the price is that several terms must stay small to preserve the ciphertext form. The authors name this the approximate eigenvector method.

Error Growth and Somewhat-Homomorphic Encryption

Correctness holds until the error coefficients approach $q$. Call $C$ $B$-bounded if $\mu$ and all coefficients of $C$ and $e$ have magnitude at most $B$. Then:

  • $C_+$ is $2B$-bounded;
  • $C_\times$ is $(N+1)B^2$-bounded.

So error grows roughly like $B^{2^L}$, doubly exponentially in the multiplicative depth $L$. Equivalently, evaluating a multivariate polynomial $P(x_1,\dots,x_t)$ of total degree $d$ on $B$-bounded inputs yields a ciphertext that is $|P|\,(N+1)^{d-1} B^d$-bounded, where $|P|$ is the $\ell_1$-norm of $P$'s coefficient vector. Taking $q$ to exceed this bound, the scheme-so-far evaluates polynomials of degree about $\log_{NB} q$. Since $q/B$ must be at most sub-exponential in $N$ for security, this gives only polynomial degree (logarithmic depth): a somewhat homomorphic encryption (SWHE) scheme, but the most homomorphic LWE-based scheme using only natural operations.

Flattening Ciphertexts: from SWHE to leveled FHE

To get leveled FHE without bootstrapping or relinearization, the error growth must be tamed. Call $C$ $B$-strongly-bounded if $\mu$ and the coefficients of $C$ all have magnitude $\le 1$ while the coefficients of $e$ have magnitude $\le B$. Evaluating a NAND as $C_3 \leftarrow I_N - C_1 \cdot C_2$ on strongly-bounded inputs keeps the message in $\{0,1\}$ and makes $C_3$'s error at most $(N+1)B$. If $C_3$'s own coefficients could be forced back to magnitude $\le 1$, strong-boundedness would be preserved and a depth-$L$ circuit would keep error at most $(N+1)^L B$; with $q/B$ sub-exponential in $N$ this supports polynomial depth, i.e. leveled FHE.

The fix is the Flatten operation, built from transformations that change vectors without changing dot products. Let $a, b$ be $k$-dimensional vectors over $\mathbb{Z}_q$, set $\ell = \lfloor \log_2 q\rfloor + 1$ and $N = k \cdot \ell$. Define:

  • $\mathrm{BitDecomp}(a) = (a_{1,0},\dots,a_{1,\ell-1},\dots,a_{k,0},\dots,a_{k,\ell-1})$, where $a_{i,j}$ is the $j$-th bit (LSB first) of $a_i$.
  • $\mathrm{BitDecomp}^{-1}(a') = \big(\sum_j 2^j a_{1,j},\dots,\sum_j 2^j a_{k,j}\big)$ — defined even when $a'$ is not a 0/1 vector.
  • $\mathrm{Flatten}(a') = \mathrm{BitDecomp}(\mathrm{BitDecomp}^{-1}(a'))$, which has 0/1 coefficients.
  • $\mathrm{Powersof2}(b) = (b_1, 2b_1,\dots,2^{\ell-1}b_1,\dots,b_k,2b_k,\dots,2^{\ell-1}b_k)$.

For matrices, each operation is applied row by row. The key identities are

$$\langle \mathrm{BitDecomp}(a),\, \mathrm{Powersof2}(b)\rangle = \langle a, b\rangle,$$

$$\langle a',\, \mathrm{Powersof2}(b)\rangle = \langle \mathrm{BitDecomp}^{-1}(a'), b\rangle = \langle \mathrm{Flatten}(a'),\, \mathrm{Powersof2}(b)\rangle.$$

Thus $\mathrm{Flatten}$ shrinks coefficients to $\{0,1\}$ without changing the product with $\mathrm{Powersof2}(b)$ and without knowing $b$. The trick is to give the secret key the special form $v = \mathrm{Powersof2}(s)$. Then for any $N\times N$ matrix $C$, $\;\mathrm{Flatten}(C)\cdot v = C \cdot v$. After computing $C_3 \leftarrow I_N - C_1 C_2$ for NAND, set $C_{\mathrm{NAND}} = \mathrm{Flatten}(C_3)$ to recover a 0/1, strongly-bounded ciphertext. This yields leveled FHE under a fixed approximate-eigenvector secret key, with no relinearization.

Key Generation, Encryption and the Reduction to LWE

Security is based on LWE by generating the public key from an LWE instance. For parameters $q, n, m = O(n\log q)$, an LWE instance is an $m \times (n+1)$ matrix $A$ for which there is an $(n+1)$-vector $s$ with first coefficient $1$ such that $e = A \cdot s$ is small. Here $A$ is public, $s$ is secret, and the approximate eigenvector is $v = \mathrm{Powersof2}(s)$ of dimension $N = (n+1)\cdot\ell$.

To encrypt $\mu \in \mathbb{Z}_q$, sample a uniform $N \times m$ matrix $R \in \{0,1\}^{N\times m}$ and output

$$C = \mathrm{Flatten}\big(\mu \cdot I_N + \mathrm{BitDecomp}(R \cdot A)\big) \in \mathbb{Z}_q^{N\times N}.$$

Because $\mathrm{Flatten}$ does not change the product with $v$,

$$C \cdot v = \mu \cdot v + \mathrm{BitDecomp}(R\cdot A)\cdot v = \mu \cdot v + R \cdot A \cdot s = \mu \cdot v + \text{small}.$$

Security argument. Let $C' = \mathrm{BitDecomp}^{-1}(C)$. Since $C$ is flattened, $C = \mathrm{BitDecomp}(C')$, so $C$ reveals nothing beyond $C'$. But $C' = \mathrm{BitDecomp}^{-1}(\mu I_N) + R\cdot A$, and $R\cdot A$ is statistically uniform by the leftover hash lemma when $m = O(n\log q)$ is chosen suitably (concretely $m > 2n\log q$ suffices). Hence $C$ is statistically independent of $\mu$ when $A$ is uniform, and semantic security reduces to LWE.

Preliminaries: LWE and GapSVP

Definition 1 (LWE). For security parameter $\lambda$, dimension $n=n(\lambda)$, modulus $q=q(\lambda)\ge 2$, and distribution $\chi$ over $\mathbb{Z}$, the $\mathrm{LWE}_{n,q,\chi}$ problem is to distinguish: (i) samples $(a_i,b_i)$ drawn uniformly from $\mathbb{Z}_q^{n+1}$; from (ii) samples where $s \leftarrow \mathbb{Z}_q^n$ is fixed, $a_i \leftarrow \mathbb{Z}_q^n$ uniform, $e_i \leftarrow \chi$, and $b_i = \langle a_i, s\rangle + e_i$. Viewing the $b_i \| a_i$ as rows of a matrix $A$ and redefining $s$ as $(1, -s)$, either $A$ is uniform or there is an $s$ with first coefficient $1$ such that $A\cdot s = e$ with $e$ drawn from $\chi$.

Definition 2 ($B$-bounded distributions). An ensemble $\{\chi_n\}$ over the integers is $B$-bounded if $\Pr_{e\leftarrow\chi_n}[\,|e| > B\,] = \mathrm{negl}(n)$.

Theorem 1 (worst-case hardness, after Regev/Peikert/MM/MP, as restated by Brakerski). Let $q$ be a prime power or product of small distinct primes and $B \ge \omega(\log n)\sqrt{n}$. Then there is an efficiently sampleable $B$-bounded $\chi$ such that an efficient algorithm solving average-case $\mathrm{LWE}_{n,q,\chi}$ yields: an efficient quantum algorithm for $\mathrm{GapSVP}_{\tilde{O}(nq/B)}$; and, if $q \ge \tilde{O}(2^{n/2})$, an efficient classical algorithm for $\mathrm{GapSVP}_{\tilde{O}(nq/B)}$. For sub-polynomial distinguishing advantage one needs $B \ge \tilde{O}(n)$ with approximation factor slightly above $\tilde{O}(n^{1.5} q/B)$.

Theorem 2 (informal, Brakerski et al.). Solving $n$-dimensional LWE with $\mathrm{poly}(n)$ modulus implies an equally efficient solution to a worst-case lattice problem (e.g. $\mathrm{GapSVP}$) in dimension $\sqrt{n}$. $\mathrm{GapSVP}_\gamma$ asks whether an $n$-dimensional lattice has a vector shorter than $d$ or none shorter than $\gamma(n)\cdot d$.

The Formal Scheme (Section 3.1) and its Security

$\mathrm{KeyGen}$ is split into $\mathrm{Setup}$, $\mathrm{SecretKeyGen}$, $\mathrm{PublicKeyGen}$; two decryption routines are given. $\mathrm{Dec}$ recovers $\mu$ in a small space (e.g. $\{0,1\}$); $\mathrm{MPDec}$ (the Micciancio-Peikert procedure) recovers any $\mu \in \mathbb{Z}_q$. The complete algorithms appear in the appendix below.

Decryption correctness: if $C$ is well-formed then $C\cdot v = \mu\cdot v + R\cdot A\cdot s = \mu\cdot v + R\cdot e$. $\mathrm{Dec}$ uses only coordinate $i$, namely $x_i = \mu v_i + \langle R_i, e\rangle$, with error $\le \|e\|_1$. In general, if $x_i = \mu v_i + e'$ with $|e'| \le q/8$ and $v_i \in (q/4, q/2]$, then $x_i/v_i$ differs from $\mu$ by less than $1/2$ and rounding recovers $\mu$.

Security (Lemma 1, implicit in Regev). Note $\mathrm{BitDecomp}^{-1}(C) = \mu\cdot G + R\cdot A$, where $G = \mathrm{BitDecomp}^{-1}(I_N)$ is (the transpose of) the Micciancio-Peikert primitive matrix used to build lattice trapdoors, and the rows of $R\cdot A$ are Regev encryptions of $0$. If params satisfy $\mathrm{LWE}_{n,q,\chi}$ and $m = O(n\log q)$, then $(A, R\cdot A)$ is computationally indistinguishable from uniform over $\mathbb{Z}_q^{m\times(n+1)} \times \mathbb{Z}_q^{N\times(n+1)}$. Since $C$ is derivable from $\mathrm{BitDecomp}^{-1}(C)$ by applying $\mathrm{BitDecomp}$, hiding $\mu$ in $C'$ implies hiding it in $C$.

Like Brakerski, GSW can also base security on $\mathrm{GapSVP}$ via a classical reduction (Peikert), where $q$ may even be exponential in $n$ — tolerable here because the fresh-ciphertext error bound $B$ need not be small to allow many homomorphic operations; only $q/B$ must be sub-exponential. Alternatively a sub-exponential $q$ with the Brakerski-Langlois-Peikert-Regev-Stehle reduction works for polynomial moduli (at the cost of a smaller $\mathrm{GapSVP}$ dimension).

Homomorphic Operations (Section 3.3) and Circuits

Beyond the basic transforms, the scheme provides $\mathrm{MultConst}, \mathrm{Add}, \mathrm{Mult}, \mathrm{NAND}$, all over the full base ring $\mathbb{Z}_q$.

MultConst$(C,\alpha)$: set $M_\alpha \leftarrow \mathrm{Flatten}(\alpha I_N)$, output $\mathrm{Flatten}(M_\alpha\cdot C)$. Then $\mathrm{MultConst}(C,\alpha)\cdot v = \alpha\mu\cdot v + M_\alpha e$, so error grows by a factor $\le N$ regardless of $\alpha$ — unlike repeated addition, whose error blows up by $\alpha$. This enables homomorphic FFTs natively over $\mathbb{Z}_q$ without field-size-dependent error growth.

Add$(C_1,C_2)$: output $\mathrm{Flatten}(C_1+C_2)$; correctness is immediate.

Mult$(C_1,C_2)$: output $\mathrm{Flatten}(C_1\cdot C_2)$, with $\mathrm{Mult}(C_1,C_2)\cdot v = \mu_1\mu_2\cdot v + \mu_2 e_1 + C_1 e_2$. The 0/1 entries of $C_1$ contribute at most a factor-$N$ blowup, but the dependence on the message $\mu_2$ is a concern, addressed by restricting to small messages.

NAND$(C_1,C_2)$: for $\mu_1,\mu_2 \in \{0,1\}$, output $\mathrm{Flatten}(I_N - C_1 C_2)$, with

$$\mathrm{NAND}(C_1,C_2)\cdot v = (I_N - C_1 C_2)\cdot v = (1 - \mu_1\mu_2)\,v - \mu_2 e_1 - C_1 e_2.$$

Because $\mu_2 \in \{0,1\}$, error grows by a factor of at most $N+1$, and the output message stays in $\{0,1\}$ — the invariant that keeps messages small.

Circuits. A Boolean circuit converted to NAND gates of depth $L$ over fresh $B$-bounded encryptions of $\{0,1\}$ yields a final error bounded by $(N+1)^L\cdot B$. For arithmetic circuits over $\mathbb{Z}_q$ with addition, multiplication, and constant-multiplication gates, one restricts inputs to be bounded by some $T$ and chooses the circuit so all intermediate values stay bounded by $T'$; the final error is then bounded by $(N+T')^L\cdot B$. This evaluates degree-$d$ polynomials when messages are bounded by about $q^{1/d}$ (a SWHE variant), and small bounded values can be converted to binary encodings for subsequent NAND-based Boolean computation.

Parameters, Performance and Bootstrapping (Section 3.4)

If flattened $C_1, C_2$ encrypt $\mu_1,\mu_2 \in \{0,1\}$ under $v$ with $B$-bounded error ($|e_i|_\infty \le B$), then $C_{\mathrm{NAND}}$ encrypts $\mathrm{NAND}(\mu_1,\mu_2)$ with $(N+1)B$-bounded error. As long as

$$q/B > 8(N+1)^L,$$

a depth-$L$ NAND circuit produces a $q/8$-bounded ciphertext that $\mathrm{Dec}$ decrypts correctly. As in prior LWE FHE, $n$ (hence $N$) grows linearly with $\log(q/B)$ to keep $2^\lambda$ security, so $q/B$ behaves like $\exp(L\log L)$ and $\kappa = \log q = O(L\log N) = O(L\log n)$. Since a NAND is dominated by multiplying two $N\times N$ matrices with $N = O(n\kappa) = \tilde{O}(nL)$:

Theorem 3. For dimension $n$ and depth $L$, the scheme evaluates depth-$L$ NAND circuits with $\tilde{O}((nL)^\omega)$ field operations per gate, where $\omega < 2.3727$ is the matrix-multiplication exponent. This beats prior LWE FHE, which need at least $\tilde{O}(n^3 L)$ field operations per gate; accounting for hidden factors, prior schemes are really $\tilde{O}(n^3 L^2)$ (operations involve general $\mathbb{Z}_q$ elements) while GSW stays $\tilde{O}((nL)^\omega)$ thanks to 0/1 ciphertext entries. The trade-off: $N = O(nL\log n)$ hides log factors in the matrix dimension. Space complexity drops from quasi-cubic to quasi-quadratic, a practical win for LWE FHE.

Bootstrapping. Since $\mathrm{Dec}$ is essentially Regev decryption (evaluable in $O(\log n)$ depth), bootstrapping works as in prior LWE FHE; dimension/modulus reduction makes its cost independent of the evaluated depth $L$, and $q/B$ can be taken quasi-polynomial in $n$, basing security on LWE for quasi-polynomial factors. The caveats: bootstrapping needs the secret key encrypted under the public key (in effect an evaluation key), which forfeits the identity-/attribute-based property and has quasi-cubic size, returning the scheme to "unnatural" operations. Finding a natural alternative to bootstrapping and achieving pure FHE without circular security remain open.

Identity-Based and Attribute-Based FHE (Section 4)

IBE and ABE provide flexible access control but traditionally allow no computation on encrypted data, and constructing an IBE allowing even "somewhat" homomorphic encryption was a stubborn open problem. Earlier ideas (e.g. building over dual-Regev with identity-based keys) only made the encryption/decryption keys identity-based, yielding a weak IBFHE because homomorphic evaluation still needed a user-specific evaluation key, which cannot be derived non-interactively from an identity — undermining IBE's non-interactivity.

GSW has no evaluation key, so it gives the first true IBFHE via a compiler. An LWE-based IBE scheme $E$ qualifies if it satisfies three properties:

  • Property 1 (Ciphertext and key vectors): for identity $\mathrm{ID}$, the decryption key $s_{\mathrm{ID}}$ and a ciphertext $c_{\mathrm{ID}}$ are vectors in $\mathbb{Z}_q^{n'}$, and the first coefficient of $s_{\mathrm{ID}}$ is $1$.
  • Property 2 (Small dot product): if $c_{\mathrm{ID}}$ encrypts $0$, then $\langle c_{\mathrm{ID}}, s_{\mathrm{ID}}\rangle$ is small.
  • Property 3 (Security): encryptions of $0$ are indistinguishable from uniform vectors over $\mathbb{Z}_q$ under LWE.

Theorem 4. Any IBE $E$ with Properties 1-3 compiles into an IBFHE. The IBFHE reuses $E$'s $\mathrm{Setup}$ and $\mathrm{KeyGen}$ (adding FHE parameters $m, \ell$ to $\mathrm{MPK}$). To encrypt $\mu \in \{0,1\}$, generate $N$ encryptions of $0$ with $E.\mathrm{Enc}$, stack them as rows of an $N\times(n+1)$ matrix $C'_{\mathrm{ID}}$, and output $C_{\mathrm{ID}} = \mathrm{Flatten}(\mu I_N + \mathrm{BitDecomp}(C'_{\mathrm{ID}}))$. With $v_{\mathrm{ID}} = \mathrm{Powersof2}(s_{\mathrm{ID}})$, decryption runs the GSW $\mathrm{Dec}$: $C_{\mathrm{ID}} v_{\mathrm{ID}} = \mu v_{\mathrm{ID}} + C'_{\mathrm{ID}} s_{\mathrm{ID}} = \mu v_{\mathrm{ID}} + \text{small}$ by Property 2; homomorphic operations are exactly those of Section 3.3. Semantic security follows from Property 3. Several LWE-based IBE schemes (GPV, Agrawal-Boneh-Boyen, Cash-Hofheinz-Kiltz-Peikert) satisfy the requirements.

ABFHE. For attribute-based FHE the authors reinterpret decryption in the Gorbunov-Vaikuntanathan-Wee (GVW) ABE-for-circuits scheme: decrypting under $x$ with key $\mathrm{sk}_y$ where $R(x,y)=1$ is viewed as deriving a sub-key $s_{x,y}$ satisfying a Property-2-like condition (if $c_x$ encrypts $0$ under $x$, then $\langle c_x, s_{x,y}\rangle$ is small). This lets the same compiler apply, producing an ABFHE where ciphertexts under a common index are processed homomorphically with no evaluation key and still decryptable by any party entitled to decrypt the originals. Full IBFHE/ABFHE details are deferred to the full version.

Algorithms

The source contains no figures; the following pseudocode reproduces every algorithm GSW defines (Sections 1.3, 2.3, 3.1, 3.3, and Theorem 4). $\ell = \lfloor\log_2 q\rfloor + 1$ and $N = (n+1)\cdot\ell$ throughout the main scheme. $I_N$ is the $N\times N$ identity.

Gadget transforms (dot-product-preserving).

BitDecomp(a):           # a is k-dimensional over Z_q
  return (a_{1,0},...,a_{1,l-1}, ..., a_{k,0},...,a_{k,l-1})
         where a_{i,j} = j-th bit of a_i (LSB first); output dim N = k*l

BitDecomp^{-1}(a'):     # well-defined even if a' is not 0/1
  return ( sum_j 2^j * a_{1,j}, ..., sum_j 2^j * a_{k,j} )

Flatten(a'):
  return BitDecomp(BitDecomp^{-1}(a'))   # result has 0/1 coefficients

Powersof2(b):           # b is k-dimensional
  return (b_1, 2 b_1, ..., 2^{l-1} b_1, ..., b_k, 2 b_k, ..., 2^{l-1} b_k)

# On matrices, each transform is applied row by row.
# Identities:

#    =  =

Basic encryption scheme (Section 3.1).

Setup(1^lambda, 1^L):
  choose modulus q of kappa = kappa(lambda,L) bits
  choose lattice dim n = n(lambda,L), error dist chi achieving >= 2^lambda security
  choose m = O(n log q)
  l = floor(log q) + 1;   N = (n+1) * l
  return params = (n, q, chi, m)

SecretKeyGen(params):
  sample t  <- Z_q^n
  s = (1, -t_1, ..., -t_n)  in Z_q^{n+1}
  v = Powersof2(s)
  return sk = s    # v is the approximate eigenvector

PublicKeyGen(params, sk):
  sample B <- Z_q^{m x n} uniform,  e <- chi^m
  b = B . t + e
  A = [ b | B ]              # (n+1)-column matrix: b then the n columns of B
  return pk = A             # note A . s = e

Enc(params, pk = A, mu in Z_q):
  sample R <- {0,1}^{N x m} uniform
  C = Flatten( mu * I_N + BitDecomp(R . A) )   in Z_q^{N x N}
  return C                  # C . v = mu * v + R.e

Dec(params, sk, C):         # recovers mu in a small space, e.g. {0,1}
  # first l coeffs of v are 1,2,...,2^{l-1}; pick v_i = 2^i in (q/4, q/2]
  C_i = i-th row of C
  x_i =             # = mu * v_i + error
  return mu' = round( x_i / v_i )

MPDec(params, sk, C):       # Micciancio-Peikert; recovers any mu in Z_q (q a power of 2)
  # q = 2^{l-1}; first l-1 coeffs of v are 1,2,...,2^{l-2}; g = (1,2,...,2^{l-2})
  # C.v = mu*v + small  =>  first l-1 coords are mu*g + small
  recover LSB(mu) from  mu * 2^{l-2} + small
  recover next bit from (mu - LSB(mu)) * 2^{l-3} + small
  ... continue for all bits ...  ; return mu

Homomorphic operations (Section 3.3), all over $\mathbb{Z}_q$.

MultConst(C, alpha):        # multiply by known constant alpha in Z_q
  M_alpha = Flatten(alpha * I_N)
  return Flatten(M_alpha . C)        # error grows by factor <= N, any alpha

Add(C1, C2):
  return Flatten(C1 + C2)

Mult(C1, C2):
  return Flatten(C1 . C2)            # error: mu2*e1 + C1*e2

NAND(C1, C2):               # requires mu1, mu2 in {0,1}
  return Flatten(I_N - C1 . C2)      # output mu in {0,1}; error factor <= N+1

IBE-to-IBFHE compiler (Theorem 4). Given an LWE-based IBE $E$ with Properties 1-3:

IBFHE.Setup / KeyGen:  reuse E.Setup, E.KeyGen;  publish E.MPK plus (m, l)
                       # l = floor(log q)+1,  N = (n+1)*l

IBFHE.Enc(MPK, ID, mu in {0,1}):
  generate N encryptions of 0 via E.Enc(MPK, ID, 0)
  C'_ID = N x (n+1) matrix whose rows are those ciphertexts
  C_ID  = Flatten( mu * I_N + BitDecomp(C'_ID) )
  return C_ID

IBFHE.Dec(sk_ID, C_ID):
  v_ID = Powersof2(s_ID)             # s_ID = E decryption key for ID
  return GSW.Dec(v_ID, C_ID)         # C_ID . v_ID = mu*v_ID + small (Property 2)
