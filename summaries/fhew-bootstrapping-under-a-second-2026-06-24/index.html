FHEW: Bootstrapping Homomorphic Encryption in Less Than a Second

PDF, IACR ePrint 2014/816 · full view

Motivation: bootstrapping is the FHE bottleneck

Every fully homomorphic encryption (FHE) scheme since Gentry's 2009 construction relies on noisy lattice ciphertexts. Each homomorphic operation grows the noise, and once it crosses a threshold the ciphertext stops decrypting. Gentry's bootstrapping fixes this by homomorphically evaluating the decryption circuit on an encrypted secret key, refreshing the ciphertext back to a low noise level. It is what makes unbounded computation possible, and it is the dominant cost in every practical implementation. At the time of writing, the fastest reference implementation (HElib, Halevi and Shoup, Crypto 2014) needed about six minutes per bootstrap.

Ducas and Micciancio study bootstrapping "in vitro": the simplest useful setting. Given two LWE-encrypted bits E(b1) and E(b2), compute an encryption of their logical NAND and return it refreshed to the same low-noise form as the inputs. NAND is complete for boolean circuits, so a fast bootstrapped NAND yields a full FHE. Their prototype performs one homomorphic NAND plus a full refresh in about 0.69 seconds on a single 3 GHz core, roughly 500x faster than HElib's per-bootstrap time, in about 600 lines of C++ (released as FHEW on GitHub). Security rests on worst-case hardness of standard lattice problems (LWE / Ring-LWE) plus the usual circular-security assumption needed to publish a compact evaluation key.

LWE encryption, modulus switching, key switching

The base scheme is plain symmetric LWE. With dimension n, message modulus t, ciphertext modulus q and randomized rounding function chi, a message m in Z_t is encrypted under secret key s in Z_q^n as

LWE_s^{t/q}(m) = (a, chi(a.s + m*q/t) mod q)  in  Z_q^{n+1},   a <- Z_q^n uniform

When t divides q this is (a, a.s + e + m*q/t mod q). Decryption computes m' = round( t*(b - a.s) / q ) mod t. A ciphertext decrypts correctly whenever its error satisfies |err| < q/(2t). The notation LWE_s^{t/q}(m, E) means the subset of encryptions of m whose error is bounded by E.

Two helper conversions are central to the pipeline. Modulus switching rescales a ciphertext from modulus Q to q with the scaled randomized rounding [x]_{Q:q} = floor(q*x/Q) + B, B a Bernoulli rounding bit; the resulting rounding error is subgaussian of parameter sqrt(2*pi). Lemma 5 gives the output error parameter sqrt( (q*sigma/Q)^2 + 2*pi*(||s||^2 + 1) ). Key switching converts a ciphertext under key z into one under a different key s using auxiliary encryptions k_{i,j,v} = LWE_s^{q/q}(v*z_i*B_ks^j); it expands each coordinate a_i in base B_ks and computes

KeySwitch((a,b), K) = (0, b) - sum_{i,j} k_{i,j, a_{i,j}}

Lemma 6: this yields an encryption of the same message with output error parameter sqrt( alpha^2 + N*d_ks*sigma^2 ). To keep key-switching noise small the LWE secret s is taken binary, which is proven as hard as standard LWE (Brakerski et al. 2013).

A new low-noise homomorphic NAND gate

LWE is approximately additively homomorphic: adding E(m1) and E(m2) gives a noisier E(m1+m2). Working mod 2 this only gives XOR. The key trick is to move to arithmetic mod 4 during the cycle. Assume the two input bits arrive in a shifted form: ciphertexts c_i in LWE_s^{4/q}(m_i, q/16), with message modulus t=4 and error bound q/16 (versus the standard t=2, E=q/4). Then NAND is a key-free affine combination.

Lemma 7 gives the algorithm HomNAND mapping two such ciphertexts to LWE_s^{2/q}(NAND, q/4):

HomNAND((a0,b0), (a1,b1)) = ( -a0 - a1 ,  5q/8 - b0 - b1 )

Assuming 2t = 8 divides q so that 5q/8 is an integer, the result (a,b) satisfies

b - a.s - (1 - m0*m1)*q/2  =  (q/4)*( 1/2 - (m0 - m1)^2 )  -  (e0 + e1)
                           =  +/- q/8  -  (e0 + e1)

so (a,b) is a genuine LWE_s^{2/q} encryption of 1 - m0*m1 = m0 NAND m1 with error at most |+/- q/8 - (e0+e1)| < q/8 + q/16 + q/16 = q/4.

HomNAND uses no key material and only a handful of modular additions. The whole homomorphic-circuit problem therefore reduces to a single refreshing function

Refresh : LWE_s^{2/q}(m, q/4)  ->  LWE_s^{4/q}(m, q/16)

which restores the shifted, low-noise input form. One Refresh is needed per gate (inputs are supplied pre-refreshed), so a gate costs essentially one Refresh. Crucially, previous homomorphic AND methods required input error O(sqrt(q)); this construction tolerates O(q), a far weaker demand on the refresh step.

The homomorphic accumulator and the Refresh / bootstrapping procedure

Refresh is bootstrapping: homomorphically evaluate LWE decryption round( 2*(b - a.E(s)) / q ) mod 2 on an encrypted key E(s). Decryption is a scalar product mod q followed by a most-significant-bit (rounding) extraction. Following Alperin-Sheriff and Peikert, an intermediate encryption scheme E with message space Z_q is used: it can encrypt s componentwise and supports affine maps b - a.E(s) = E(b - a.s). The needed structure is abstracted as a Homomorphic Accumulator.

Definition 1: a quadruple (E, Init, Incr, msbExtract) with moduli t, q. Write ACC <- v for Init(v) and ACC +<- E(v) for Incr(ACC, E(v)). After ACC <- v0 then a sequence of increments by E(v1)..E(v_l), ACC is an "l-encryption" of v = sum v_i mod q. It is E-correct if msbExtract(ACC) yields an LWE_s^{t/q}(v, E(l)) ciphertext with overwhelming probability. For this application t=4 and we need E(l) <= q/16.

The refresh procedure is parametrized by base B_r with d_r = ceil(log_{B_r} q). The refreshing key holds K_{i,c,j} = E(c*s_i*B_r^j mod q) for c in 0..B_r-1, j in 0..d_r-1, i in 1..n (about n*B_r*d_r ciphertexts). The algorithm initializes the accumulator to b + q/4, then folds in the scalar product -a.s coordinate by coordinate via base-B_r digits, and extracts the msb:

Algorithm 1.  Refresh_K(a, b),  for K = {K_{i,c,j}}_{i<=n, c<=B_r, j<=d_r}

    ACC <- b + (q/4)
    for i = 1, ..., n do
        Compute the base-B_r representation of  -a_i = sum_j B_r^j * a_{i,j}  (mod q)
        for j = 0, ..., d_r - 1 do  ACC +<- K_{i, a_{i,j}, j}
    end for
    Output msbExtract(ACC).

Theorem 8 (correctness): with a correct accumulator, on input c in LWE_s^{2/q}(m, q/4) and a valid key, Refresh outputs a ciphertext in LWE_s^{t/q}(m, E(n*d)). The implicit accumulator value at the end of the loop is

v - q/4 = b + sum_{i,j} a_{i,j} s_i B_r^j = b + sum_i s_i ( sum_j B_r^j a_{i,j} )
        = b - sum_i a_i s_i = (q/2) m + e

Since |e| < q/4, we get 0 < e + q/4 < q/2, hence 0 < v < q/2 when m=0 and q/2 < v < q when m=1. So msb(v) = m, and msbExtract returns the refreshed encryption. The number of accumulator increments per refresh is n*d_r.

Realizing the accumulator with Ring-GSW external products

The accumulator is instantiated with a private-key, ring variant of the GSW cryptosystem. Work in the power-of-two cyclotomic ring R = Z[X]/(X^N + 1), R_Q = R/QR, with N = 2^K, q dividing 2N, gadget base B_g (assumed a power of 3, Q = B_g^{d_g}). A parameter u, invertible mod Q and close to Q/(2t), is fixed with delta = u - Q/(2t), |delta| < 1. The decisive idea, generalizing Alperin-Sheriff and Peikert, is to encode the additive group Z_q directly into the multiplicative group of roots of unity of R: a message m is carried in the exponent as Y^m, where Y = X^{2N/q}. The roots of unity {1, X, ..., X^{N-1}, -1, ..., -X^{N-1}} form a cyclic group of order 2N, and Z_q embeds as <Y>. This means each Z_q element is one ciphertext, not a vector of |C| binary ciphertexts as in prior work.

The gadget matrix is G = (I, B_g*I, ..., B_g^{d_g-1}*I) in R_Q^{2 d_g x 2}. The scheme operations:

E_z(m):  pick a <- R_Q^{2 d_g} uniform,  e <- subgaussian (param zeta) over R^{2 d_g}
         output  E_z(m) = [ a ,  a.z + e ]  +  u * Y^m * G    in  R_Q^{2 d_g x 2}

Init (ACC <- v):   ACC := u * Y^v * G    in R_Q^{2 d_g x 2}   (noiseless encryption of v)

Incr (ACC +<- C):  decompose  u^{-1} ACC = sum_{i=1}^{d_g} B_g^{i-1} * D_i
                   (each D_i has coefficients in {(1-B_g)/2, ..., (B_g-1)/2})
                   update  ACC := [D_1, ..., D_{d_g}] . C

Incr is exactly the GSW external product: gadget-decompose the accumulator, then multiply by the fresh ciphertext C. Because messages live in the exponent, the multiplicative homomorphism of GSW (Y^m * Y^{m'} = Y^{m+m'}) becomes the additive accumulation the refresh loop needs. Fact 9: if ACC encrypts m and C encrypts m', then ACC +<- C encrypts m + m' with new error e'' = e + [C_1, ..., C_{d_g}] . e'. The asymmetric noise growth (the small, bounded decomposition multiplies the fresh error) is what keeps GSW noise nearly additive.

msb extraction and the testing vector

Extracting the top bit converts the ring accumulator back into a single LWE ciphertext. It uses a testing vector t = - sum_{i=0}^{q/2 - 1} (Y^i as a coefficient vector). The defining property is t . (Y^v as a vector) = -1 for 0 <= v < N and +1 for N <= v < 2N, i.e. t . Y^v = -(-1)^{msb(v)}. (In the simple case q = 2N the testing vector is just -(1,1,...,1).) msbExtract takes a switching key K from the ring key z to the LWE key s and runs:

Algorithm 2.  msbExtract_K(ACC),  K = {k_{i,j,w}}_{i<=N, j<=B_ks, w<=d_ks}
Require: switching key  k_{i,j,w} <- LWE_s^{q/q}( w * z_i * d_ks^j )  from z to s;
         ACC an l-encryption of v.

  1: [a^t, b^t] <- ( [ -0^t, t^t, -0^t, ..., -0^t ] . anticirc(ACC) )  in Z_Q^{2N}
  2: c  <- (a, b_0 + u)                  in LWE_{vec z}^{t/Q}( msb(v) )
  3: c' <- KeySwitch(c, K)               in LWE_s^{t/Q}( msb(v) )
  4: c'' <- ModSwitch(c')                in LWE_s^{t/q}( msb(v) )
  5: Return c'.

Line 1 contracts the second row [a, b'] of the accumulator against the testing vector. Since b' = anticirc(a).z + u*(Y^v vector) + e, and the coordinates of the embedded Y^v sum to (-1)^{msb(v)}, using 1 - (-1)^x = 2x for a bit x gives c = (a, a.vec(z) + t.e + 2u*msb(v)), an LWE_{vec z}^{t/Q} encryption of msb(v) because u is approximately Q/(2t). Key switching moves it to key s, modulus switching brings it down to modulus q, producing the refreshed LWE ciphertext.

Efficient Incr (Algorithm 3) keeps both the accumulator and the bootstrapping-key ciphertexts in FFT/NTT form. Each increment costs 4*d_g backward FFTs and 4*d_g^2 forward FFTs; with the NTT, 4*d_g forward transforms can be traded for a few additions mod Q by reconstructing D_1 from the others.

Noise analysis

Invertibility in R is handled via the factorization X^N + 1 = (X^{N/2} + X^{N/4} - 1)(X^{N/2} - X^{N/4} - 1) mod 3 into two irreducibles (Fact 2), giving fields of order 3^{N/2}; with Hensel lifting (Lemma 3) random matrices over R_Q (Q a power of 3) are invertible with overwhelming probability (Lemma 4). This underlies Fact 12, which bounds the spectral norm of the stacked decomposition matrices: s1([D^(0), ..., D^(l-1)]) = O(B_g * sqrt(N*d_g*l)).

Lemma 11 (intermediate error): assuming Ring-LWE hardness, for an l-encryption with l >= omega(sqrt(log N)), the ciphertext c of line 2 of Algorithm 2 has subgaussian error of parameter

beta = O( zeta * B_g * sqrt( q * N * d_g * l ) )    with mean 2*delta

derived because the final error is e = (testing-vector pattern) . anticirc([D^(0), ..., D^(l-1)]) . (e^(1), ..., e^(l)), where ||t|| = sqrt(q/2), the decomposition block has spectral norm O(B_g sqrt(N d_g l)), and the concatenated fresh errors are subgaussian of parameter zeta. Combining Lemma 11 with the key-switch and modulus-switch noise (Lemmata 6 and 5) gives the full accumulator correctness:

Theorem 10.  Assuming Ring-LWE_{R,Q,chi}, the accumulator is E-correct with

  E(l) = sqrt( (q^2/Q^2) * ( zeta^2 * B_g^2 * l * q * N * d_g  +  sigma^2 * N * d_ks ) )
         +  ||s||^2 * omega( sqrt(log n) ).

Asymptotic parameters: binary secret s; B_g, B_r, B_ks = Theta(1) so d_g, d_r, d_ks = O(log n); sigma, zeta = omega(sqrt(log n)); 2N = q = Theta(n); Q = n^2 * log n * omega(log n). This yields refresh error E(n*d) = O(n) <= q/16, satisfying the NAND requirement. Refresh costs d*n homomorphic operations, versus d*n * O(log^3 q / log log q) in Alperin-Sheriff and Peikert; the pre-switch refresh error is parameter alpha = O(n^2 log n) here versus Theta(n^{5/2} log^3 n / log log n) there, with memory cut by O(log^2 q / log log q).

Concrete parameters, security and the sub-second timing

For the implementation the proof-only constraints (B_g a power of 3, Q a power of B_g) are dropped, requiring only B_g^{d_g} >= Q. Under the central-limit heuristic, refreshed error behaves as a Gaussian of standard deviation beta = sqrt( (q^2/Q^2)*( (zeta^2 B_r^2 / 12)*n*d_r*(q/2)*2*N*d' + sigma^2*N*d_ks ) + (||s||^2 + 1)/12 ), with d' = B_g - 1 + Q/B_g^{d_g} (here d' = 2.5).

Chosen parameters: LWE n = 500, Q = 2^32, sigma = 2^17, q = 2^9; Ring-GSW N = 2^10, zeta = 1.4; gadget B_g = 2^11, d_g = 3, u = Q/8 + 1; bootstrapping key B_r = 2^3, d_r = 2; key-switching key B_ks = 2^4, d_ks = 7. Resulting sizes: bootstrapping key 1032 MB, key-switching key 314 MB, and 48,000 FFTs per NAND gate. Final error parameter beta = 6.94, giving per-NAND error probability p <= 2^-31 (with r = (q/8)/(sqrt(2)*beta)); applying HomNAND before the switches lowers this to 2^-45. Security: delta-binLWE(n, Q, sigma, 2^-64) = 1.0064 and delta-LWE(N, Q, zeta, 2^-64) = 1.0064, corresponding (per BKZ 2.0 estimates) to block size > 190 and > 2^100 enumeration nodes per call. Binary-secret LWE security accounts for the modulus-switching attack via delta-binLWE(n,q,sigma,eps) = min over q' <= q of delta-LWE(n, q', sqrt((q'/q)^2 sigma^2 + n/24), eps).

Implementation uses double-precision complex FFT (FFTW) rather than NTT, with Q a power of two so reduction mod Q is free; a negacyclic FFT of rank N is read off the odd indices of a rank-2N real-data FFT, hitting ~6 Gflops and ~10 microseconds per FFT at 2N = 2048. The product magnitude S = B_g*Q*sqrt(N)/4 = 2^46 sits just within double precision; small floating-point errors fold into the accumulator noise. Result: one HomNAND plus one Refresh runs in 0.69 seconds on a single 3 GHz 64-bit core using 2.2 GB of memory, within an order of magnitude of HElib's amortized per-bit bootstrapping cost, in roughly 600 lines of C++ (FHEW, released on GitHub).

Extensions and conclusions

The technique is not limited to NAND. Working mod 4, a majority gate on x1,x2,x3 is the msb test on the shifted sum (x1+x2+x3) - 0.5. Larger moduli t handle arbitrary weighted threshold gates sum w_i x_i > h. Replacing msbExtract with a membership test for any anti-symmetric set S in Z_t (x in S iff x + t/2 not in S) generalizes further: with t = 6, an arbitrary-fan-in XOR x1 XOR ... XOR xk is one Refresh via the test (x1 + ... + xk mod 6) in {1,3,5}, giving xor-for-almost-free. Since the membership test is cheap relative to the rest of Refresh, multi-output gates (e.g. a 3-input, 2-output add-with-carry at t=6 in a single shot) cost little extra. Taking weighted combinations sum 2^i x_i and testing subsets of Z_{2^{k+2}} can in principle realize any boolean gate, though cost grows exponentially in fan-in k.

Open problems left by the authors: combining this fast sub-second bootstrapping with techniques that do more complex computation between bootstraps, fully exploiting the ring-LWE message space inside the accumulator, and merging with the CRT techniques of Alperin-Sheriff and Peikert. The scheme still needs a circular-security assumption to publish a compact evaluation key.
