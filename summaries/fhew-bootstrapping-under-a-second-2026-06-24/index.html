FHEW: Bootstrapping Homomorphic Encryption in Less Than a Second

EUROCRYPT 2015 (LNCS 9056, pp. 617-640, Springer) · 2014 · full view ↗

Problem and motivation: bootstrapping is the FHE bottleneck

Every fully homomorphic encryption (FHE) scheme since Gentry's 2009 construction is built on noisy lattice ciphertexts. Each homomorphic operation increases the noise, and once the noise crosses a threshold the ciphertext no longer decrypts correctly. Gentry's bootstrapping removes this barrier by homomorphically evaluating the decryption circuit on an encrypted copy of the secret key, producing a fresh low-noise encryption of the same plaintext. Bootstrapping is what makes unbounded homomorphic computation possible, and it is the dominant cost in every practical FHE implementation. At the time of writing, the state-of-the-art reference implementation (HElib, Halevi and Shoup, Crypto 2014) needed about six minutes per bootstrap, making even the simplest unbounded computation macroscopically slow.

Ducas and Micciancio study bootstrapping in vitro, in the simplest useful setting. Given two LWE-encrypted bits $E(b_1)$ and $E(b_2)$, compute an encryption of their logical NAND and return it refreshed to the same low-noise form as the inputs. Concretely, given $c_0, c_1 \in \mathrm{LWE}_s^{2/q}(m_i)$ for $i = 0, 1$, produce $c \in \mathrm{LWE}_s^{2/q}(m)$ where $m = 1 - m_0 m_1 = m_0 \,\bar{\wedge}\, m_1$. NAND is a complete basis for boolean circuits, so a fast fully-bootstrapped NAND yields a full FHE (the private-key scheme converts to public-key by standard techniques). Their prototype performs one homomorphic NAND plus a full refresh in about $0.69$ seconds on a single $3$ GHz $64$-bit core, roughly $500\times$ faster per bootstrap than HElib and within one order of magnitude of HElib's amortized per-bit cost, in about $600$ lines of C++ (released as FHEW on GitHub). Security rests on the worst-case hardness of standard lattice problems (LWE / Ring-LWE) plus the circular-security assumption needed to publish a compact homomorphic evaluation key, which is common to all FHE schemes since Gentry.

The improvement comes from two techniques. First, a new homomorphic NAND gate that operates modulo $4$ instead of modulo $2$ and introduces far less noise than prior homomorphic AND constructions, relaxing the demand on the refresh step. Second, a ring variant of the Alperin-Sheriff and Peikert (CRYPTO 2014) bootstrapping method that encodes each element of $\mathbb{Z}_q$ as a single ciphertext (a root of unity carried in the exponent) rather than as a vector of $|C|$ binary ciphertexts, cutting both time and memory.

Preliminaries: distributions, the cyclotomic ring, and invertibility

Bold lower-case letters $a, b, \dots$ denote column vectors over $\mathbb{Z}$ or a ring $R$, bold upper-case $A, B, \dots$ denote matrices, $\cdot$ is scalar or matrix product as applicable, and $\lVert \cdot \rVert$ is the Euclidean norm. For a vector over $\mathbb{Z}_Q$ the norm means the shortest norm over its residue class in $\mathbb{Z}^n$.

Randomized rounding and subgaussianity. A randomized rounding function $\chi : \mathbb{R} \to \mathbb{Z}$ maps each $x$ to a distribution over $\mathbb{Z}$ with $\chi(x + n) = \chi(x) + n$ for integers $n$; the rounding error is $\chi(x) - x$. A random variable $X$ over $\mathbb{R}$ is subgaussian with parameter $\alpha > 0$ if $\mathbb{E}[\exp(2\pi t X)] \le \exp(\pi \alpha^2 t^2)$ for all $t$, which forces Gaussian tail decay

$$\Pr\{|X| \ge t\} \le 2 \exp\!\big(-\pi t^2 / \alpha^2\big) \qquad \text{for all } t \ge 0.$$

Any $B$-bounded symmetric variable is subgaussian with parameter $B\sqrt{2\pi}$, and the concatenation of independent subgaussian vectors with common parameter $\alpha$ is again subgaussian of parameter $\alpha$.

The cyclotomic ring. Let $N$ be a power of $2$ defining $\Phi_{2N}(X) = X^N + 1$ and the ring $R = \mathbb{Z}[X]/(X^N + 1)$, with $R_Q = R/QR$. Each $a = a_0 + a_1 X + \dots + a_{N-1} X^{N-1}$ is identified with its coefficient vector $\overrightarrow{a} = (a_0, \dots, a_{N-1}) \in \mathbb{Z}^N$, so $R \cong \mathbb{Z}^N$ as an additive group. The ring is also identified with the sub-ring of anti-circulant $N \times N$ matrices: multiplication by $r$ is the linear map $x \mapsto r \cdot x$, whose matrix is $\overrightarrow{\!\!\Rightarrow}\!r$ with first column $\overrightarrow{r}$ (other columns are cyclic rotations with the wrapped entries negated). The spectral norm $s_1(R) = \sup_{x \ne 0} \lVert R x \rVert / \lVert x \rVert$ is consistent between a ring element and its anti-circulant matrix. Fact 1 (adapted from Ducas-Micciancio CRYPTO 2014) bounds the spectral norm of a random matrix $R \leftarrow D^{w \times k}$ with subgaussian entries of parameter $\alpha$: with overwhelming probability $s_1(R) \le \alpha \sqrt{N} \cdot O(\sqrt{w} + \sqrt{k} + \omega(\sqrt{\log N}))$.

Invertibility in $R$. Because $R_Q$ is never a field for a power-of-two cyclotomic, invertibility needs care. For $Q$ a power of $3$ it is close to a field. Fact 2: for $N = 2^k$ with $k \ge 3$,

$$X^N + 1 = (X^{N/2} + X^{N/4} - 1)(X^{N/2} - X^{N/4} - 1) \bmod 3,$$

and both factors are irreducible in $\mathbb{F}_3[X]$, giving two fields $F_1, F_2$ of order $3^{N/2}$. Lemma 3 (Hensel) lifts invertibility from $R_p$ to $R_{p^e}$. Lemma 4: for $Q$ a power of $3$, a random matrix $D \leftarrow \mathcal{D}^{k \times k}$ whose reduction mod $3$ is near-uniform over $R_3$ is invertible with overwhelming probability, since the count of invertible matrices over a field of size $\mathsf{q}$ obeys $\#\mathrm{GL}(k, \mathsf{q}) = \mathsf{q}^{k^2} \prod_{i=0}^{k-1}(1 - \mathsf{q}^{i-k}) \ge \mathsf{q}^{k^2}\big(1 - \tfrac{1}{\mathsf{q}-1}\big)$, so the failure probability is at most $2/(3^{N/2} - 1)$. These facts underpin the noise analysis (Fact 12).

LWE symmetric encryption, modulus switching, key switching

The base scheme is plain symmetric LWE, parametrized by dimension $n$, message modulus $t \ge 2$, ciphertext modulus $q = n^{O(1)}$, and randomized rounding $\chi : \mathbb{R} \to \mathbb{Z}$ (typically $|\chi(x) - x| < q/2t$). The message space is $\mathbb{Z}_t$; the key $s \in \mathbb{Z}_q^n$ is uniform or a short random vector. A message $m \in \mathbb{Z}_t$ encrypts as

$$\mathrm{LWE}_s^{t/q}(m) = \big(a,\ \chi(a \cdot s + m q / t) \bmod q\big) \in \mathbb{Z}_q^{n+1}, \qquad a \leftarrow \mathbb{Z}_q^n \text{ uniform}.$$

When $t \mid q$ this is $(a,\ a \cdot s + e + m q / t \bmod q)$ with $e \leftarrow \chi(0)$. Decryption computes

$$m' = \big\lfloor t(b - a \cdot s)/q \big\rceil \bmod t \in \mathbb{Z}_t.$$

The error of $(a,b) \in \mathrm{LWE}_s^{t/q}(m)$ is $\mathrm{err}(a,b) = (b - a \cdot s - mq/t) \bmod q$ reduced to $[-q/2, q/2]$. Decryption is correct whenever $|\mathrm{err}| < q/2t$, because then $|(t/q) e| < 1/2$. The notation $\mathrm{LWE}_s^{t/q}(m, E)$ denotes encryptions of $m$ with error bounded by $E$.

Modulus switching rescales a ciphertext from modulus $Q$ to $q$ using the scaled randomized rounding $[x]_{Q:q} = \lfloor q x / Q \rfloor + B$, where $B$ is Bernoulli with $\Pr\{B = 1\} = (qx/Q) - \lfloor qx/Q \rfloor$. Then $\mathbb{E}[[x]_{Q:q}] = qx/Q$ and the rounding error $[x]_{Q:q} - qx/Q$ is subgaussian of parameter $\sqrt{2\pi}$. Applied coordinatewise, $\mathrm{ModSwitch}(a, b) = ([a_1]_{Q:q}, \dots, [a_n]_{Q:q},\ [b]_{Q:q})$. Lemma 5: for $c \in \mathrm{LWE}_s^{t/Q}(m)$ with subgaussian error of parameter $\sigma$, the output $[c]_{Q:q}$ is an $\mathrm{LWE}_s^{t/q}(m)$ ciphertext with subgaussian error of parameter

$$\sqrt{(q\sigma/Q)^2 + 2\pi(\lVert s \rVert^2 + 1)}.$$

In practice one uses ordinary rounding $\lfloor \cdot \rceil$, in which case the central-limit heuristic predicts error close to a Gaussian of standard deviation $\sqrt{(q\sigma/Q)^2 + (\lVert s \rVert^2 + 1)/12}$ (the factor $1/12$ is the variance of a uniform on $[-\tfrac12, \tfrac12]$).

Key switching converts a ciphertext under key $z \in \mathbb{Z}_q^N$ into one under a different key $s \in \mathbb{Z}_q^n$. It is parametrized by base $B_{ks}$ with $d_{ks} = \lceil \log_{B_{ks}} q \rceil$, and uses auxiliary ciphertexts $k_{i,j,v} \in \mathrm{LWE}_s^{q/q}(v z_i B_{ks}^j)$ for $i = 1, \dots, N$, $v \in \{0, \dots, B_{ks}\}$, $j = 0, \dots, d_{ks} - 1$ (these are not individually decryptable since the message is interpreted mod $t = q$). It expands each coordinate $a_i = \sum_j a_{i,j} B_{ks}^j$ in base $B_{ks}$ and outputs

$$\mathrm{KeySwitch}((a,b), K) = (0, b) - \sum_{i,j} k_{i,j,a_{i,j}}.$$

Lemma 6: given input error parameter $\alpha$ and switching-key error parameter $\sigma$, the result is an encryption of the same message with output error parameter $\sqrt{\alpha^2 + N d_{ks}\, \sigma^2}$, since the accumulated key-switch error $E = \sum_{i,j} e_{i,j,a_{i,j}}$ is subgaussian of parameter $\sigma\sqrt{N d_{ks}}$. To keep this noise small, the LWE secret $s$ is chosen binary, which Brakerski et al. (STOC 2013) proved as hard as standard LWE.

A new low-noise homomorphic NAND gate

LWE is approximately additively homomorphic: summing $E(m_1)$ and $E(m_2)$ gives a noisier $E(m_1 + m_2)$. Working modulo $2$ this only yields XOR. The key idea is to move to arithmetic modulo 4 during the cycle. Assume the two input bits arrive in a shifted form: ciphertexts $c_i \in \mathrm{LWE}_s^{4/q}(m_i, q/16)$ with message modulus $t = 4$ and error bound $E = q/16$ (versus the standard binary parameters $t = 2$, $E = q/4$). Adding them gives the encryption of a value $\in \{0, 1\}$ when $m_0 \,\bar{\wedge}\, m_1 = 1$ and the value $2$ when $m_0 \,\bar{\wedge}\, m_1 = 0$; moving from that to the encryption of $m_0 \,\bar{\wedge}\, m_1$ is then a key-free affine transformation.

Lemma 7 gives the algorithm $\mathrm{HomNAND} : \mathrm{LWE}_s^{4/q}(m_0, q/16) \times \mathrm{LWE}_s^{4/q}(m_1, q/16) \to \mathrm{LWE}_s^{2/q}(m_0 \,\bar{\wedge}\, m_1,\ q/4)$:

$$\mathrm{HomNAND}\big((a_0,b_0),(a_1,b_1)\big) = \Big(-a_0 - a_1,\ \tfrac{5q}{8} - b_0 - b_1\Big).$$

Assuming $2t = 8$ divides $q$ so that $5q/8$ is an integer, the output $(a,b)$ satisfies

$$b - a \cdot s - (1 - m_0 m_1)\tfrac{q}{2} = \tfrac{q}{4}\Big(\tfrac{1}{2} - (m_0 - m_1)^2\Big) - (e_0 + e_1) = \pm \tfrac{q}{8} - (e_0 + e_1),$$

so $(a,b)$ is a genuine $\mathrm{LWE}_s^{2/q}$ encryption of $1 - m_0 m_1 = m_0 \,\bar{\wedge}\, m_1$ with error

$$\big|\pm \tfrac{q}{8} - (e_0 + e_1)\big| < \tfrac{q}{8} + \tfrac{q}{16} + \tfrac{q}{16} = \tfrac{q}{4}.$$

HomNAND uses no key material and only a handful of modular additions. The entire homomorphic-circuit problem therefore reduces to one refreshing function

$$\mathrm{Refresh} : \mathrm{LWE}_s^{2/q}(m, q/4) \ \longrightarrow\ \mathrm{LWE}_s^{4/q}(m, q/16),$$

which restores the shifted, low-noise input form. Refresh needs key material and is far more expensive than HomNAND, accounting for essentially the whole cost of homomorphic circuit evaluation. Figure 1 shows the full per-gate cycle: HomNAND maps the two refreshed mod-$4$ inputs to a noisy mod-$2$ ciphertext $\mathrm{LWE}_s^{2/q}(m, q/4)$; the accumulator (ACC) operations lift it to a ring ciphertext $\mathrm{LWE}_{\overrightarrow{z}}^{4/Q}(m, E_1(nd_r))$; KeySwitch produces $\mathrm{LWE}_s^{4/Q}(m, E_2(nd_r))$; and ModSwitch brings it back to the refreshed $\mathrm{LWE}_s^{4/q}(m, E(nd_r))$ under the original key. A circuit with $g$ gates needs $n + g$ refreshes (one per input wire and per gate output); supplying inputs pre-refreshed leaves essentially one Refresh per gate. Crucially, previous homomorphic AND methods required input error $O(\sqrt{q})$, whereas this construction tolerates $O(q)$ - a far weaker demand on the refresh step that can make the whole scheme faster.

The homomorphic accumulator and the Refresh procedure

Refresh is exactly Gentry bootstrapping: homomorphically evaluate the LWE decryption $\lfloor 2(b - a \cdot E(s))/q \rceil \bmod 2 \simeq E(m)$ on an encrypted key $E(s)$. Decryption is a scalar product modulo $q$ followed by a most-significant-bit (rounding) extraction. The target scheme $\mathrm{LWE}^{4/q}$ is not versatile enough for this, so following Alperin-Sheriff and Peikert an intermediate scheme $E$ with message space $\mathbb{Z}_q$ is used: it encrypts $s$ componentwise as $E(s) = (E(s_1), \dots, E(s_n))$ and supports affine maps $b - a \cdot E(s) = E(b - a \cdot s)$. The needed structure is abstracted as a Homomorphic Accumulator.

Definition 1: a Homomorphic Accumulator Scheme is a quadruple $(E, \mathrm{Init}, \mathrm{Incr}, \mathrm{msbExtract})$ with moduli $t, q$, where $E$ and $\mathrm{msbExtract}$ may use key material tied to an LWE key $s$. Write $\mathrm{ACC} \leftarrow v$ for $\mathrm{Init}(v)$ and $\mathrm{ACC} \mathrel{+}\!\leftarrow E(v)$ for $\mathrm{Incr}(\mathrm{ACC}, E(v))$. After $\mathrm{ACC} \leftarrow v_0$ followed by increments $E(v_1), \dots, E(v_\ell)$, ACC is an $\ell$-encryption of $v = \sum_i v_i \bmod q$. The scheme is $E$-correct for a function $E$ if, for any $\ell$-encryption ACC of $v$, the output $c \leftarrow \mathrm{msbExtract}(\mathrm{ACC})$ lies in $\mathrm{LWE}_s^{t/q}(v, E(\ell))$ with overwhelming probability. Correctness assumes all increments are freshly and independently generated by direct application of $E$ (not by homomorphic operations on existing ciphertexts). For this application $t = 4$ and the requirement is $E(\ell) \le q/16$.

The refresh procedure is parametrized by a base $B_r$ with $d_r = \lceil \log_{B_r} q \rceil$. The refreshing key holds $K_{i,c,j} = E(c\, s_i B_r^j \bmod q)$ for $c \in \{0, \dots, B_r - 1\}$, $j = 0, \dots, d_r - 1$, $i = 1, \dots, n$, about $n B_r d_r \approx n(B_r/\log B_r)\log q$ ciphertexts. The algorithm initializes the accumulator to $b + q/4$, folds in the scalar product $-a \cdot s$ coordinate by coordinate via base-$B_r$ digits, then extracts the msb:

Algorithm 1.  Refresh_K(a, b)
    key K = { K_{i,c,j} }  for i <= n, c <= B_r, j <= d_r,  K_{i,c,j} = E(c * s_i * B_r^j mod q)

    ACC <- b + (q/4)
    for i = 1, ..., n do
        compute base-B_r representation:  -a_i = sum_j  B_r^j * a_{i,j}   (mod q)
        for j = 0, ..., d_r - 1 do
            ACC +<- K_{i, a_{i,j}, j}
    end for
    Output  msbExtract(ACC)

Theorem 8 (correctness): with a correct accumulator, on input $c \in \mathrm{LWE}_s^{2/q}(m, q/4)$ and a valid refreshing key, Refresh outputs a ciphertext in $\mathrm{LWE}_s^{t/q}(m, E(nd))$. The implicit accumulator value at the end of the loop is

$$v - \tfrac{q}{4} = b + \sum_{i,j} a_{i,j} s_i B_r^j = b + \sum_i s_i \Big(\sum_j B_r^j a_{i,j}\Big) = b - \sum_i a_i s_i = \tfrac{q}{2} m + e,$$

where $e$ is the input error. Since $|e| < q/4$, we get $0 < e + q/4 < q/2$, hence $0 < v < q/2$ when $m = 0$ and $q/2 < v < q$ when $m = 1$. Therefore $\mathrm{msb}(v) = m$ and msbExtract returns the refreshed encryption. The number of accumulator increments per refresh is $n d_r$.

Realizing the accumulator with Ring-GSW external products

The accumulator is instantiated with a private-key, ring variant of the GSW cryptosystem. Work in the power-of-two cyclotomic ring $R = \mathbb{Z}[X]/(X^N + 1)$ with $R_Q = R/QR$, $N = 2^K$, $q \mid 2N$, and gadget base $B_g$. For the proof one assumes $Q = B_g^{d_g}$ and $B_g$ a power of $3$. A parameter $u$ invertible mod $Q$ and close to $Q/2t$ is fixed, with $\delta = u - Q/2t$ and $|\delta| < 1$ (since $Q$ is a power of $3$, one of $\lfloor Q/2t \rfloor, \lceil Q/2t \rceil$ is invertible).

The decisive idea, generalizing Alperin-Sheriff and Peikert, is to encode the additive group $\mathbb{Z}_q$ directly into the multiplicative group of roots of unity of $R$: a message $m \in \mathbb{Z}_q$ is carried in the exponent as $Y^m$ with $Y = X^{2N/q}$. The roots of unity $\langle X \rangle = \{1, X, \dots, X^{N-1}, -1, -X, \dots, -X^{N-1}\}$ form a cyclic group of order $2N \cong \mathbb{Z}_{2N}$, and $\mathbb{Z}_q \cong \langle Y \rangle$ embeds as a subgroup. Each $\mathbb{Z}_q$ element is therefore one ciphertext, not a vector of $|C|$ binary ciphertexts as in prior work. The gadget matrix is $G = (I, B_g I, \dots, B_g^{d_g - 1} I) \in R_Q^{2 d_g \times 2}$. The scheme operations are:

E_z(m):   pick  a <- R_Q^{2 d_g}  uniform,   e <- chi^{2 d_g}  subgaussian param zeta
          output   E_z(m) = [ a ,  a . z + e ]  +  u * Y^m * G     in  R_Q^{2 d_g x 2}

Init (ACC <- v):    ACC := u * Y^v * G       in R_Q^{2 d_g x 2}   (noiseless encryption of v)

Incr (ACC +<- C):   decompose  u^{-1} * ACC = sum_{i=1..d_g} B_g^{i-1} * D_i
                    each D_i in R^{2 d_g x 2}, coeffs in {(1-B_g)/2, ..., (B_g-1)/2}
                    update  ACC := [D_1, ..., D_{d_g}] . C

Incr is exactly the GSW external product: gadget-decompose the accumulator, then multiply by the fresh ciphertext $C$. Init sets ACC to a trivial noiseless encryption of $v$. Because messages live in the exponent, the multiplicative homomorphism of GSW, $Y^m \cdot Y^{m'} = Y^{m + m'}$, becomes the additive accumulation the refresh loop requires. Fact 9: if $\mathrm{ACC} = [a, a \cdot z + e] + u Y^m G$ and $C = [a', a' \cdot z + e'] + u Y^{m'} G$, then $\mathrm{ACC} \mathrel{+}\!\leftarrow C$ has the form $[a'', a'' \cdot z + e''] + u Y^{m+m'} G$ with

$$e'' = e + [C_1, \dots, C_{d_g}] \cdot e'.$$

The asymmetric noise growth - the small, bounded decomposition multiplies only the fresh error $e'$ - is what keeps GSW noise nearly additive across the $nd_r$ increments.

Most-significant-bit extraction and the testing vector

Extracting the top bit converts the ring accumulator back into a single LWE ciphertext. It uses a testing vector

$$t = -\sum_{i=0}^{q/2 - 1} \overrightarrow{Y^i},$$

where each $\overrightarrow{Y^i}$ is the coefficient vector of $Y^i$. Writing $\overrightarrow{Y^v}$ as $x_{v \cdot 2N/q}$ with $x_i = \overrightarrow{X^i}$ for $0 \le i < N$ and $x_i = -x_{i-N}$ otherwise, the coordinates of $x_i$ sum to $(-1)^{\mathrm{msb}(i)}$, so the defining property is $t \cdot \overrightarrow{Y^v} = -(-1)^{\mathrm{msb}(v)}$, that is $-1$ for $0 \le v < N$ and $+1$ for $N \le v < 2N$. (In the simple case $q = 2N$ the testing vector is just $t = -(1, 1, \dots, 1)$.) msbExtract takes a switching key $K$ from the ring key $z$ to the LWE key $s$:

Algorithm 2.  msbExtract_K(ACC)
Require: switching key  K = { k_{i,j,w} }  for i <= N, j <= B_ks, w <= d_ks,  from z to s,
              k_{i,j,w} <- LWE_s^{q/q}( w * z_i * d_ks^j )
         ACC is an l-encryption of v

  1:  [ a^t , b^t ]  <-  ( [ -0^t, t^t, -0^t, ..., -0^t ] . anticirc(ACC) )    in  Z_Q^{2N}
        // a = t^t . anticirc(a),  contracting the testing vector against ACC's second row
  2:  c   <-  ( a , b_0 + u )                in LWE_{vec z}^{t/Q}( msb(v) )
  3:  c'  <-  KeySwitch(c, K)                in LWE_s^{t/Q}( msb(v) )
  4:  c'' <-  ModSwitch(c')                  in LWE_s^{t/q}( msb(v) )
  5:  Return c'

Let $[a, b']$ be the second row of $\mathrm{ACC} \in R^{2 d_g \times 2}$. If ACC is a GSW encryption of $v$, then $\overrightarrow{b'} = \overrightarrow{\!\!\Rightarrow}\!a \cdot \overrightarrow{z} + u \cdot \overrightarrow{Y^v} + e$ for small error $e$. Line 1 contracts the testing vector against ACC's second row, and using the identity $1 - (-1)^x = 2x$ for a bit $x$ gives

$$c = (a,\ b_0 + u) = \big(a,\ a \cdot \overrightarrow{z} + t \cdot e + 2u\, \mathrm{msb}(v)\big), \qquad a = t^{\mathsf{T}} \cdot \overrightarrow{\!\!\Rightarrow}\!a,$$

an $\mathrm{LWE}_{\overrightarrow{z}}^{t/Q}$ encryption of $\mathrm{msb}(v)$ because $u \approx Q/2t$. Key switching (line 3) moves it to key $s$, and modulus switching (line 4) brings it down to modulus $q$, producing the refreshed LWE ciphertext.

Efficient increment. To make Incr fast, both the accumulator and the bootstrapping-key ciphertexts are kept in FFT/NTT form:

Algorithm 3.  Incr( hatACC in (FFT domain),  C in (FFT domain) )
    ACC <- FFT^{-1}( hatACC )
    decompose  u^{-1} * ACC = sum_{i=1..d_g} B_g^{i-1} * D_i
    set  D = [ D_1, ..., D_{d_g} ]   in  R^{2 d_g x 2 d_g}
    D <- FFT(D)
    Return  D (.) C        // pointwise product in the FFT domain

Each increment costs $4 d_g$ backward FFTs and $4 d_g^2$ forward FFTs. With the NTT, $4 d_g$ forward transforms can be traded for a few additions mod $Q$ by reconstructing the last block from the others: $D_1 = u^{-1}\mathrm{ACC} - \sum_{i=2}^{d_g} B_g^{i-1} D_i \bmod Q$.

Noise analysis and asymptotic correctness

Invertibility (Facts 2-4) lets one treat fresh ciphertexts as uniform random matrices under Ring-LWE. Fact 12 bounds the spectral norm of the stacked decomposition matrices: for fresh $C^{(i)} \leftarrow E_z(v^{(i)})$ with $\ell = \omega(\sqrt{\log n})$, setting $D^{(i)}$ to be the base-$B_g$ decomposition blocks of $u^{-1}\mathrm{ACC}^{(i)}$, the proof shows by induction that the $D^{(i)}$ are independent uniform over $R_{[B_g]}^{2 d_g \times 2 d_g}$ (coefficients in $\{(1-B_g)/2, \dots, (B_g-1)/2\}$) and invertible, hence by Fact 1

$$s_1\big([D^{(0)}, \dots, D^{(\ell-1)}]\big) = O\big(B_g \sqrt{N d_g\, \ell}\big).$$

Lemma 11 (intermediate error): assuming Ring-LWE hardness, for an $\ell$-encryption with $\ell \ge \omega(\sqrt{\log N})$, the ciphertext $c$ of line 2 of Algorithm 2 has subgaussian error with mean $2\delta$ and parameter

$$\beta = O\big(\varsigma\, B_g \sqrt{q\, N d_g\, \ell}\big).$$

This follows because applying Fact 9 $\ell$ times gives $\mathrm{ACC}^{(\ell)} = [A, A z + e] + u X^v G$ with $e = \sum_{i=1}^{\ell} D^{(i-1)} e^{(i)}$, so the final error is $e = [\overrightarrow{0}^{\mathsf{T}}, t^{\mathsf{T}}, \overrightarrow{0}^{\mathsf{T}}, \dots] \cdot \overrightarrow{\!\!\Rightarrow}\!\big([D^{(0)}, \dots, D^{(\ell-1)}]\big) \cdot \overrightarrow{(e^{(1)}, \dots, e^{(\ell)})}$, where $\lVert t \rVert = \sqrt{q/2}$, the decomposition block has spectral norm $O(B_g \sqrt{N d_g \ell})$, and the concatenated fresh errors are subgaussian of parameter $\varsigma$. Combining Lemma 11 with the key-switch and modulus-switch noise (Lemmata 6 and 5) gives:

Theorem 10. Assuming $\text{Ring-LWE}_{R,Q,\chi}$, the accumulator is $E$-correct with

$$E(\ell) = \sqrt{\frac{q^2}{Q^2}\big(\varsigma^2 B_g^2\, \ell\, q\, N d_g + \sigma^2 N d_{ks}\big)} + \lVert s \rVert^2 \cdot \omega\big(\sqrt{\log n}\big).$$

Asymptotic parameters. Take a binary secret $s$ (to minimize the $\lVert s \rVert$-dependent term); bases $B_g, B_r, B_{ks} = \Theta(1)$ so $d_g, d_r, d_{ks} = O(\log n)$; error parameters $\sigma, \varsigma = \omega(\sqrt{\log n})$; ring dimension $2N = q = \Theta(n)$; and $Q = n^2 \cdot \log n \cdot \omega(\log n)$. This yields refresh error $E(nd) = O(n) \le q/16$, satisfying the NAND requirement.

Efficiency comparison. Refresh is dominated by $dn$ homomorphic operations, versus $dn \cdot O(\log^3 q / \log\log q)$ in Alperin-Sheriff and Peikert. That polylog factor is large in practice: for $q = 2 \cdot 3 \cdot 5 \cdot 7 \cdot 11 = 2310$ it is $2^2 + 3^2 + 5^2 + 7^2 + 11^2 = 208$, while memory shrinks by $O(\log^2 q / \log\log q)$ (a factor $28$ in that example). The scheme also avoids randomized decomposition in Incr, simplifying the implementation. The pre-switch refresh error has parameter $\alpha = O(n^2 \log n)$ here versus $\Theta(n^{5/2} \log^3 n / \log\log n)$ there.

Security estimation methodology

Security follows the Lindner-Peikert analysis. To build an $\epsilon$-distinguisher against LWE in dimension $n$, modulus $q$, and rounding standard deviation $\sigma$, the best known lattice-reduction attack must reach root Hermite factor

$$\delta = \delta\text{-LWE}(n, q, \sigma, \epsilon) = 2^{(\log_2^2 \rho)/(4 n \log_2 q)}, \qquad \rho = \frac{q}{\sigma}\sqrt{2 \ln(1/\epsilon)}.$$

Binary-secret LWE is weaker than uniform-secret LWE because a binary secret lets an attacker switch to a smaller modulus $q' \le q$ without much affecting the relative error: a switch from $q$ to $q'$ yields $\text{binLWE}$ samples with error parameter $\sigma' = \sqrt{(q'/q)^2 \sigma^2 + \lVert s \rVert/12} \approx \sigma q'/q$ (this is precisely the property exploited for the scheme's own correctness), whereas the same switch on standard LWE gives $\sigma' \approx \sigma \sqrt{n}$. The binary-secret security is therefore estimated as the minimum over moduli:

$$\delta\text{-binLWE}(n, q, \sigma, \epsilon) = \min_{q' \le q}\ \delta\text{-LWE}\!\Big(n,\ q',\ \sigma' = \sqrt{(q'/q)^2 \sigma^2 + n/24},\ \epsilon\Big),$$

computed by standard numerical optimization. To thwart an attack that shifts every coordinate of $s$ by $-1/2$, the signs of the entries of $s$ are randomized (which only helps security and does not affect the error analysis), and one assumes $\lVert s \rVert \le n/2$ (true for half of all binary secrets; otherwise resample). The Ring-GSW scheme is evaluated by $\delta\text{-LWE}(N, Q, \varsigma, \epsilon)$, ignoring the ring structure since no algorithm is known to exploit it.

Concrete parameters, implementation and the sub-second benchmark

For the implementation the proof-only constraints ($B_g$ a power of $3$, $Q$ a power of $B_g$) are dropped, requiring only $B_g^{d_g} \ge Q$; the authors verified this relaxation does not significantly change the distribution of $\mathrm{err}(\mathrm{Refresh}(c))$. Under the central-limit heuristic the refreshed error behaves as a Gaussian of standard deviation

$$\beta = \sqrt{\frac{q^2}{Q^2}\Big(\frac{\varsigma^2 B_r^2}{12}\cdot n d_r \cdot \frac{q}{2}\cdot 2 N d' + \sigma^2 N d_{ks}\Big) + \frac{\lVert s \rVert^2 + 1}{12}},$$

where $d' = B_g - 1 + Q/B_g^{d_g}$ accounts for the smaller bound on the final decomposition coordinate ($d' = 2.5$ here instead of $d_g = 3$), and the $1/12$ factors are the variance of a uniform on $[-\tfrac12, \tfrac12]$.

Chosen parameters. LWE: $n = 500$, $Q = 2^{32}$, $\sigma = 2^{17}$, $q = 2^9$. Ring-GSW: $N = 2^{10}$, $\varsigma = 1.4$. Gadget matrix: $B_g = 2^{11}$, $d_g = 3$, $u = Q/8 + 1$. Bootstrapping key: $B_r = 2^3$, $d_r = 2$. Key-switching key: $B_{ks} = 2^4$, $d_{ks} = 7$.

Resulting sizes and counts. Bootstrapping key $4 n N d_r B_r d_g \log_2 Q$ bits $= 1032$ MB; key-switching key $n N B_{ks} d_{ks} \log_2 Q$ bits $= 314$ MB; FFTs per NAND gate $4 n d_r d_g (d_g + 1) = 48{,}000$. Final error parameter $\beta = 6.94$, giving per-NAND error probability $p = 1 - \mathrm{erf}(r/\sqrt{2}) \le 2^{-31}$ with $r = (q/8)/(\sqrt{2}\,\beta)$ (the bound uses $|\mathrm{err}(c_1) + \mathrm{err}(c_2)| < q/8$, sufficient but looser than $|\mathrm{err}(c_i)| < q/16$); applying HomNAND before the switches lowers this to $2^{-45}$.

Security. $\delta\text{-binLWE}(n, Q, \sigma, 2^{-64}) = 1.0064$ for the LWE scheme and $\delta\text{-LWE}(N, Q, \varsigma, 2^{-64}) = 1.0064$ for the Ring-GSW scheme. Per BKZ 2.0 estimates, reaching root Hermite factor $1.0065$ needs block size $> 190$, and each enumeration call would visit more than $2^{100}$ nodes of the pruned tree (a preliminary estimate, pending more involved analyses such as Liu-Nguyen).

FFT implementation. To avoid prime-field arithmetic and costly reduction mod $Q$, the prototype uses the complex FFT (FFTW) with primitive $2N$-th root $\omega = \exp(2\pi \imath / 2N)$ rather than an NTT, choosing $Q$ a power of two so reduction mod $Q$ is essentially free. The needed negacyclic FFT of rank $N$ is read off the odd indices of a rank-$2N$ FFT, with a further factor-$2$ saving from real-input data. With vectorized instructions this reaches about $6$ Gflops on a single $3$ GHz $64$-bit Intel core, around $10$ microseconds per FFT at $2N = 2048$. Products of two integer polynomials, one with coefficients $< B_g/2$, have magnitude $S = B_g Q \sqrt{N}/4 = 2^{46}$; correctness needs final relative error $\epsilon$ with $S\epsilon \le 1/2$. At double precision FFTW reports error standard deviation $\approx 2^{-52} \approx \epsilon_0 \sqrt{\log N}$ (with $\epsilon_0 = 2^{-54}$), barely sufficient; any small floating-point error simply folds into the accumulator noise as an extra per-operation term.

Result. One HomNAND plus one Refresh runs in $0.69$ seconds on a single $3$ GHz $64$-bit core using $2.2$ GB of memory (about twice the prediction, because the bootstrapping key is stored in FFT form at double precision). This is within one order of magnitude of HElib's amortized per-bit bootstrapping cost, in roughly $600$ lines of C++ (FHEW, released on GitHub).

Extensions, limitations and open problems

The technique is not limited to NAND. Still using arithmetic modulo $4$, a majority gate on $x_1, x_2, x_3 \in \{0,1\}$ is computed by forming $y = x_1 + x_2 + x_3 \in \{0,1,2,3\}$ and testing $y \ge 2$, which is the msb test applied to the shifted sum $y - 0.5$; the computation is almost identical to NAND (adding three ciphertexts may need slightly smaller noise). This generalizes to arbitrary weighted threshold gates $\sum_i w_i x_i > h$ via arithmetic modulo a larger $t > 2 \sum_i |w_i|$.

Replacing msbExtract with a membership test for any anti-symmetric set $S \subset \mathbb{Z}_t$ (i.e. $x \in S \iff x + t/2 \notin S$) generalizes further. With $t = 6$, an arbitrary fan-in XOR $x_1 \oplus \dots \oplus x_k$ is one Refresh via the test $(x_1 + \dots + x_k \bmod 6) \in \{1, 3, 5\}$ - giving xor-for-almost-free as in prior FHE schemes. Because the membership test is much cheaper than the rest of Refresh, multi-output gates cost little extra: at $t = 6$ an add-with-carry ($3$ inputs, $2$ outputs) runs in a single shot instead of $5$ binary gates. Taking weighted combinations $\sum_i 2^i x_i$ and testing subsets of $\mathbb{Z}_{2^{k+2}}$ can in principle realize any boolean gate (adders, S-boxes), though cost grows exponentially in fan-in $k$.

Limitations. Like all FHE since Gentry, releasing a compact homomorphic evaluation key requires a circular-security assumption. The benchmark problem is genuinely simpler than HElib's (a single bit operation per bootstrap, no SIMD packing), and the security parameters and gate set differ, so a precise head-to-head comparison is hard. The preliminary security estimate would benefit from more refined cryptanalysis. Open problems: combining this fast sub-second bootstrapping with techniques that perform substantially more complex computation between bootstraps; fully exploiting the ring-LWE message space inside the accumulator; merging with the CRT techniques of Alperin-Sheriff and Peikert; and realizing more complex multi-output gates.

HomNAND( c0 = (a0, b0),  c1 = (a1, b1) )      // Lemma 7
Input:   c0, c1 in LWE_s^{4/q}(m0, q/16) and LWE_s^{4/q}(m1, q/16)
Output:  c in LWE_s^{2/q}(m0 NAND m1, q/4)
  1: a <- -a0 - a1                  (mod q)
  2: b <- (5q/8) - b0 - b1          (mod q)    // assumes 8 = 2t divides q
  3: Return (a, b)
ModSwitch( c = (a, b) ) : Z_Q^{n+1} -> Z_q^{n+1}      // Lemma 5
  for each coordinate x in (a_1, ..., a_n, b):
      [x]_{Q:q} = floor(q*x/Q) + B,   B ~ Bernoulli( (q*x/Q) - floor(q*x/Q) )
  Return ( [a_1], ..., [a_n], [b] )
KeySwitch( c = (a, b),  K )                          // Lemma 6, eq. (4)
Input:   c in LWE_z^{t/q}(m);   K = { k_{i,j,v} = LWE_s^{q/q}(v * z_i * B_ks^j) }
Output:  c' in LWE_s^{t/q}(m)
  1: for i = 1, ..., N:  write  a_i = sum_j a_{i,j} * B_ks^j   (base B_ks)
  2: Return  (0, b) - sum_{i,j} k_{i, j, a_{i,j}}
E_z(m)          // Ring-GSW encryption, message in the exponent
  a <- R_Q^{2 d_g}  uniform ;   e <- chi^{2 d_g}  subgaussian param zeta
  Return  [ a ,  a . z + e ]  +  u * Y^m * G       in R_Q^{2 d_g x 2}
  where G = (I, B_g I, ..., B_g^{d_g - 1} I)

Init(v):   ACC := u * Y^v * G                        // noiseless encryption of v

Incr(ACC, C):                                        // GSW external product
  decompose  u^{-1} * ACC = sum_{i=1..d_g} B_g^{i-1} * D_i   (coeffs in {(1-B_g)/2 .. (B_g-1)/2})
  ACC := [ D_1, ..., D_{d_g} ] . C
Algorithm 1.  Refresh_K(a, b)                        // Theorem 8
Input:   (a, b) in LWE_s^{2/q}(m, q/4)
         refreshing key K = { K_{i,c,j} = E(c * s_i * B_r^j mod q) },
                            c in {0..B_r-1}, j in {0..d_r-1}, i in {1..n}
Output:  c in LWE_s^{t/q}(m, E(n*d_r))
  1: ACC <- b + (q/4)                                // Init
  2: for i = 1, ..., n do
  3:     write  -a_i = sum_{j} B_r^j * a_{i,j}        (mod q,  base B_r)
  4:     for j = 0, ..., d_r - 1 do
  5:         ACC +<- K_{i, a_{i,j}, j}                // Incr
  6: Return  msbExtract_K(ACC)
Algorithm 2.  msbExtract_K(ACC)                      // Lemma 11 / Theorem 10
Input:   ACC, an l-encryption of v under ring key z
         switching key K = { k_{i,j,w} = LWE_s^{q/q}(w * z_i * d_ks^j) }, z -> s
         testing vector  t = - sum_{i=0}^{q/2 - 1}  vec(Y^i)
Output:  c'' in LWE_s^{t/q}( msb(v) )                // = refreshed LWE ciphertext
  1: [ a^t, b^t ] <- [ -0^t, t^t, -0^t, ..., -0^t ] . anticirc(ACC)     in Z_Q^{2N}
  2: c   <- ( a, b_0 + u )           in LWE_{vec z}^{t/Q}( msb(v) )     // u approx Q/2t
  3: c'  <- KeySwitch(c, K)          in LWE_s^{t/Q}( msb(v) )
  4: c'' <- ModSwitch(c')            in LWE_s^{t/q}( msb(v) )
  5: Return c''
