### Automation
- `automation/run-weekly.sh` - Friday cron (`0 12 * * 5`), writes one post per paper
- `automation/run-conferences.sh` - monthly cron (`13 9 1 * *`), refreshes conferences
- `automation/poll-and-summarize.sh` - polls GitHub issues for deep-summary requests; runs during weekday 17:30-17:55 slot and off-peak evenings/weekends
- `automation/publish-summaries.py` - copies `~/research-material/**/summary.html` into `summaries/`, writes `_data/summaries.yml`
- `automation/sync-summaries.sh` - wired as global Stop hook in `~/.claude/settings.json`; runs publish-summaries.py + commits + pushes on every claude session exit
- `automation/logs/git-push.log` - logged every time Claude runs a `git push` (PostToolUse Bash hook in settings.json)

**All headless `claude -p` calls run on `--model opus`** (switched from sonnet this session for better deep-summary quality).

### Bookmark system (localStorage)
Two lists stored in `localStorage['rt_lists']`:
- Key `saved` = **Bookmarked** list (fa-bookmark, gold #f0a500)
- Key `toread` = **Saved** list (fa-book-open, blue #4ea1d3)

**The two lists are mutually exclusive** - adding to one removes from the other.

Cards with items in either list get a colored left-border accent (gold=Bookmarked, blue=Saved) on ALL pages (`.paper-card[data-bm]` CSS in bookmarks-js.html).

Items stored per entry: `{ title, url, cat, kind, tags[] }` - `tags` added this session for sub-category grouping.

**My Research / sidebar lists** render with:
1. Deep Summaries section first
2. Then categories in TOPICS order
3. Within each category: sub-groups by `tags[0]` if items have tags (e.g. TFHE, FPGA under FHE)
4. Group headings: small uppercase label style (`.rt-group h3`, `.rt-group h4`)

### Deep Summaries page (`_layouts/summaries.html`)
- Category filter tiles + "Bookmarked" tile (right-aligned)
- Each summary card has: Save (book-open) + Bookmark (bookmark) + Archive (eye-slash) buttons
- Archive (eye-slash): hides card from listing, stored in `localStorage['sm_hidden']`
- "Show archived (N)" expander at bottom when any items are archived
- MutationObserver keeps "Bookmarked" filter live when bookmarks change
- `window._rtRenderAll` exported from bookmarks-js.html for summaries.html to call

### Deep Summaries on-demand flow
1. Paper card "Generate Deep Summary" button opens a prefilled GitHub issue (label `summary-request`)
2. Poller picks it up, runs `claude -p automation/summarize-request-prompt.md` via `summarize-anything` (model: opus)
3. Stop hook publishes; button flips to "View Deep Summary" (matches by arXiv id)

### summarize-anything skill
- Path: `/home/nik/.claude/skills/summarize-anything/`
- CLI: `python3 -m scripts.cli {probe|detect|parse-pdf|build-html}`
- Outputs to `~/research-material/<category>/<slug>-<date>/{summary.html,content.json,assets/,source.pdf}`
- IMPORTANT: in batch runs, each agent must read its own PDF directly - concurrent agents sharing a temp filename cause the wrong paper's text to be used (race condition)
- ePrint (eprint.iacr.org) blocks direct fetch - use Wayback Machine

### Usage gating
- `~/.claude/statusline.py` writes `~/.claude/rate-limits-cache.json` with `rate_limits` + `written_at`
- Poller skips if `five_hour` or `seven_day` `used_percentage` >= 95
